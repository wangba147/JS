为什么需要类与模块

MOTIVATION

🤔 从混乱到有序

到目前为止，我们写的代码都是"散装"的——函数、变量都丢在全局作用域。当代码量超过几百行，这种方式就会崩溃：

  • 命名冲突：多个文件里都有 getData() 函数，互相覆盖
  • 依赖混乱：不知道谁依赖谁，加载顺序错了就报错
  • 难以复用：相似功能的代码到处复制粘贴
  • 难以维护：改一个地方可能影响全局

类的本质是将数据（属性）和行为（方法）打包在一起。模块的本质是将相关的类、函数、变量封装在独立文件中。

❌ 散装代码（不推荐）

// 全局变量到处都是
let viewerLat = 39.9;
let viewerLon = 116.4;
let viewerZoom = 15;

function setView(lat, lon, zoom) {
  viewerLat = lat;
  viewerLon = lon;
  viewerZoom = zoom;
}

function getView() {
  return { lat: viewerLat, lon: viewerLon, zoom: viewerZoom };
}

✅ 类 + 模块（推荐）

// CameraView.js 模块
export class CameraView {
  #lat;
  #lon;
  #zoom;

  constructor(lat, lon, zoom) {
    this.#lat = lat;
    this.#lon = lon;
    this.#zoom = zoom;
  }

  setView(lat, lon, zoom) {
    this.#lat = lat;
    this.#lon = lon;
    this.#zoom = zoom;
  }

  getView() {
    return { lat: this.#lat, lon: this.#lon, zoom: this.#zoom };
  }
}

class 基本语法

SYNTAX

🔧 class 关键字

ES6 引入 class 关键字，让面向对象编程更直观。注意：这只是语法糖，底层仍是原型继承。

// 定义一个类（首字母通常大写）
class Marker {
  // 构造函数：创建实例时自动调用
  constructor(name, lat, lon) {
    this.name = name;
    this.lat  = lat;
    this.lon  = lon;
  }

  // 实例方法
  getPosition() {
    return `${this.name}: (${this.lat}, ${this.lon})`;
  }
}

// 使用 new 关键字创建实例
const m1 = new Marker('北京', 39.9, 116.4);
const m2 = new Marker('上海', 31.2, 121.5);

console.log(m1.getPosition()); // "北京: (39.9, 116.4)"
console.log(m2.getPosition()); // "上海: (31.2, 121.5)"
console.log(typeof Marker);         // "function" ← 类本质是函数
ℹ️
关键点：new 关键字是必须的。不用 new 调用类会抛出 TypeError。这与普通函数不同。

构造函数 constructor

CONSTRUCTOR

🏗️ 对象的"出生过程"

constructor 是类的特殊方法，在 new 创建实例时自动调用，用于初始化对象属性。一个类只能有一个构造函数。

class GeoPoint {
  constructor(lat, lon, altitude = 0) {
    this.latitude  = lat;
    this.longitude = lon;
    this.altitude  = altitude; // 默认值 0
    this.timestamp  = Date.now();   // 自动记录创建时间
  }

  info() {
    return `坐标：(${this.latitude}, ${this.longitude}, ${this.altitude}m)`;
  }
}

const p1 = new GeoPoint(30.5, 114.3);      // altitude = 0（默认值）
const p2 = new GeoPoint(30.5, 114.3, 8848); // altitude = 8848

console.log(p1.info()); // "坐标：(30.5, 114.3, 0m)"
console.log(p2.info()); // "坐标：(30.5, 114.3, 8848m)"
💡
可以不写 constructor：如果你不写 constructor，JavaScript 会自动生成一个空的 constructor() {}，相当于无参构造。

实例方法与 this

METHODS

🔍 this 的指向

在类的方法中，this 指向调用该方法的实例对象。但要注意——JavaScript 中 this 是动态的，容易丢失上下文。

class Layer {
  constructor(name) {
    this.name = name;
    this.visible = true;
  }

  toggle() {
    this.visible = !this.visible;
    console.log(`${this.name}: ${this.visible ? '显示' : '隐藏'}`);
  }
}

const layer = new Layer('建筑模型');
layer.toggle(); // "建筑模型: 隐藏"
layer.toggle(); // "建筑模型: 显示"

// ⚠️ 陷阱：this 丢失
const fn = layer.toggle;
// fn(); ← 报错！this 变成了 undefined（严格模式）

// ✅ 修复方案1：箭头函数属性（推荐）
class Layer2 {
  constructor(name) { this.name = name; }
  toggle = () => { // 箭头函数绑定 this
    console.log(this.name);
  };
}

// ✅ 修复方案2：.bind()
const fn2 = layer.toggle.bind(layer);
fn2(); // 正常工作
⚠️
this 丢失是高频坑：当你把方法作为回调传递（如事件监听器、setTimeout）时，this 很容易丢失。箭头函数属性是最简洁的解决方案。

静态方法与属性

STATIC

📚 属于类本身，不属于实例

static 关键字定义的方法和属性，只能通过类名访问，不能通过实例访问。适合定义工具函数、常量、工厂方法等。

class GeoUtils {
  // 静态属性（ES2022+）
  static EARTH_RADIUS = 6371;        // km
  static KM_PER_DEGREE = 111.32;    // 每度约 111km

  // 静态方法：度数转弧度
  static toRadians(degrees) {
    return degrees * Math.PI / 180;
  }

  // 静态方法：计算两点近似距离（简化版）
  static approxDistance(lat1, lon1, lat2, lon2) {
    const dLat = (lat2 - lat1) * this.KM_PER_DEGREE;
    const dLon = (lon2 - lon1) * this.KM_PER_DEGREE
                * Math.cos(this.toRadians(lat1));
    return Math.sqrt(dLat ** 2 + dLon ** 2);
  }

  // 静态工厂方法
  static fromString(coordStr) {
    const [lat, lon] = coordStr.split(',').map(Number);
    return { lat, lon };
  }
}

// 直接通过类名调用，无需 new
console.log(GeoUtils.EARTH_RADIUS);              // 6371
console.log(GeoUtils.toRadians(180));            // 3.1415...
console.log(GeoUtils.approxDistance(39.9, 116.4, 31.2, 121.5));
// 约 1069 km（北京到上海）

// ⚠️ 不能通过实例调用
const g = new GeoUtils();
// g.toRadians(90); ← TypeError!

私有字段（#）

PRIVATE

🔒 真正私有的属性

ES2022 引入 # 前缀来定义私有字段。私有字段只能在类内部访问，外部完全不可见，也无法被继承。

class BankAccount {
  // 私有字段：外部无法访问
  #balance = 0;
  #pin;

  constructor(initialBalance, pin) {
    this.#balance = initialBalance;
    this.#pin = pin;
  }

  // 公有方法：可以访问私有字段
  deposit(amount) {
    if (amount > 0) this.#balance += amount;
    return this.#balance;
  }

  withdraw(amount, pin) {
    if (pin !== this.#pin) return '密码错误';
    if (amount <= this.#balance) {
      this.#balance -= amount;
      return `取款成功，余额：${this.#balance}`;
    }
    return '余额不足';
  }

  // 私有方法（ES2022+）
  #validateAmount(amount) {
    return amount > 0 && Number.isFinite(amount);
  }
}

const account = new BankAccount(1000, '1234');
console.log(account.deposit(500)); // 1500
console.log(account.balance);      // undefined（私有字段不可见）
// account.#balance; ← SyntaxError!
// account.#pin;     ← SyntaxError!
ℹ️
命名约定对比：在 # 出现之前，开发者用 _balance（下划线前缀）表示"约定私有"，但这只是约定，外部依然可以访问。# 是真正的语言级私有。

getter / setter

ACCESSORS

🎯 像访问属性一样调用方法

get 和 set 让你定义"计算属性"——访问时像普通属性，但背后执行的是方法。常用于数据验证和格式化。

class Temperature {
  #celsius = 0;

  constructor(celsius) {
    this.#celsius = celsius;
  }

  // getter：读取时计算
  get celsius() {
    return this.#celsius;
  }

  get fahrenheit() {
    return this.#celsius * 9 / 5 + 32;
  }

  get kelvin() {
    return this.#celsius + 273.15;
  }

  // setter：设置时验证
  set celsius(value) {
    if (value < -273.15) {
      throw new Error('温度不能低于绝对零度！');
    }
    this.#celsius = value;
  }

  set fahrenheit(value) {
    this.#celsius = (value - 32) * 5 / 9;
  }
}

const temp = new Temperature(25);
console.log(temp.celsius);    // 25（不需要括号！）
console.log(temp.fahrenheit); // 77
console.log(temp.kelvin);     // 298.15

temp.fahrenheit = 32;          // 使用 setter，等价于 0°C
console.log(temp.celsius);    // 0

// temp.celsius = -300; ← Error: 温度不能低于绝对零度！

类的继承 extends

INHERITANCE

🌱 子类继承父类的属性和方法

用 extends 创建子类，用 super 调用父类的构造函数和方法。

// 父类：基础地图实体
class MapEntity {
  constructor(id, name) {
    this.id = id;
    this.name = name;
    this.visible = true;
  }

  show() { this.visible = true; }
  hide() { this.visible = false; }

  getInfo() {
    return `[${this.id}] ${this.name}`;
  }
}

// 子类1：标注点
class Marker extends MapEntity {
  constructor(id, name, lat, lon) {
    super(id, name);  // ← 必须先调用 super()
    this.lat = lat;
    this.lon = lon;
  }

  getInfo() {
    // 调用父类方法并扩展
    return super.getInfo() + ` 位置：(${this.lat}, ${this.lon})`;
  }
}

// 子类2：多边形区域
class Polygon extends MapEntity {
  constructor(id, name, coordinates) {
    super(id, name);
    this.coordinates = coordinates;
    this.area = null;
  }

  calculateArea() {
    // 简化面积计算（实际 CesiumJS 中有更精确的方法）
    this.area = this.coordinates.length * 10; // 示意
    return this.area;
  }
}

// 使用
const marker = new Marker('m001', '天安门', 39.9, 116.4);
console.log(marker.getInfo());
// "[m001] 天安门 位置：(39.9, 116.4)"

console.log(marker instanceof Marker);    // true
console.log(marker instanceof MapEntity); // true（子类实例也是父类实例）

const polygon = new Polygon('p001', '建筑区域', [[0,0],[1,0],[1,1]]);
polygon.hide();   // 继承自父类的方法
console.log(polygon.visible); // false
⚠️
子类构造函数必须先调用 super()！在使用 this 之前必须调用 super()，否则会抛出 ReferenceError。这是 JS 引擎的硬性要求。

ES6 模块：import / export

MODULES

📦 导出（export）

模块是独立的文件，通过 export 暴露接口，import 引入依赖。一个文件就是一个模块。

// ===== geometry.js（模块文件）=====

// 命名导出：可以导出多个
export const PI = Math.PI;

export function degreesToRadians(deg) {
  return deg * PI / 180;
}

export function radiansToDegrees(rad) {
  return rad * 180 / PI;
}

// 默认导出：一个模块只能有一个 default
export default class Vector3 {
  constructor(x, y, z) {
    this.x = x; this.y = y; this.z = z;
  }
}

// 也可以先定义再导出
const EARTH_RADIUS = 6371;
export { EARTH_RADIUS };

📥 导入（import）

// ===== main.js（入口文件）=====

// 导入默认导出（可以任意命名）
import Vector3 from './geometry.js';

// 导入命名导出（必须用原名，或用 as 重命名）
import { PI, degreesToRadians as toRad, EARTH_RADIUS } from './geometry.js';

// 同时导入默认和命名
// import Vector3, { PI, degreesToRadians } from './geometry.js';

// 导入所有命名导出为一个命名空间对象
// import * as Geo from './geometry.js';
// Geo.PI, Geo.degreesToRadians(90)...

console.log(PI);                    // 3.14159...
console.log(toRad(180));           // 3.14159...
console.log(EARTH_RADIUS);         // 6371

const v = new Vector3(1, 2, 3);
console.log(v.x, v.y, v.z);      // 1 2 3

📄 在 HTML 中使用模块

在 <script> 标签中添加 type="module"：

<!-- index.html -->
<script type="module">
  import { initMap } from './map.js';
  initMap();
</script>
特性普通 scriptES Module
作用域全局模块级（隔离）
加载方式同步（阻塞）默认 defer（异步）
严格模式需手动声明自动严格模式
this 指向windowundefined
import/export不支持支持
💡
模块化是大型项目的必备技能。CesiumJS 源码由成百上千个 ES Module 组成。理解模块化是理解任何现代框架的基础。

🌎 CesiumJS 关联场景

CESIUM

🌐 场景 1：用类封装 Cesium 实体管理

在实际 CesiumJS 项目中，你会用类来管理地图上的实体（标记、线条、多边形等），每个类负责一种实体类型：

import * as Cesium from 'cesium';

class FlightTracker {
  #viewer;
  #entity;

  constructor(viewer, flightId, lat, lon, altitude) {
    this.#viewer = viewer;
    this.#entity = viewer.entities.add({
      id: flightId,
      position: Cesium.Cartesian3.fromDegrees(lon, lat, altitude),
      point: { pixelSize: 10, color: Cesium.Color.RED },
      label: { text: flightId }
    });
  }

  updatePosition(lat, lon, altitude) {
    this.#entity.position =
      Cesium.Cartesian3.fromDegrees(lon, lat, altitude);
  }

  remove() {
    this.#viewer.entities.remove(this.#entity);
  }
}

// 使用：创建多个航班追踪器
const flight1 = new FlightTracker(viewer, 'CA1234', 39.9, 116.4, 10000);
const flight2 = new FlightTracker(viewer, 'MU5678', 31.2, 121.5, 8000);

🌐 场景 2：静态工具类 — 坐标转换

CesiumJS 大量使用笛卡尔坐标和地理坐标的转换，这正是静态方法的完美用武之地：

class CoordConverter {
  static toCartesian(lon, lat, alt = 0) {
    // 实际 CesiumJS 中使用 Cesium.Cartesian3.fromDegrees
    return Cesium.Cartesian3.fromDegrees(lon, lat, alt);
  }

  static toDegrees(cartesian) {
    const cartographic = Cesium.Cartographic.fromCartesian(cartesian);
    return {
      lat: Cesium.Math.toDegrees(cartographic.latitude),
      lon: Cesium.Math.toDegrees(cartographic.longitude),
      alt: cartographic.height
    };
  }
}

🌐 场景 3：继承 — 图层体系

CesiumJS 中的图层概念天然适合用继承来组织：

// 基础图层类
class BaseLayer {
  constructor(name, viewer) {
    this.name = name;
    this.viewer = viewer;
    this.visible = true;
  }
  toggle() { this.visible = !this.visible; }
  getInfo() { return `${this.name}（${this.visible ? '可见' : '隐藏'}）`; }
}

// 影像图层
class ImageryLayer extends BaseLayer {
  constructor(name, viewer, url) {
    super(name, viewer);
    this.url = url;
    this.type = '影像';
  }
  getInfo() { return super.getInfo() + ` [影像源：${this.url}]`; }
}

// 矢量图层
class VectorLayer extends BaseLayer {
  constructor(name, viewer, geoJsonUrl) {
    super(name, viewer);
    this.geoJsonUrl = geoJsonUrl;
    this.type = '矢量';
  }
}

// 统一管理所有图层
const layers = [
  new ImageryLayer('卫星图', viewer, 'https://...'),
  new VectorLayer('国界线', viewer, './borders.geojson'),
  new VectorLayer('行政区划', viewer, './admin.geojson')
];

📋 今日知识点总结

  • ✅ class 是 ES6 引入的面向对象语法糖，底层仍是原型继承
  • ✅ constructor 在 new 时自动调用，用于初始化实例属性
  • ✅ static 定义的方法/属性属于类本身，不需要实例即可调用
  • ✅ # 前缀定义真正私有的字段，外部完全无法访问
  • ✅ get/set 让你像访问属性一样调用方法，支持验证和计算
  • ✅ extends 实现继承，super() 必须在子类构造函数中先调用
  • ✅ export/import 实现模块化，一个文件一个模块，作用域隔离
  • ✅ HTML 中使用 <script type="module"> 启用 ES Module
  • ✅ 模块化 + 类 = 大型项目的代码组织基石，是理解 CesiumJS 源码的必备技能
