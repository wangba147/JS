相机系统概览：Viewer-Scene-Camera

基础

📷 为什么需要相机控制？

在 CesiumJS 中，三维地球场景本身只是一个数据容器；真正决定用户看到什么的，是相机（Camera）。它就像一部虚拟摄像机，决定了：

  • 摄像机在哪里（位置 position）
  • 朝哪个方向看（方向 direction）
  • 视野朝上/朝下/侧倾多少（heading/pitch/roll）
  • 观察目标是什么（target / trackedEntity）

掌握相机控制，才能把“三维地球”变成“可交互的 GIS 应用”。

🛠 相机层级结构

Cesium.Viewer
容器 + 默认 UI
↓
Viewer.scene
场景对象
↓
Scene.camera
相机对象
↓
Camera.position / heading / pitch / roll
实际可控制属性

🔍 核心访问路径

// 最常用：从 viewer 访问相机
const camera = viewer.camera;
const camera2 = viewer.scene.camera; // 等效
const camera3 = Cesium.Camera.DEFAULT_VIEW_RECTANGLE; // 默认视图矩形

Camera 核心属性与坐标系

坐标

🛠 核心属性一览

属性类型含义
positionCartesian3相机在 WGS84 坐标系中的位置（地球中心为原点，单位：米）
directionCartesian3相机朝向的 3D 方向向量（单位向量，读属性）
upCartesian3相机的"向上"方向（通常指向北）
rightCartesian3相机的"向右"方向（读属性）
frustumPerspectiveFrustum视锥体（fov、near、far）

🌍 坐标转换：世界坐标 ↔ 经纬度

// 从经纬度 → Cartesian3（世界坐标）
const beijing = Cesium.Cartesian3.fromDegrees(116.3972, 39.9075, 1000);
// 参数: 经度, 纬度, 高度(米)

// 从 Cartesian3 → 经纬度（将世界坐标转回经纬度+高度）
const cartographic = Cesium.Cartographic.fromCartesian(beijing);
const lon = Cesium.Math.toDegrees(cartographic.longitude);
const lat = Cesium.Math.toDegrees(cartographic.latitude);
const h   = cartographic.height;

console.log(`北京位置: 经度 ${lon.toFixed(4)}, 纬度 ${lat.toFixed(4)}, 高度 ${h}m`);
💡
理解坐标转换是相机控制的核心：CesiumJS 用 Cartesian3（世界坐标系，以地球中心为原点）存储位置，而人类习惯用经纬度。每个相机操作几乎都涉及 fromDegrees 转换。
3

setView：瞬间定位

核心

🎯 setView 把相机"瞬间"移动到指定位置

setView 是最直接的相机定位方式——没有任何动画过渡，立即跳转到目标视角。适合：初始化场景、重置视角、响应按钮点击。

// 方式一：CameraFlyTo 对象（最常用）
viewer.camera.setView({
  destination: Cesium.Cartesian3.fromDegrees(116.3972, 39.9075, 50000),
  orientation: {
    heading: Cesium.Math.toRadians(0),    // 0° = 朝北
    pitch:   Cesium.Math.toRadians(-45),  // -45° = 俯视
    roll:    Cesium.Math.toRadians(0)      // 0° = 不倾斜
  }
});

📝 setView 的多种参数形式

// 方式二：仅设目标 + 方位角（相机自动计算距离）
viewer.camera.setView({
  destination: Cesium.Cartesian3.fromDegrees(121.47, 31.23, 10000)
});

// 方式三：Rectangle 目标（自动 fit）
viewer.camera.setView({
  destination: Cesium.Rectangle.fromDegrees(115.0, 39.0, 118.0, 41.0)
});

// 方式四：直接给 position + direction + up（低级 API）
viewer.camera.setView({
  position: Cesium.Cartesian3.fromDegrees(110, 35, 200000),
  direction: Cesium.Cartesian3.negate(Cesium.Cartesian3.UNIT_Z, new Cesium.Cartesian3()),
  up: Cesium.Cartesian3.UNIT_Y
});
⚠
setView vs flyTo：setView 是瞬间跳转，flyTo 是动画飞行。初始化场景用 setView；用户点击按钮跳转用 flyTo（更友好）。
4

flyTo：平滑飞行

核心

✈ flyTo 的三种目标类型

flyTo 是 CesiumJS 最具标志性的交互——相机带着过渡动画飞到目标。参数与 setView 基本一致，但多出了飞行时长控制。

// 1. 飞到 Cartesian3 位置
viewer.camera.flyTo({
  destination: Cesium.Cartesian3.fromDegrees(116.40, 39.90, 5000),
  orientation: {
    heading: Cesium.Math.toRadians(0),
    pitch:   Cesium.Math.toRadians(-30),
    roll:    Cesium.Math.toRadians(0)
  },
  duration: 3  // 飞行耗时 3 秒
});

// 2. 飞到 Entity（自动计算最佳视角）
const entity = viewer.entities.add({
  position: Cesium.Cartesian3.fromDegrees(121.47, 31.23),
  point: { pixelSize: 10, color: Cesium.Color.RED }
});
viewer.flyTo(entity, { duration: 2 });

// 3. 飞到 Rectangle 范围（自动 fit）
viewer.camera.flyTo({
  destination: Cesium.Rectangle.fromDegrees(113, 22, 114, 23),
  duration: 2
});

🕐 flyTo 完整参数说明

参数类型默认值含义
destinationCartesian3 | Rectangle—目标位置或范围
orientationObject—heading / pitch / roll
durationNumber3.0飞行秒数
easingFunctionEasingFunction—缓动函数（如 Cesium.EasingFunction.QUADRATIC_IN_OUT）
completeCallback—飞行完成后的回调
cancelCallback—飞行被取消后的回调
maximumHeightNumber—飞行路径最高点（避免飞太低撞地）
offsetHeadingPitchRange—相对于目标的偏移量

🏃 飞行完成回调：链式操作

// 飞到北京 → 等待 1 秒 → 飞到上海
viewer.camera.flyTo({
  destination: Cesium.Cartesian3.fromDegrees(116.40, 39.90, 50000),
  duration: 2,
  complete: () => {
    console.log('已到达北京！准备飞往上海...');
    setTimeout(() => {
      viewer.camera.flyTo({
        destination: Cesium.Cartesian3.fromDegrees(121.47, 31.23, 50000),
        duration: 2
      });
    }, 1000);
  }
});
💡
flyTo 返回 Promise？flyTo 本身不返回 Promise，但 complete 回调能实现类似效果。如果你需要 await 风格的飞行，可以用 Cesium.viewer.flyTo 封装成一个返回 Promise 的工具函数。
5

zoomTo / lookAt / lookAtTransform

进阶

🔎 zoomTo — 自动计算最佳视角并飞行

zoomTo 是 flyTo 的一个快捷方式：CesiumJS 自动计算包含目标的最佳视角范围，你不需要手动指定 destination。

// 缩放到单个 Entity
viewer.zoomTo(entity);

// 缩放到 EntityCollection
viewer.zoomTo(viewer.entities);

// 缩放到 Cesium3DTileset（加载完成后）
tileset.readyPromise.then(() => {
  viewer.zoomTo(tileset);
});

// zoomTo 也支持 offset（偏置）
viewer.zoomTo(entity, new Cesium.HeadingPitchRange(
  Cesium.Math.toRadians(45),  // heading offset
  Cesium.Math.toRadians(-30), // pitch offset
  5000                          // range (米)
));

👀 lookAt — 盯着某个位置看

lookAt 不改变相机位置，只改变朝向。适合：让相机"转头"看向另一个目标。

// 让相机"转头"看向上海
const target = Cesium.Cartesian3.fromDegrees(121.47, 31.23);
viewer.camera.lookAt(target, new Cesium.HeadingPitchRange(
  Cesium.Math.toRadians(0),
  Cesium.Math.toRadians(-45),
  10000
));

🛠 lookAtTransform — 相对变换注视

lookAtTransform 是高级 API，让相机注视一个 4×4 变换矩阵的目标。常用于：跟随卫星/飞机的本地坐标系。

// 注视一个变换矩阵（例如跟随移动的飞机）
const transform = Cesium.Transforms.eastNorthUpToFixedFrame(
  Cesium.Cartesian3.fromDegrees(121.47, 31.23, 5000)
);
viewer.camera.lookAtTransform(transform, new Cesium.HeadingPitchRange(
  Cesium.Math.toRadians(0),
  Cesium.Math.toRadians(-30),
  5000
));
6

Heading / Pitch / Roll 视角参数

参数

📊 三个旋转自由度

参数旋转轴范围含义
heading绕 Z 轴（垂直）0° ~ 360°水平方位角：0° = 朝北, 90° = 朝东, 180° = 朝南
pitch绕 Y 轴（水平）-90° ~ 90°俯仰角：-90° = 正下, 0° = 水平, 90° = 正上
roll绕 X 轴（前后）0° ~ 360°侧倾角：0° = 水平, 非 0 = 倾斜

🌐 常见视角组合

// 俯视北京（标准俯视）
{ heading: toRadians(0),   pitch: toRadians(-90), roll: toRadians(0) }

// 从东南方向看北京（最常见）
{ heading: toRadians(225), pitch: toRadians(-45),  roll: toRadians(0) }

// 从正西方向斜着看（类似电影镜头）
{ heading: toRadians(270), pitch: toRadians(-20),  roll: toRadians(0) }

// "荷兰角"——侧倾效果（少见）
{ heading: toRadians(0),   pitch: toRadians(-45),  roll: toRadians(15) }

🎯 动态设置 heading/pitch/roll

// 直接设置相机朝向（不移动位置）
viewer.camera.setView({
  orientation: {
    heading: Cesium.Math.toRadians(180),
    pitch:   Cesium.Math.toRadians(-60),
    roll:    Cesium.Math.toRadians(0)
  }
});

// 绕 Z 轴旋转（改变 heading）
viewer.camera.lookAt(target, new Cesium.HeadingPitchRange(
  Cesium.Math.toRadians(90),  // heading = 朝东
  Cesium.Math.toRadians(-30),
  5000
));

屏幕交互拾取：pickEllipsoid 与 pickPosition

交互

👆 从屏幕像素反算世界坐标

CesiumJS 提供了两个关键函数，把用户点击的屏幕坐标 (x, y) 转换为三维世界坐标：

方法返回适用场景
camera.pickEllipsoidCartesian3拾取地球表面上的点
scene.pickPositionCartesian3拾取任意对象上的点（含地形、3D Tiles）
scene.pickObject拾取图元对象（Entity/Tileset）

👆 pickEllipsoid：点击地球获取经纬度

const handler = new Cesium.ScreenSpaceEventHandler(viewer.canvas);

handler.setInputAction((movement) => {
  const cartesian = viewer.camera.pickEllipsoid(
    movement.position,
    viewer.scene.globe.ellipsoid
  );

  if (cartesian) {
    const cartographic = Cesium.Cartographic.fromCartesian(cartesian);
    const lon = Cesium.Math.toDegrees(cartographic.longitude);
    const lat = Cesium.Math.toDegrees(cartographic.latitude);
    const h   = cartographic.height;

    console.log(`点击位置: 经度 ${lon.toFixed(4)}° 纬度 ${lat.toFixed(4)}° 高度 ${h.toFixed(1)}m`);

    // 在点击位置放一个标记
    viewer.entities.add({
      position: cartesian,
      point: { pixelSize: 8, color: Cesium.Color.YELLOW }
    });
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

🔍 pickPosition vs pickEllipsoid

// pickPosition 能拾取到地形和 3D Tiles 上的点
handler.setInputAction((movement) => {
  const pickedPosition = viewer.scene.pickPosition(movement.endPosition);
  if (Cesium.defined(pickedPosition)) {
    console.log('pickPosition 成功:', pickedPosition);
  } else {
    console.log('没有命中任何物体（可能点到了天空）');
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

// scene.pick 返回图元信息（知道用户点到了哪个 Entity）
handler.setInputAction((movement) => {
  const picked = viewer.scene.pick(movement.position);
  if (Cesium.defined(picked) && picked.id instanceof Cesium.Entity) {
    const entity = picked.id;
    console.log('点中了:', entity.name);  // Entity 对象
    console.log('图元类型:', picked.primitive);
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

ScreenSpaceCameraController：默认交互控制

交互

🛠 控制默认鼠标/触控行为

CesiumJS 自带完整的鼠标交互（拖拽旋转、滚轮缩放、右键平移）。通过 ScreenSpaceCameraController 可以精细控制这些行为。

const controller = viewer.scene.screenSpaceCameraController;

// 开启/关闭缩放
controller.enableZoom = true;

// 开启/关闭旋转
controller.enableRotate = true;

// 开启/关闭平移
controller.enableTranslate = true;

// 开启/关闭倾斜（鼠标右键上下拖拽）
controller.enableTilt = true;

// 开启/关闭 look（鼠标右键左右拖拽）
controller.enableLook = true;

// 碰撞检测：缩放时避免穿透地面
controller.enableCollisionDetection = true;

// 最小/最大缩放距离（米）
controller.minimumZoomDistance = 100;
controller.maximumZoomDistance = 20000000;

👀 自定义鼠标交互方式

// 修改默认的缩放事件（例如：改用右键缩放）
controller.zoomEventTypes = [
  Cesium.CameraEventType.RIGHT_DRAG,
  Cesium.CameraEventType.WHEEL,
  Cesium.CameraEventType.PINCH
];

// 修改旋转事件（改为中键拖拽）
controller.rotateEventTypes = [
  Cesium.CameraEventType.LEFT_DRAG,
  Cesium.CameraEventType.PINCH
];

// 限制相机倾斜角度
controller.minimumPitch = Cesium.Math.toRadians(-80); // 不让用户看到背面
controller.maximumPitch = Cesium.Math.toRadians(-10); // 不让用户太贴近地面
⚠
禁用所有默认交互：如果要做自定义交互（如点击测量、绘制多边形），可能需要临时禁用默认相机控制。controller.enableZoom = false; controller.enableRotate = false; controller.enableTranslate = false;

相机事件：moveStart / moveEnd / changed

事件

🕐 三种相机事件

事件触发时机典型用途
moveStart相机开始移动显示加载动画、暂停数据更新
moveEnd相机移动完成重新计算视野内数据、LOD 更新
changed相机属性每次变化实时更新 UI 面板（heading/pitch/高度）

🔧 moveStart / moveEnd 实战

// 相机开始移动时显示提示
viewer.camera.moveStart.addEventListener(() => {
  console.log('相机开始移动...');
  // 显示 loading 指示器
});

// 相机移动完成时执行操作
viewer.camera.moveEnd.addEventListener(() => {
  console.log('相机移动完成');
  const pos = viewer.camera.positionCartographic;
  console.log(`当前位置: 经度 ${Cesium.Math.toDegrees(pos.longitude).toFixed(2)}° 纬度 ${Cesium.Math.toDegrees(pos.latitude).toFixed(2)}°`);
});

🔬 changed 事件：实时跟踪相机状态

// 实时更新相机状态面板
viewer.camera.changed.addEventListener((percentage) => {
  // percentage: 0.0 ~ 1.0，表示当前移动进度（仅 flyTo 时有意义）
  const c = viewer.camera;
  const pos = c.positionCartographic;
  document.getElementById('cam-info').textContent =
    `Lon: ${Cesium.Math.toDegrees(pos.longitude).toFixed(2)}° | ` +
    `Lat: ${Cesium.Math.toDegrees(pos.latitude).toFixed(2)}° | ` +
    `H: ${pos.height.toFixed(0)}m | ` +
    `Pitch: ${Cesium.Math.toDegrees(c.pitch).toFixed(1)}°`;
});
💡
changed 事件性能提示：changed 在每帧渲染时都触发，频率很高（60fps）。在回调里避免 DOM 操作，尽量用 requestAnimationFrame 或防抖。

相机动画与巡航

高级

✈ 多段飞行巡航（城市巡礼）

结合 complete 回调和 clock.onTick，可以实现自动巡航效果。

// 定义巡礼路线
const cities = [
  { name: '北京', lon: 116.40, lat: 39.90, h: 50000 },
  { name: '上海', lon: 121.47, lat: 31.23, h: 50000 },
  { name: '广州', lon: 113.26, lat: 23.13, h: 50000 },
  { name: '成都', lon: 104.07, lat: 30.57, h: 50000 }
];

let currentCityIndex = 0;

function flyToNextCity() {
  if (currentCityIndex >= cities.length) {
    console.log('巡礼结束！');
    return;
  }
  const city = cities[currentCityIndex];
  console.log(`飞往 ${city.name}...`);

  viewer.camera.flyTo({
    destination: Cesium.Cartesian3.fromDegrees(city.lon, city.lat, city.h),
    orientation: {
      heading: Cesium.Math.toRadians(0),
      pitch:   Cesium.Math.toRadians(-60),
      roll:    Cesium.Math.toRadians(0)
    },
    duration: 3,
    complete: () => {
      currentCityIndex++;
      setTimeout(flyToNextCity, 2000); // 停留 2 秒
    }
  });
}

flyToNextCity(); // 开始巡礼

🔄 用 clock.onTick 实现平滑旋转

// 让相机绕目标点匀速旋转
const center = Cesium.Cartesian3.fromDegrees(116.40, 39.90, 0);
const rotationSpeed = 0.001; // 弧度/秒

viewer.clock.onTick.addEventListener((clock) => {
  const currentHeading = viewer.camera.heading;
  const newHeading = currentHeading + rotationSpeed;

  viewer.camera.lookAt(center, new Cesium.HeadingPitchRange(
    newHeading,
    Cesium.Math.toRadians(-45),
    50000
  ));
});

🏃 封装一个 FlyToPromise

// 将 flyTo 封装成返回 Promise 的函数（支持 await）
function flyToPromise(viewer, options) {
  return new Promise((resolve, reject) => {
    try {
      viewer.camera.flyTo({
        ...options,
        complete: resolve,
        cancel: reject
      });
    } catch (e) {
      reject(e);
    }
  });
}

// 使用 async/await 风格的连续飞行
async function tourChina(viewer) {
  await flyToPromise(viewer, {
    destination: Cesium.Cartesian3.fromDegrees(116.40, 39.90, 50000),
    duration: 2
  });
  console.log('已到达北京！');

  await flyToPromise(viewer, {
    destination: Cesium.Cartesian3.fromDegrees(121.47, 31.23, 50000),
    duration: 2
  });
  console.log('已到达上海！巡礼完成。');
}
💡
Promise 封装 = JS 基础知识直接回报：这正是 Day 11（异步编程）中学习的 Promise 和 async/await 的实战应用。把回调式的 flyTo 包装成 await 风格，代码可读性大幅提升。

🛠 综合示例：点击查看详情 + 自动飞到目标

// 1. 创建几个城市标注
const cities = [
  viewer.entities.add({ name: '北京', position: Cesium.Cartesian3.fromDegrees(116.40, 39.90), point: { pixelSize: 10, color: Cesium.Color.RED }, label: { text: '北京', font: '14px sans-serif' } }),
  viewer.entities.add({ name: '上海', position: Cesium.Cartesian3.fromDegrees(121.47, 31.23), point: { pixelSize: 10, color: Cesium.Color.BLUE }, label: { text: '上海', font: '14px sans-serif' } }),
];

// 2. 点击拾取 + 自动飞行
const handler = new Cesium.ScreenSpaceEventHandler(viewer.canvas);
handler.setInputAction((click) => {
  const picked = viewer.scene.pick(click.position);
  if (Cesium.defined(picked) && picked.id instanceof Cesium.Entity) {
    const entity = picked.id;
    alert(`点击了：${entity.name}`);
    viewer.flyTo(entity, { duration: 1.5 });

📋 今日知识点总结

  • 🎯 setView 瞬间定位，适合初始化场景和重置视角
  • ✈ flyTo 平滑飞行，支持 duration、complete 回调、easingFunction
  • 🔎 zoomTo 自动计算最佳视角并飞行，支持 Entity/Tileset/Rectangle
  • 👀 lookAt 不移动位置只改变朝向；lookAtTransform 注视变换矩阵
  • 📊 heading/pitch/roll 控制三维视角，heading=方位角、pitch=俯仰、roll=侧倾
  • 👆 pickEllipsoid 点击地球获取经纬度；pickPosition 支持地形和 3D Tiles；scene.pick 拾取 Entity
  • 🛠 ScreenSpaceCameraController 控制默认交互行为（缩放/旋转/平移/碰撞检测）
  • 🕐 moveStart/moveEnd/changed 相机事件，用于加载状态、UI 面板、实时跟踪
  • ✈ clock.onTick + lookAt 实现自动旋转；complete 回调链 实现多段巡航
  • 🚀 flyToPromise 封装 用 Promise 包装 flyTo，支持 async/await 风格
