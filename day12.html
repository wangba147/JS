Web API 全景图：浏览器即操作系统

核心概念

🌐 什么是 Web API？

Web API 是浏览器提供给 JavaScript 的接口集合。它们不是 JavaScript 语言本身的一部分（不属于 ECMAScript 规范），而是浏览器这个"宿主环境"额外提供的功能。Node.js 没有 document、没有 localStorage，因为它们属于浏览器运行时，不属于 JS 语言核心。

🏠 你的 Web 应用
↓
ECMAScript 核心
变量/函数/对象/类/Promise
DOM API
document / Element 事件
BOM API
navigator / location / history
↓
浏览器引擎
渲染引擎 + JS 引擎(V8/SpiderMonkey)
💡
核心区分：ECMAScript = 语言本身（if/for/function/class/Promise）；Web API = 浏览器提供的额外超能力（DOM、fetch、Canvas、Geolocation、WebSocket……）。日常开发中你两者都在用，但面试时常问"这是 JS 还是浏览器 API？"——比如 setTimeout 是浏览器 API，不是 ECMAScript！

Storage API：localStorage / sessionStorage

持久化

💾 浏览器存储三剑客

浏览器提供多种本地存储方式，各有适用场景。最常用的三种：

特性localStoragesessionStorageCookie
容量 ~5MB ~5MB ~4KB
生命周期 永久（手动清除） 标签页关闭即销毁 可设过期时间
作用域 同源所有标签页共享 仅当前标签页 同源，可设 path
随请求发送 不会（省带宽） 不会（省带宽） 会（影响性能）
API 复杂度 极简 极简 需要手动解析

🔧 localStorage 核心 API

// ===== 写入 =====
localStorage.setItem('userName', 'Alice');
localStorage.theme = 'dark';           // 也可以用属性写法

// ===== 读取 =====
const name = localStorage.getItem('userName'); // "Alice"
const missing = localStorage.getItem('xxx');   // null（不存在）

// ===== 删除 =====
localStorage.removeItem('userName'); // 删除单个
localStorage.clear();                  // 清空全部

// ===== 遍历 =====
for (let i = 0; i < localStorage.length; i++) {
  const key   = localStorage.key(i);
  const value = localStorage.getItem(key);
  console.log(key, value);
}

🔑 对象存取技巧（JSON 序列化）

localStorage 只能存储字符串。存储对象/数组必须用 JSON.stringify，取出时用 JSON.parse：

const user = { name: 'Alice', age: 28, roles: ['admin'] };

// 存储
localStorage.setItem('user', JSON.stringify(user));

// 读取
const saved = JSON.parse(localStorage.getItem('user'));
console.log(saved.name); // "Alice"

// 封装成工具函数
function setJSON(key, value) {
  localStorage.setItem(key, JSON.stringify(value));
}
function getJSON(key, defaultVal = null) {
  const raw = localStorage.getItem(key);
  try { return raw ? JSON.parse(raw) : defaultVal; }
  catch { return defaultVal; }
}
💡
localStorage 的限制：①仅 5MB 左右；②同步 API（会阻塞主线程，不宜存大量数据）；③只能存字符串。如需大容量、结构化、异步存储，用 IndexedDB（见下文 CesiumJS 关联场景）。
⚠️
sessionStorage 的坑：在新标签页中 window.open(url) 会复制当前 sessionStorage，但用户手动 Ctrl+T 打开的新标签页不会共享 sessionStorage。如果发现数据"莫名其妙丢了"，先检查是不是这个原因。

Fetch API：现代网络请求

网络

🌐 告别 XMLHttpRequest

fetch() 是现代浏览器内置的网络请求 API，基于 Promise，语法简洁，是 XMLHttpRequest 的现代替代品。

// 基本 GET 请求
const response = await fetch('https://api.example.com/data');
const data    = await response.json();  // 解析 JSON

// POST 请求（发送 JSON）
const res = await fetch('https://api.example.com/submit', {
  method:  'POST',
  headers: { 'Content-Type': 'application/json' },
  body:    JSON.stringify({ name: 'Alice', age: 28 })
});

🔧 Response 对象：别被状态码骗了

重要：fetch() 只在网络故障时 reject（如断网、DNS 解析失败）。HTTP 404/500 等错误状态码不会导致 reject——你需要手动检查 response.ok。

async function safeFetch(url, options = {}) {
  const res = await fetch(url, options);

  if (!res.ok) {  // 等价于 status 不在 200-299
    throw new Error(`HTTP ${res.status}: ${res.statusText}`);
  }
  return await res.json();
}

// 常用 Response 方法
const json  = await res.json();   // JSON
const text  = await res.text();   // 纯文本
const blob  = await res.blob();   // 二进制（图片/文件）
const buf   = await res.arrayBuffer();  // 原始字节 → WebGL/Web Worker

📦 上传文件与 FormData

const formData = new FormData();
formData.append('file', fileInput.files[0]);
formData.append('description', '地形高度图');

// fetch 会自动设置 Content-Type 为 multipart/form-data
const res = await fetch('https://api.example.com/upload', {
  method: 'POST',
  body:   formData
});
💡
fetch vs XHR：fetch 基于 Promise，语法更清晰；但 fetch 默认不携带 cookie（需要设 credentials: 'include'），也没有原生超时（需配合 AbortController 实现）。

Geolocation API：获取用户位置

定位

🌎 navigator.geolocation

浏览器可以获取用户的地理位置（需用户授权），返回经度、纬度、海拔、精度等信息。这是所有地图应用的基础——也是 CesiumJS 定位功能的底层依赖。

if ('geolocation' in navigator) {
  navigator.geolocation.getCurrentPosition(
    (position) => {
      const { latitude, longitude } = position.coords;
      console.log(`位置: ${latitude}, ${longitude}`);
      console.log(`精度: ${position.coords.accuracy} 米`);
    },
    (err) => console.error('定位失败:', err.message),
    {
      enableHighAccuracy: true,  // 请求高精度（会耗电）
      timeout:           10000,  // 10秒超时
      maximumAge:        0       // 不使用缓存位置
    }
  );
}

🔄 持续追踪位置：watchPosition

getCurrentPosition 只获取一次。watchPosition 会持续追踪位置变化（就像手机导航），返回一个 watchId 用于取消。

const watchId = navigator.geolocation.watchPosition(
  (pos) => {
    console.log(`移动到了: ${pos.coords.latitude}, ${pos.coords.longitude}`);
    // 实时更新地图上的位置标记
  },
  (err) => console.error(err),
  { enableHighAccuracy: true, maximumAge: 1000 }
);

// 停止追踪
navigator.geolocation.clearWatch(watchId);

Canvas API：2D 绘图入门

图形

🎨 Canvas 是什么？

<canvas> 是一个像素画布——你可以用 JavaScript 在上面绘制任意图形。它有两个上下文：2d（二维绘图）和 webgl（三维渲染，CesiumJS 的核心）。理解 Canvas 2D 是理解 WebGL/CesiumJS 渲染管线的第一步。

🎨 基本绘图流程

// 1. 获取 canvas 和 2D 上下文
const canvas  = document.querySelector('#myCanvas');
const ctx     = canvas.getContext('2d');

// 2. 设置样式
ctx.fillStyle   = '#818cf8';   // 填充颜色
ctx.strokeStyle = '#ffffff';   // 描边颜色
ctx.lineWidth   = 2;            // 线宽
ctx.font        = '18px sans-serif';

// 3. 绘制形状
ctx.fillRect(10, 10, 100, 60);   // 填充矩形
ctx.strokeRect(130, 10, 100, 60); // 描边矩形

// 4. 路径绘图
ctx.beginPath();
ctx.arc(300, 40, 30, 0, Math.PI * 2);  // 圆形
ctx.fill();

// 5. 文字
ctx.fillText('Hello Canvas!', 350, 45);

🎨 实时 Canvas 演示

💡
Canvas 2D vs WebGL：Canvas 2D 是 CPU 渲染的 2D 绘图 API；WebGL 是 GPU 加速的 3D 渲染，通过 canvas.getContext('webgl') 获取。CesiumJS 底层用的就是 WebGL——它比 Canvas 2D 复杂得多，但基本概念（坐标系、变换、着色器）都是从这里延伸的。

Web Workers：多线程计算

性能

⚙️ 为什么需要 Worker？

JavaScript 默认是单线程的——复杂计算会冻结整个页面（回想 Day 11 的同步阻塞例子）。Web Worker 让你在独立的后台线程中运行 JS 代码，不阻塞 UI。

🔧 主线程代码

// 创建 Worker 实例
const worker = new Worker('worker.js');

// 发送数据给 Worker
worker.postMessage({
  data: hugeArray,
  type: 'processTerrain'
});

// 接收 Worker 返回的结果
worker.onmessage = (e) => {
  console.log('Worker 计算结果:', e.data);
};

// 错误处理
worker.onerror = (err) => console.error(err);

// 终止 Worker
worker.terminate();

🔧 worker.js 文件

// Worker 线程代码
self.onmessage = (e) => {
  const { data, type } = e.data;

  if (type === 'processTerrain') {
    // 处理地形数据（不阻塞 UI）
    const result = data.map(h => h * 1.2);
    self.postMessage(result);
  }
};

// Worker 也可以 importScripts
self.importScripts('/lib/geometry.js');
⚠️
Worker 的三大限制：①不能访问 DOM（没有 document、window）；②通过 postMessage 传递数据是值拷贝（大数组会消耗内存，推荐用 Transferable Objects）；③不能使用 localStorage，但可以用 IndexedDB。

🚀 内联 Worker（不用单独文件）

// 把 Worker 代码写成字符串，转成 Blob URL
const code = `
  self.onmessage = (e) => {
    const fib = (n) => n <= 1 ? n : fib(n-1) + fib(n-2);
    self.postMessage(fib(e.data));
  };
`;
const blob = new Blob([code], { type: 'application/javascript' });
const worker = new Worker(URL.createObjectURL(blob));

worker.postMessage(40);
worker.onmessage = (e) => console.log('Fibonacci(40):', e.data);

History API & URL API

路由

🔗 History API：无刷新改变 URL

SPA（单页应用）的核心技术。通过 history.pushState 改变浏览器地址栏但不刷新页面——这是 React Router、Vue Router 等所有前端路由库的底层基础。

// pushState：新增一条历史记录
history.pushState({ page: 1 }, '', '/page/1');
//           ↑ state 对象      ↑ title(废弃)  ↑ 新 URL

// replaceState：替换当前历史记录（不新增）
history.replaceState({ page: 2 }, '', '/page/2');

// 监听浏览器前进/后退（用户点按钮）
window.addEventListener('popstate', (e) => {
  console.log('用户导航到了:', location.pathname);
  console.log('State:', e.state);  // pushState 传的 state 对象
});

🔍 URL API：不用正则解析 URL

原生 URL 构造函数可以优雅地解析和构建 URL——告别手写正则的痛苦：

const url = new URL('https://cesium.com/ion/assets?type=3DTiles&id=12345');

console.log(url.protocol); // "https:"
console.log(url.hostname); // "cesium.com"
console.log(url.pathname); // "/ion/assets"
console.log(url.searchParams.get('type')); // "3DTiles"
console.log(url.searchParams.get('id'));   // "12345"

// 动态构建 URL
url.searchParams.set('version', '2.0');
url.searchParams.append('format', 'json');
console.log(url.href);
// "https://cesium.com/ion/assets?type=3DTiles&id=12345&version=2.0&format=json"

// URLSearchParams 也可以单独使用
const params = new URLSearchParams(location.search);
console.log([...params]); // [["type","3DTiles"],["id","12345"]]

WebSocket：实时双向通信

实时

📡 HTTP vs WebSocket

HTTP 是"请求-响应"模式——客户端问一句，服务器答一句。WebSocket 是全双工通道——建立连接后，双方可以随时互发消息，无需轮询。这是 CesiumJS 实时数据推送（飞机轨迹、船舶 AIS、IoT 传感器）的传输层。

特性HTTPWebSocket
通信模式请求→响应（单向）双向任意发
连接每请求新建一次连接持久
延迟高（握手开销）极低
协议http://https://ws://wss://

🔧 WebSocket 客户端使用

// 1. 建立连接
const ws = new WebSocket('wss://realtime.example.com/tracks');

// 2. 连接成功
ws.onopen = () => {
  console.log('WebSocket 已连接');
  // 发送订阅消息
  ws.send(JSON.stringify({
    type: 'subscribe',
    channel: 'aircraft_SHA'
  }));
};

// 3. 接收消息
ws.onmessage = (event) => {
  const pos = JSON.parse(event.data);
  console.log(`飞机 ${pos.flightId}: 经度=${pos.lng} 纬度=${pos.lat} 高度=${pos.alt}`);
  // 更新 CesiumJS 实体位置
};

// 4. 错误与关闭
ws.onerror = (err) => console.error('WebSocket 错误', err);
ws.onclose = () => console.log('WebSocket 已断开');

// 5. 主动关闭
ws.close(1000, '正常关闭');

🔃 自动重连机制

WebSocket 连接可能断掉（网络波动、服务器重启），生产环境必须有重连逻辑：

class ReconnectingWS {
  constructor(url, maxRetries = 5) {
    this.url = url;
    this.maxRetries = maxRetries;
    this.retries = 0;
    this.connect();
  }
  connect() {
    this.ws = new WebSocket(this.url);
    this.ws.onopen  = () => { this.retries = 0; console.log('已连接'); };
    this.ws.onclose = () => {
      if (this.retries < this.maxRetries) {
        const delay = Math.min(1000 * 2 ** this.retries, 30000);
        console.log(`将在 ${delay/1000} 秒后重连...`);
        setTimeout(() => { this.retries++; this.connect(); }, delay);
      } else {
        console.error('重连次数已达上限');
      }
    };
  }
}

CesiumJS 关联场景

目标

🌐 场景 1：Fetch API 加载 GeoJSON / CZML / 3D Tiles

CesiumJS 中所有数据源——GeoJSON（矢量）、CZML（动态时序）、3D Tiles（三维模型）——本质上都是通过网络请求加载的。底层用的就是 fetch 或 XMLHttpRequest。

// CesiumJS 核心数据加载流程
async function loadGeoJSONOverlay(viewer) {
  // 1. 用 fetch 从服务器拉取 GeoJSON
  const response = await fetch('/data/earthquakes.geojson');
  const geojson  = await response.json();

  // 2. 加载到 Cesium 场景
  const dataSource = await Cesium.GeoJsonDataSource.load(geojson, {
    stroke: Cesium.Color.RED,
    fill:   Cesium.Color.RED.withAlpha(0.3)
  });
  viewer.dataSources.add(dataSource);
}

🗺️ 场景 2：Web Workers 处理大规模地形数据

CesiumJS 处理全球地形时，需要在后台进行大量的高度采样和几何计算。这些工作都在 Web Workers 中完成，保证主线程不卡顿。

// CesiumJS 内部使用 Worker 处理地形瓦片
// 伪代码——实际 Cesium 源码类似这样：

// 主线程：创建地形 Worker
const terrainWorker = new Worker('/Cesium/Workers/createVerticesFromHeightmap.js');

terrainWorker.postMessage({
  heights:    heightmapData,  // 高度图像素数据
  tileRect:   tileRectangle,  // 瓦片范围
  resolution: 65               // 网格分辨率
}, [heightmapData.buffer]);  // Transferable 零拷贝传输

terrainWorker.onmessage = (e) => {
  const { vertices, indices, boundingSphere } = e.data;
  // 主线程拿到顶点数据后，交给 WebGL 渲染
  glBuffer = context.createVertexBuffer(vertices);
};

🌎 场景 3：Geolocation + CesiumJS 定位

获取用户当前位置，让 CesiumJS 飞到那里——这是很多 GIS 应用的入口功能。

function flyToMyLocation(viewer) {
  if (!('geolocation' in navigator)) {
    return alert('您的浏览器不支持定位');
  }

  navigator.geolocation.getCurrentPosition(
    (pos) => {
      const { latitude, longitude } = pos.coords;

      // 让 CesiumJS 飞过去
      viewer.camera.flyTo({
        destination: Cesium.Cartesian3.fromDegrees(
          longitude, latitude, 5000   // 高度 5000 米
        ),
        orientation: {
          heading: Cesium.Math.toRadians(0),
          pitch:   Cesium.Math.toRadians(-45),
          roll:    0
        },
        duration: 3                  // 3 秒飞行动画
      });

      // 添加一个位置标记
      viewer.entities.add({
        position: Cesium.Cartesian3.fromDegrees(longitude, latitude),
        point: {
          pixelSize: 12,
          color: Cesium.Color.DODGERBLUE
        },
        label: {
          text: '我的位置',
          font: '14px sans-serif'
        }
      });
    },
    (err) => console.warn('定位失败:', err.message)
  );
}

📡 场景 4：WebSocket 实时追踪飞机/船舶

这是 CesiumJS 最典型的实时应用场景——用 WebSocket 接收飞机/船舶/车辆的位置更新，然后实时更新地球上的实体：

async function startLiveTracking(viewer) {
  const ws = new WebSocket('wss://adsb.example.com/live');

  // 维护飞机 ID → Cesium Entity 的映射
  const aircraftMap = new Map();

  ws.onmessage = (event) => {
    const { flightId, lng, lat, alt } = JSON.parse(event.data);
    const pos = Cesium.Cartesian3.fromDegrees(lng, lat, alt);

    if (aircraftMap.has(flightId)) {
      // 更新已有飞机的位姿
      const entity = aircraftMap.get(flightId);
      entity.position = pos;
    } else {
      // 新飞机：创建 Cesium 实体
      const entity = viewer.entities.add({
        id:      flightId,
        name:    flightId,
        position: pos,
        model: {
          uri: '/models/airplane.glb',
          minimumPixelSize: 32
        }
      });
      aircraftMap.set(flightId, entity);
    }
  };

  ws.onclose = () => {
    console.warn('实时追踪已断开，3秒后重连...');
    setTimeout(() => startLiveTracking(viewer), 3000);
  };
}

💾 场景 5：IndexedDB 缓存 CesiumJS 离线数据

CesiumJS 的地形和影像瓦片数量巨大，每次都从服务器请求太慢。IndexedDB 可以缓存瓦片到本地，实现离线使用：

// CesiumJS 的离线缓存思路
async function cacheTileOffline(tileUrl) {
  const db    = await openDB('CesiumCache', 1);
  const tx    = db.transaction('tiles', 'readwrite');
  const store = tx.objectStore('tiles');

  // 先查缓存
  const cached = await store.get(tileUrl);
  if (cached) return cached.data;

  // 缓存未命中：fetch + 存入 IndexedDB
  const res  = await fetch(tileUrl);
  const data = await res.arrayBuffer();
  await store.put({ url: tileUrl, data: data, timestamp: Date.now() });
  return data;
}

📋 今日知识点总结

  • ✅ Web API 是浏览器提供的接口集合，不属于 ECMAScript 核心——包括 DOM、BOM、fetch、Canvas、Worker 等
  • ✅ Storage API：localStorage（永久/5MB/同源共享）；sessionStorage（标签页关闭即销毁）；Cookie（小/4KB/随请求发送）
  • ✅ localStorage 只能存字符串，存取对象须用 JSON.stringify / JSON.parse
  • ✅ Fetch API：基于 Promise 的现代网络请求 API；HTTP 404/500 不会导致 reject，需手动检查 response.ok
  • ✅ 用 AbortController 给 fetch 加超时；用 FormData 上传文件
  • ✅ Geolocation API：getCurrentPosition（一次定位）；watchPosition（持续追踪）
  • ✅ Canvas API：2d 上下文用于像素绘图；webgl 上下文用于 3D 渲染（CesiumJS 的底层）
  • ✅ Web Workers：独立后台线程，不阻塞 UI；不能访问 DOM；通过 postMessage/onmessage 通信
  • ✅ History API：pushState/replaceState 无刷新改变 URL；popstate 监听导航
  • ✅ URL API：new URL() 解析；searchParams 读写查询参数——告别正则
  • ✅ WebSocket：全双工实时通信；onmessage 接收；send 发送；需实现重连机制
  • ✅ CesiumJS 中：fetch 加载数据源、Worker 处理地形、Geolocation 定位、WebSocket 推实时轨迹、IndexedDB 缓存离线瓦片
