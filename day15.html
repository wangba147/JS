材质系统回顾：从纯色到高级效果

基础

🎨 什么是材质（Material）？

在 CesiumJS 中，Material 决定了 Entity 图形看起来是什么样子——颜色、纹理、光效、透明度、动画。你可以理解为：Entity 的形状 = Graphics，Entity 的外观 = Material。

昨天我们接触了 7 种材质，今天深入每一种的细节，并加入新的高级材质和 3D Tiles。

🛠️ 材质系统架构

你的代码
Entity.material = ...
↓
MaterialProperty
Color / Image / PolylineGlow / Stripe / ...
↓
Cesium.Material
WebGL 着色器 + 纹理
↓
GPU 渲染
Fragment Shader 执行

📊 材质全景分类

类别材质名适用图形效果描述
基础Color所有纯色填充，最常用
纹理Imagepolygon/ellipse/rectangle/corridor/wall/box贴图片纹理
折线PolylineGlowpolyline光晕发光效果

ColorMaterialProperty 与颜色进阶

颜色

🔴 基用法

Color 材质是最简单的——直接传一个 Cesium.Color 对象即可。但"简单"不代表"弱"，透明度组合是 2D 覆盖区域的杀手锏。

// 最简形式
material: Cesium.Color.RED

// 带透明度（最常用！）
material: Cesium.Color.RED.withAlpha(0.3)

// 从 CSS 颜色字符串创建
material: Cesium.Color.fromCssColorString('#61dafb')
material: Cesium.Color.fromCssColorString('rgba(255, 0, 0, 0.5)')

// 从 HSL 创建（色相、饱和度、亮度）
material: Cesium.Color.fromHsl(0.0, 1.0, 0.5, 0.3)  // 红色半透明

// 从 Bytes 创建（0-255 范围）
material: new Cesium.Color(255/255, 0/255, 0/255, 0.3)

🎨 颜色进阶技巧

// 颜色混合（两个颜色按比例混合）
const mixed = Cesium.Color.RED.blend(Cesium.Color.BLUE, 0.5); // 50% 红 + 50% 蓝

// 亮度调整
const brighter = Cesium.Color.RED.brighten(0.5, new Cesium.Color());
const darker = Cesium.Color.RED.darken(0.5, new Cesium.Color());

// 预设颜色速查
console.log(Cesium.Color.RED);         // { red: 1, green: 0, blue: 0, alpha: 1 }
console.log(Cesium.Color.CYAN);       // { red: 0, green: 1, blue: 1, alpha: 1 }
console.log(Cesium.Color.TRANSPARENT); // alpha = 0
💡
withAlpha 是 2D 覆盖的灵魂：行政区划、雷达范围、灾害区域等需要半透明叠加在底图之上。Cesium.Color.RED.withAlpha(0.3) 是你每天都会写的代码。

ImageMaterialProperty：贴图的艺术

纹理

🖼 基础用法

Image 材质把一张图片贴到图形表面，就像给一面墙贴壁纸。适用于：雷达覆盖图、热力图、卫星云图、道路纹理、地块标记。

viewer.entities.add({
  rectangle: {
    coordinates: Cesium.Rectangle.fromDegrees(115.0, 39.0, 118.0, 41.0),
    material: new Cesium.ImageMaterialProperty({
      image: 'radar-coverage.png',   // 图片 URL
      transparent: true,          // 支持透明像素
      repeat: new Cesium.Cartesian2(1, 1) // 重复次数
    })
  }
});

🎨 图片贴图进阶

// 重复平铺（4×4 网格）
material: new Cesium.ImageMaterialProperty({
  image: 'tile-texture.jpg',
  repeat: new Cesium.Cartesian2(4, 4), // 4 列 × 4 行
  color: Cesium.Color.WHITE,    // 白色 = 不染色（原色显示）
  transparent: false
})

// 带颜色叠加（图片 × 颜色 = 最终色）
material: new Cesium.ImageMaterialProperty({
  image: 'cloud.png',
  color: Cesium.Color.BLUE.withAlpha(0.6), // 蓝色调云图
  transparent: true
})

// 用 Canvas 作为纹理源（动态生成！）
const canvas = document.createElement('canvas');
canvas.width = 256;
canvas.height = 256;
const ctx = canvas.getContext('2d');
// ... 在 canvas 上画热力图
ctx.fillRect(0, 0, 256, 256);

material: new Cesium.ImageMaterialProperty({
  image: canvas // 直接传 Canvas 对象！
})
💡
Canvas 纹理 = 无限可能：你可以用 Canvas API 实时绘制热力图、渐变、动画，然后把 Canvas 直接传给 ImageMaterialProperty。这是"动态材质"最简单也最强大的方式，不需要 WebGL 着色器知识。

Polyline 材质全家桶

折线

🌏 三种专属折线材质

材质视觉效果典型应用
PolylineGlow光晕 + 中心亮线航线、轨迹、能量流
PolylineArrow带方向箭头物流路线、风向
PolylineDash虚线（可自定义间隔）规划路线、临时路径

➔ PolylineGlow — 发光折线

viewer.entities.add({
  polyline: {
    positions: Cesium.Cartesian3.fromDegreesArray([
      116.39, 39.91,
      121.47, 31.23
    ]),
    width: 10,
    material: new Cesium.PolylineGlowMaterialProperty({
      glowPower: 0.25,     // 光晕范围 0-1，越大越弥散
      taperPower: 1.0,    // 1.0=不缩窄；0=细到消失
      color: Cesium.Color.CYAN
    })
  }
});

➔ PolylineArrow — 箭头折线

viewer.entities.add({
  polyline: {
    positions: Cesium.fromDegreesArrayHeights([
      116.39, 39.91, 0,
      121.47, 31.23, 0
    ]),
    width: 8,
    material: new Cesium.PolylineArrowMaterialProperty(
      Cesium.Color.ORANGE
    )
  }
});

➔ PolylineDash — 虚线折线

viewer.entities.add({
  polyline: {
    positions: Cesium.Cartesian3.fromDegreesArray([
      116.39, 39.91,
      113.26, 23.13
    ]),
    width: 4,
    material: new Cesium.PolylineDashMaterialProperty({
      color: Cesium.Color.YELLOW,
      gapColor: Cesium.Color.TRANSPARENT,  // 间隔颜色
      dashLength: 16.0,              // 实线长度
      dashPattern: 255.0             // 255=全实；可调间隔模式
    })
  }
});
💡
折线材质的 JS 基础：这些材质本质上是在 WebGL Fragment Shader 中用数学公式计算每像素的颜色。你不需要写 Shader，CesiumJS 帮你封装好了。但理解"材质 = 函数(位置, 时间) → 颜色"这个概念很重要。

Checkerboard / Stripe / Grid：特殊材质

图案

🎨 Checkerboard — 棋盘格

material: new Cesium.CheckerboardMaterialProperty({
  evenColor: Cesium.Color.WHITE,
  oddColor:  Cesium.Color.BLUE,
  repeat: new Cesium.Cartesian2(8, 8)
})

测试、调试用，也适合做"分区标记"。

🔧 Stripe — 条纹

material: new Cesium.StripeMaterialProperty({
  evenColor: Cesium.Color.WHITE,
  oddColor:  Cesium.Color.BLACK,
  repeat: 10,
  offset: 0,
  orientation: Cesium.StripeOrientation.HORIZONTAL
})

斑马纹，方向可设水平/垂直。

🔬 Grid — 网格

material: new Cesium.GridMaterialProperty({
  color: Cesium.Color.YELLOW,
  cellAlpha: 0.2,    // 单元格透明度
  lineCount: new Cesium.Cartesian2(8, 8),
  lineThickness: new Cesium.Cartesian2(2.0, 2.0),
  lineOffset: new Cesium.Cartesian2(0.0, 0.0)
})

网格线+半透明单元格，适合做分区/经纬网。

🖼 所有图案材质共同参数

参数含义
evenColor主色（棋盘白格/条纹主条）
oddColor交替色
repeat重复次数（Cartesian2 或 Number）
offset偏移量（Stripe 专属）
cellAlpha单元格透明度（Grid 专属）

CompositeMaterialProperty：材质组合

高级

🎨 组合材质 = 多层叠加

Composite 材质允许你把多个材质叠在一起，像 Photoshop 的图层混合。最常见的是：图片 + 颜色叠加。

// 方式一：ImageMaterialProperty 自带的 color 参数
material: new Cesium.ImageMaterialProperty({
  image: 'heatmap.png',
  color: Cesium.Color.RED.withAlpha(0.7), // 图片 × 红色 70%
  transparent: true
})

// 方式二：Fabric JSON 自定义材质（更灵活）
material: new Cesium.Material({
  fabric: {
    type: 'Image',
    uniforms: {
      image: 'radar.png',
      color: new Cesium.Color(1, 0, 0, 0.5)
    }
  }
})
⚠️
Fabric JSON 是底层 API：CesiumJS 内部用 Fabric 描述材质（类似 WebGL Shader 的配置化版本）。日常开发用 MaterialProperty 就够了，只有需要自定义着色器时才用 Fabric。初学者不建议直接用 Fabric。

MaterialProperty 与 CallbackProperty：动态材质

动态

⏰ 用 CallbackProperty 让材质每帧变化

就像 Day 14 中让 Entity position 每帧变化一样，material 也可以用 CallbackProperty 动态更新。

// 动态颜色：红 → 绿交替闪烁
let flashColor = false;

viewer.entities.add({
  position: Cesium.Cartesian3.fromDegrees(116.40, 39.90),
  ellipse: {
    semiMajorAxis: 30000,
    semiMinorAxis: 30000,
    material: new Cesium.ColorMaterialProperty(
      new Cesium.CallbackProperty(() => {
        flashColor = !flashColor;
        return flashColor
          ? Cesium.Color.RED.withAlpha(0.8)
          : Cesium.Color.GREEN.withAlpha(0.4);
      }, false)
    )
  }
});

🕐 用 Canvas 实现动态热力图材质

// 创建动态 Canvas 纹理
const heatCanvas = document.createElement('canvas');
heatCanvas.width = 512;
heatCanvas.height = 512;
const heatCtx = heatCanvas.getContext('2d');

function drawHeatFrame(time) {
  // 清空画布
  heatCtx.clearRect(0, 0, 512, 512);
  // 根据时间绘制不同热力分布
  const radius = 50 + time * 20;
  const gradient = heatCtx.createRadialGradient(256, 256, 0, 256, 256, radius);
  gradient.addColorStop(0, 'rgba(255,0,0,0.8)');
  gradient.addColorStop(0.5, 'rgba(255,165,0,0.4)');
  gradient.addColorStop(1, 'rgba(0,0,255,0.1)');
  heatCtx.fillStyle = gradient;
  heatCtx.fillRect(0, 0, 512, 512);
}

// 每秒更新一次 Canvas 内容
viewer.clock.onTick.addEventListener((clock) => {
  const t = Cesium.JulianDate.secondsDifference(clock.currentTime, startTime);
  drawHeatFrame(t);
});

viewer.entities.add({
  rectangle: {
    coordinates: Cesium.Rectangle.fromDegrees(115.0, 39.0, 118.0, 41.0),
    material: new Cesium.ImageMaterialProperty({
      image: heatCanvas,   // 传 Canvas 对象，自动同步更新
      transparent: true
    })
  }
});
💡
Canvas 动态材质 = 低门槛高收益：不需要学习 GLSL 着色器语言，用熟悉的 Canvas 2D API 就能做出"动态热力图"、"渐变扩散"等高级效果。这是 JS 基础学习中 Canvas API（Day 12）的直接回报。

3D Tiles：Cesium 的王牌功能

核心

🌏 什么是 3D Tiles？

3D Tiles 是 OGC（开放地理空间联盟）制定的国际标准，用于在 Web 上流式传输和渲染海量三维地理数据。它是 CesiumJS 最强大的功能——让你加载包含数百万个建筑、数百 GB 的倾斜摄影、几十亿个点云点的数据集，而浏览器还能流畅运行。

一句话：3D Tiles = 海量 3D 地理数据的"视频流"——先看粗略版，再逐步加载精细版。

🛠️ 3D Tiles 数据格式

🏠Batched 3D Model
(b3dm)建筑/设施模型
📷Instanced 3D Model
(i3dm)树木/路灯等重复模型
💨Point Cloud
(pnts)激光雷达点云
🖼Composite
(cmpt)多种格式组合
📊Vector Data
(vctr)矢量数据
🔥glTF/glb现代 3D 模型标准

📊 3D Tiles LOD（层次细节）机制

远处 → 低精度 Tile
几百个三角形
↓ 镜头靠近
中等 → 中精度 Tile
几千个三角形
↓ 继续靠近
近处 → 高精度 Tile
数万个三角形，纹理完整

📦 tileset.json — 3D Tiles 的入口

每个 3D Tiles 数据集都有一个 tileset.json 文件作为入口，描述了 Tile 树结构、边界范围、LOD 策略。

// tileset.json 结构示意
{
  "asset": { "version": "1.0" },
  "geometricError": 500,       // 根节点几何误差（LOD 阈值）
  "root": {
    "boundingVolume": {           // 范围（球/盒子/区域）
      "region": [2.0, 0.69, 2.01, 0.70, 0, 100]
    },
    "content": { "uri": "0.b3dm" },
    "children": [                  // 子 Tile
      { "geometricError": 100, "content": { "uri": "1.b3dm" } },
      { "geometricError": 50, "content": { "uri": "2.b3dm" } }
    ]
  }
}
💡
3D Tiles 与 Entity 的关系：Entity 是"手动创建单个对象"；3D Tiles 是"自动加载海量对象"。当你需要显示 100 个标注 → 用 Entity；当你需要显示 10 万栋建筑 → 用 3D Tiles。

3D Tiles 加载实战：城市建筑、地形、点云

实战

🏠 加载 Cesium Ion 上的 OSM Buildings

Cesium Ion 提供了免费的 OpenStreetMap 全球建筑数据，是最容易上手的 3D Tiles 数据集。

const osmBuildings = viewer.scene.primitives.add(
  new Cesium.Cesium3DTileset({
    url: Cesium.IonResource.fromAssetId(96188) // OSM Buildings 资产 ID
  })
);

// 飞到建筑区域
viewer.zoomTo(osmBuildings);

🌌 加载 Cesium World Terrain（全球地形）

// 方式一：通过 viewer 配置（推荐）
const viewer = new Cesium.Viewer('cesiumContainer', {
  terrain: Cesium.Terrain.fromWorldTerrain()  // 自动加载全球地形
});

// 方式二：手动添加 Terrain Provider
viewer.scene.setTerrain(
  Cesium.Terrain.fromWorldTerrain({
    requestVertexNormals: true,  // 请求光照 normals
    requestWaterMask: true       // 请求水面效果
  })
);

📷 加载自定义 3D Tiles 数据

// 从 URL 加载
const tileset = viewer.scene.primitives.add(
  new Cesium.Cesium3DTileset({
    url: './data/tileset.json'  // 本地或远程 tileset.json
  })
);

// 加载完成后飞到数据范围
tileset.readyPromise.then((t) => {
  console.log('Tileset 加载成功！');
  console.log('总 Tile 数量:', t.statistics.numberOfTilesTotal);
  viewer.zoomTo(t);
}).catch((err) => {
  console.error('加载失败:', err);
});

🛠️ Tileset 常用配置参数

参数类型含义
urlString | Resourcetileset.json 地址
maximumScreenSpaceErrorNumberLOD 精度阈值（默认16），越小越精细
maximumMemoryUsageNumber内存上限 MB（默认512）
dynamicScreenSpaceErrorBoolean动态 LOD 调整（移动时降低精度）
showBoolean是否显示
modelMatrixMatrix4变换矩阵（平移/旋转/缩放）
skipLevelOfDetailBoolean跳级加载（更快但有跳跃）
baseScreenSpaceErrorNumber基础 LOD 阈值
⚠️
maximumScreenSpaceError 是性能关键：默认值 16 表示"屏幕上误差超过 16 像素就加载更精细的 Tile"。调低（如 2）= 更精细但更慢；调高（如 32）= 更粗略但更快。生产环境建议 4-8。

3D Tiles 样式与过滤：Cesium3DTileStyle

样式

🎨 Cesium3DTileStyle — 批量设置外观

3D Tiles 中的每个 Tile 可以包含成千上万个建筑/对象，每个对象有自己的属性（高度、名称、类型等）。Cesium3DTileStyle 让你基于这些属性批量设置颜色、显示/隐藏。

// 简单样式：所有建筑统一颜色
osmBuildings.style = new Cesium.Cesium3DTileStyle({
  color: 'color("white")',
  show: true
});

// 条件样式：根据建筑高度着色
osmBuildings.style = new Cesium.Cesium3DTileStyle({
  color: {
    conditions: [
      ['${Height} >= 100', 'color("red")'],       // 高层：红色
      ['${Height} >= 50',  'color("orange")'],    // 中层：橙色
      ['${Height} >= 20',  'color("yellow")'],    // 低层：黄色
      ['true',              'color("white")']      // 默认：白色
    ]
  },
  show: {
    conditions: [
      ['${Height} >= 5', true],   // 只显示高度 ≥ 5 的建筑
      ['true', false]                // 其他隐藏
    ]
  }
});

🔬 样式表达式语法

表达式含义
${Height}读取 Tile 的 Height 属性
${name}读取 name 属性
color("red")纯色
color("#ff0000", 0.5)带透明度
rgb(255, 0, 0)RGB 颜色
rgba(255, 0, 0, 0.5)RGBA 颜色
${Height} * 2数学运算
${Height} >= 100条件判断
regExp(${name}, "医院|学校")正则匹配

🛠️ 实战：只显示医院和学校

osmBuildings.style = new Cesium.Cesium3DTileStyle({
  color: {
    conditions: [
      ['regExp(${name}, "医院")', 'color("#ff0000")'],    // 红色
      ['regExp(${name}, "学校")', 'color("#0000ff")'],    // 蓝色
      ['true', 'color("white")']
    ]
  },
  show: {
    conditions: [
      ['regExp(${name}, "医院|学校")', true],
      ['true', false]
    ]
  }
});
💡
样式 = JS 条件逻辑的地图版：你写的 ${Height} >= 100 条件表达式，本质上就是 JavaScript 的 if/else，只是用声明式语法写在了 3D Tiles 上。这和 CSS 选择器、SQL WHERE 是同一思想——声明式过滤。

🛠️ 综合示例：城市热力图 + 建筑高度着色

📋 今日知识点总结

  • 🎨 Material 决定图形外观：Color（纯色）、Image（贴图）、PolylineGlow/Arrow/Dash（折线专用）、Checkerboard/Stripe/Grid（图案）、Composite（组合）

  • 🎨 Image 材质：可贴图片 URL、Canvas 对象；Canvas 是动态材质的最低门槛方式
  • 🎨 动态材质：CallbackProperty 让颜色每帧变化；Canvas + clock.onTick 让纹理实时更新
  • 🎨 3D Tiles 是 OGC 标准，用于海量 3D 地理数据流式加载；LOD 机制保证性能
  • 🎨 Cesium3DTileset：url / maximumScreenSpaceError / maximumMemoryUsage 是核心参数
  • 🎨 Cesium3DTileStyle：用 ${property} 表达式批量着色和过滤，conditions 语法 = 声明式 if/else
  • 🎨 材质 + 3D Tiles = 可视化左右手：Entity 材质处理自建标注，3D Tiles 处理外部海量数据
