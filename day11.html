同步 vs 异步：被"阻塞"的真相

核心概念

⚡ JavaScript 是单线程的

JavaScript 在浏览器中只有一个主线程（叫"调用栈"或 Call Stack）。同一时间只能做一件事。如果某件事需要 3 秒，那么整个页面会冻结 3 秒——用户什么也点不了，滚动也不行。

🔴 同步代码（阻塞）

console.log('1: 开始');

// 模拟耗时操作：循环 3 亿次
const start = Date.now();
while (Date.now() - start < 3000) {}
// ↑ 3 秒内页面完全卡死

console.log('2: 结束');

🟢 异步代码（非阻塞）

console.log('1: 开始');

setTimeout(() => {
  console.log('2: 3秒后执行');
}, 3000);
// ↑ 不会卡住页面！

console.log('3: 立即执行');
// 输出顺序：1 → 3 → 2
💡
核心理解：异步不是"同时做多件事"（那是多线程）。异步是"把耗时任务交给别人，我继续往下走，等它做完再通知我"。这个"别人"在浏览器里叫 Web APIs（setTimeout、fetch、DOM 事件等），在 Node.js 里叫 libuv。

回调函数与"回调地狱"

历史遗留

💬 回调函数：最原始的异步方案

异步操作完成后，把"下一步做什么"作为函数参数传进去。这个被传进去的函数就叫"回调函数"（callback）。

// setTimeout 的第二个参数就是一个回调函数
setTimeout(function() {
  console.log('2秒后执行这个回调');
}, 2000);

// 事件监听也是回调
document.addEventListener('click', function() {
  console.log('点击时执行这个回调');
});

// Node.js 风格回调 (error-first callback)
fs.readFile('/data.json', (err, data) => {
  if (err) return console.error(err);
  console.log(data);
});
⚠️
回调地狱（Callback Hell / Pyramid of Doom）：当异步任务需要按顺序执行时（A 做完做 B，B 做完做 C），层层嵌套的回调让代码变成"箭头金字塔"——读起来像一坨意大利面，错误处理几乎无法统一。

😱 回调地狱示例

// 模拟：加载用户 → 加载该用户的订单 → 加载订单详情 → 加载商品信息
loadUser(userId, (err, user) => {
  if (err) return handleError(err);
  loadOrders(user.id, (err, orders) => {
    if (err) return handleError(err);
    loadOrderDetail(orders[0].id, (err, detail) => {
      if (err) return handleError(err);
      loadProduct(detail.productId, (err, product) => {
        if (err) return handleError(err);
        console.log(product.name); //  终于拿到结果!
      });
    });
  });
});
// 每多加一层依赖，缩进就多一级...这就是回调地狱

👇 后面你会看到，Promise 和 async/await 如何把这个"箭头金字塔"变成"平直流水线"。

Promise 核心概念：三种状态

基石

🧾 Promise 是什么？

Promise 是 ES6 引入的一个对象，代表一个将来才会完成的异步操作。你可以把它想象成"外卖订单"：

  • ⏳ pending（待处理）：你刚下了单，外卖还没到
  • ✅ fulfilled（已完成）：外卖送到了，你拿到了食物（resolve）
  • ❌ rejected（已拒绝）：商家取消了订单（reject）
pending 等待中
➔
fulfilled resolve(value)
或
rejected reject(error)

🔧 Promise 基本用法

// 1. 创建一个 Promise（通常由 API 库创建，但了解原理很重要）
const promise = new Promise((resolve, reject) => {
  // 这里执行异步操作
  setTimeout(() => {
    const success = Math.random() > 0.3;
    if (success) {
      resolve({ lat: 39.9, lng: 116.4 });  // 成功：传结果
    } else {
      reject(new Error('定位失败'));        // 失败：传错误
    }
  }, 1000);
});

// 2. 消费 Promise —— .then / .catch / .finally
promise
  .then(result => {
    console.log(' 成功！位置:', result.lat, result.lng);
  })
  .catch(error => {
    console.error(' 失败:', error.message);
  })
  .finally(() => {
    console.log(' 无论成功失败都执行');  // 比如：关闭 loading
  });
ℹ️
Promise 的三个关键规则：
1. Promise 的状态一旦改变就不可逆（pending → fulfilled 就不能再变成 rejected）
2. .then() 总是返回一个新的 Promise，这是链式调用的基础
3. .catch() 本质上是 .then(null, errorHandler) 的语法糖
4

Promise 链式调用

模式

🔗 用 .then() 串联异步操作

每个 .then() 返回一个新 Promise，所以你可以用平直的形式写出顺序依赖的异步操作——告别回调地狱！

// 回调地狱 → Promise 链（"平直流水线"）
loadUser(userId)
  .then(user => {
    console.log('用户:', user.name);
    return loadOrders(user.id);      // 返回 Promise
  })
  .then(orders => {
    console.log('订单数:', orders.length);
    return loadOrderDetail(orders[0].id);
  })
  .then(detail => {
    console.log('详情:', detail);
    return loadProduct(detail.productId);
  })
  .then(product => {
    console.log(' 最终商品:', product.name);
  })
  .catch(err => {
    console.error(' 任意环节出错都会到这里:', err);
  });

🚀 .then() 返回值规则

.then() 返回...下一个 .then() 收到...
普通值 return 42自动包装成 Promise.resolve(42)
Promise return fetch(...)等待该 Promise 完成，拿到结果
thenable 对象 {then:fn}调用其 .then() 方法
抛出错误 throw err跳到最近的 .catch()
无 returnundefined（包装成 resolved Promise）
⚠️
常见坑：忘记 return。如果你在 .then() 里调了一个异步函数但没有 return，后面的 .then() 不会等待它完成，会立即收到 undefined。

Promise 静态方法大合集

工具箱

🛠️ 6 个核心静态方法

方法行为何时用
Promise.all 等全部成功才成功；一个失败即失败 所有数据缺一不可：同时加载地形+影像+3DTiles
Promise.allSettled 等全部完成（不管成败），返回每项结果 批量请求但不想互相影响：加载多个 GeoJSON 图层
Promise.race 第一个完成的决定结果 超时控制：fetch 超 5 秒就失败
Promise.any 第一个成功的返回；全部失败才 reject 多 CDN 容灾：哪个先加载完用哪个
Promise.resolve 包装成已成功的 Promise 统一异步接口
Promise.reject 包装成已失败的 Promise 快速创建 rejected Promise

Promise.all

// 全部成功才成功
const results = await Promise.all([
  fetch('/api/tiles'),
  fetch('/api/terrain'),
  fetch('/api/imagery')
]);
// 少一个都拿不到 results

Promise.allSettled

// 宽容并发：好用的给结果
// 失败的也不影响别人
const results = await Promise.allSettled([
  loadLayer('roads'),
  loadLayer('buildings'),
  loadLayer('trees')
]);
results.forEach(r => {
  if (r.status === 'fulfilled')
    addToMap(r.value);
});

Promise.race

// 超时控制
const timeout = new Promise((_, reject) =>
  setTimeout(() =>
    reject('请求超时'), 5000)
);
const data = await Promise.race([
  fetch('/api/geo'),
  timeout
]);

Promise.any

// 多 CDN 容灾
const terrain = await Promise.any([
  fetch('https://cdn1.com/terrain'),
  fetch('https://cdn2.com/terrain'),
  fetch('https://cdn3.com/terrain')
]);
// 哪个 CDN 先返回就用哪个
💡
选型速记：all = 缺一不可，allSettled = 爱谁谁，race = 先到先得，any = 有一个成功就行。在 CesiumJS 中，加载图层常用 allSettled（某层失败不影响其他），初始化关键资源用 all（地形+影像缺一不可）。

async/await：异步的终极语法

ES2017

🌟 让异步代码"看起来像同步"

async/await 是 ES2017 引入的语法糖，它建立在 Promise 之上。async 函数总是返回一个 Promise，await 会暂停函数执行直到 Promise 完成。

// Promise 链式写法
function getCityInfo(name) {
  return fetch(`/api/city/${name}`)
    .then(res => res.json())
    .then(data => data.coordinates);
}

// async/await 写法 —— 清晰 10 倍！
async function getCityInfo(name) {
  const res = await fetch(`/api/city/${name}`);
  const data = await res.json();
  return data.coordinates;
}

📚 async/await 核心规则

规则说明
1. async 函数自动返回 Promiseasync fn() { return 42 } 等价于 fn() { return Promise.resolve(42) }
2. await 只能在 async 函数内用顶层 await 在 ES2022 模块中可用，但普通脚本不可以
3. await 会暂停执行等待 Promise 完成，不阻塞主线程（其他代码照常运行）
4. await 后可以跟任何值如果不是 Promise，自动包装成 Promise.resolve(value)

🔄 CesiumJS 中的 async/await 实战

CesiumJS 的许多操作返回 Promise——资源加载、地形请求、影像获取等。下面是典型写法：

// CesiumJS 中初始化场景的 async 写法
async function initCesiumApp(containerId) {
  // 1. 创建 Viewer
  const viewer = new Cesium.Viewer(containerId);

  // 2. 等待地形加载（返回 Promise）
  const terrainProvider = await Cesium.createWorldTerrainAsync();
  viewer.terrainProvider = terrainProvider;

  // 3. 加载 3D Tiles 建筑模型
  const tileset = await Cesium.Cesium3DTileset.fromUrl(
    'https://assets.cesium.com/tilesets/boston/tileset.json'
  );
  viewer.scene.primitives.add(tileset);

  // 4. 飞到目标位置（也是异步的）
  await viewer.flyTo(tileset);

  console.log(' 场景初始化完成！');
  return viewer;
}

async/await 错误处理

实践

🛡️ 用 try/catch 包裹 await

async/await 的最大优势之一：可以用同步风格的 try/catch 来处理异步错误，不再需要 .catch() 链。

async function safeLoadTileset(url, viewer) {
  try {
    const tileset = await Cesium.Cesium3DTileset.fromUrl(url);
    viewer.scene.primitives.add(tileset);
    return tileset;
  } catch (err) {
    console.error('3D Tiles 加载失败:', err.message);
    showUserMessage('抱歉，建筑模型加载失败');
    return null;
  }
}
⚠️
常见坑：忘记 try/catch。如果 async 函数内部抛出错误但外部没有 .catch()，会变成 UnhandledPromiseRejection。Node.js 15+ 会直接终止进程！

🔥 高级：Go 风格的错误处理

有些人喜欢用 [error, data] 元组模式，避免过多的 try/catch 嵌套：

// 包装函数：把 Promise 转成 [error, data]
async function to(promise) {
  try {
    const data = await promise;
    return [null, data];
  } catch (error) {
    return [error, null];
  }
}

// 使用——不用 try/catch，像 Go 一样
const [err, tileset] = await to(
  Cesium.Cesium3DTileset.fromUrl(url)
);
if (err) {
  handleError(err);
  return;
}
viewer.scene.primitives.add(tileset);

🔃 并行 await：别串行化不该串行的

多个 await 按顺序写会变成串行执行——这是新手常见性能坑：

// ❌ 错误：串行执行（慢）
const user    = await fetchUser();    // 1秒
const orders  = await fetchOrders();  // +1秒 = 总共2秒
const config  = await fetchConfig();  // +1秒 = 总共3秒

// ✅ 正确：并行执行（快）——它们不互相依赖！
const [user, orders, config] = await Promise.all([
  fetchUser(),
  fetchOrders(),
  fetchConfig()
]);
// 同时发起 3 个请求，等最慢的那个 → 约1秒

口诀：如果 B 依赖 A 的结果，串行 await；如果 A、B、C 互不依赖，用 Promise.all 并行。

事件循环：微任务 vs 宏任务

底层机制

🌀 JavaScript 的执行模型

理解事件循环是面试必考题，更是理解异步执行顺序的关键。记住这个口诀："同步先跑完，微任务清空，宏任务一个一个来"

同步代码（调用栈）
➔
微任务队列
Promise.then, MutationObserver
➔
宏任务队列
setTimeout, setInterval, I/O
每轮事件循环：执行1个宏任务 → 清空所有微任务 → 渲染（如果需要） → 下一个宏任务

🔬 经典面试题：输出顺序？

console.log(1);

setTimeout(() => console.log(2), 0);

Promise.resolve().then(() => console.log(3));

console.log(4);

// 答案：1 → 4 → 3 → 2
// 原因：同步先跑(1,4) → 清空微任务(3) → 执行宏任务(2)

微任务 (Microtask)

  • Promise.then()
  • Promise.catch()
  • Promise.finally()
  • async/await（本质是 Promise）
  • queueMicrotask()
  • MutationObserver

每轮清空所有微任务

宏任务 (Macrotask)

  • setTimeout
  • setInterval
  • setImmediate（Node.js）
  • I/O 操作
  • UI 渲染
  • postMessage

每轮只取一个执行

💡
实际意义：在 CesiumJS 中，你经常需要"等下一帧"再执行某些操作。推荐用 requestAnimationFrame（属于渲染队列，在宏任务之后）。对于"等这个 tick 结束"，用 Promise.resolve().then(...) 或 queueMicrotask 把代码放到微任务。

异步并发控制模式

进阶

🏭 批量请求但限制并发数

实际场景：你有 100 个 3D Tiles 文件要加载，但一次只能同时加载 5 个（浏览器并发连接限制 + 避免内存爆炸）。

// 并发控制：同时最多 N 个任务
async function asyncPool(limit, tasks) {
  const results = [];
  const executing = new Set();

  for (const task of tasks) {
    const p = Promise.resolve().then(() => task());
    results.push(p);
    const e = p.then(() => executing.delete(e));
    executing.add(e);

    if (executing.size >= limit) {
      await Promise.race(executing);  // 等有一个完成
    }
  }

  return Promise.allSettled(results);
}

// 使用：加载 100 个 tileset，同时最多 5 个
const urls = ['tile_0.json', 'tile_1.json', ...]; // 100 个
const tasks = urls.map(url => () => loadTileset(url));
const results = await asyncPool(5, tasks);

🕕 轮询与重试

CesiumJS 中加载资源可能需要重试：

// 带重试的 fetch
async function fetchWithRetry(url, maxRetries = 3, delay = 1000) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const res = await fetch(url);
      if (res.ok) return res;
      throw new Error(`HTTP ${res.status}`);
    } catch (err) {
      if (i === maxRetries - 1) throw err;
      console.warn(`重试 ${i+1}/${maxRetries}: ${url}`);
      await new Promise(r => setTimeout(r, delay * (i + 1)));  // 递增延迟
    }
  }
}

CesiumJS 关联场景

目标

🌐 场景 1：顺序初始化场景

典型的 CesiumJS 应用：创建 Viewer → 加载地形 → 加载影像 → 添加 3D Tiles → 飞到目标。每一步依赖前一步，用 async/await 清晰表达：

async function bootScene(containerId) {
  // 步骤 1：创建 Viewer
  const viewer = new Cesium.Viewer(containerId);

  // 步骤 2：加载地形（基于步骤1）
  viewer.terrainProvider = await Cesium.CesiumTerrainProvider.fromUrl(
    'https://assets.cesium.com/1/terrain/world'
  );

  // 步骤 3：加载建筑模型（基于步骤1）
  const buildings = await Cesium.createOsmBuildingsAsync();
  viewer.scene.primitives.add(buildings);

  // 步骤 4：飞过去（基于步骤3的结果）
  await viewer.flyTo(buildings);

  return viewer;
}

🌍 场景 2：并行加载多个图层

多个图层互不依赖，应该并行加载：

async function loadLayersConcurrently(viewer) {
  // 3 个图层互不依赖，同时加载
  const [roads, buildings, trees] = await Promise.allSettled([
    loadGeoJSON('/data/roads.geojson'),
    loadTileset('/tiles/boston/'),
    loadCZML('/data/trees.czml')
  ]);

  // 把加载成功的都添加到场景
  [roads, buildings, trees].forEach(result => {
    if (result.status === 'fulfilled') {
      viewer.dataSources.add(result.value);
    } else {
      console.warn('某图层加载失败，但不影响其他图层:', result.reason);
    }
  });
}

📊 场景 3：带超时的资源加载

防止某个 3D Tiles 加载卡住整个应用：

async function loadWithTimeout(promise, timeoutMs = 30000) {
  const timeout = new Promise((_, reject) =>
    setTimeout(() =>
      reject(new Error(`加载超时（${timeoutMs}ms）`)),
      timeoutMs
    )
  );
  return Promise.race([promise, timeout]);
}

// 使用：
const tileset = await loadWithTimeout(
  Cesium.Cesium3DTileset.fromUrl(url),
  20000  // 20秒超时
);

🚀 场景 4：实时数据流——轮询与增量更新

CesiumJS 中显示动态数据（飞机位置、车辆轨迹等）需要持续更新：

class RealTimeTracker {
  constructor(viewer, apiEndpoint) {
    this.viewer = viewer;
    this.endpoint = apiEndpoint;
    this.entities = new Map();
    this.running = false;
  }

  async start(intervalMs = 2000) {
    this.running = true;
    while (this.running) {
      try {
        const positions = await fetch(this.endpoint)
          .then(r => r.json());
        this.updateEntities(positions);
      } catch (err) {
        console.warn('轮询失败，下次重试:', err.message);
      }
      // 等待下一轮
      await new Promise(r => setTimeout(r, intervalMs));
    }
  }

  stop() { this.running = false; }

  updateEntities(positions) {
    positions.forEach(p => {
      let entity = this.entities.get(p.id);
      if (!entity) {
        entity = this.viewer.entities.add({
          position: Cesium.Cartesian3.fromDegrees(p.lng, p.lat),
          point: { pixelSize: 8, color: Cesium.Color.RED }
        });
        this.entities.set(p.id, entity);
      }
      // 更新位置
      entity.position = Cesium.Cartesian3.fromDegrees(p.lng, p.lat);
    });
  }
}
💡
CesiumJS 异步编程的核心：几乎所有资源加载都是异步的（地形、影像、3D Tiles、GeoJSON、CZML）。掌握 Promise/async-await 是高效使用 CesiumJS 的前提。记住：串行依赖用 async/await 链；独立资源用 Promise.allSettled 并行；超时就 race；资源重试用循环+timer；动态更新用定时轮询或 WebSocket。

题 1：从回调到 Promise 的转换

下面的 getLocation 使用回调风格。把它改写成返回 Promise 的版本 getLocationAsync，然后用 .then 消费它。

// 原始回调风格函数
function getLocation(callback) {
  setTimeout(() => {
    const lat = 39.9042, lng = 116.4074;
    callback(null, { lat, lng });  // error-first 风格

📋 今日知识点总结

  • ✅ JavaScript 是单线程的，异步通过事件循环 + 任务队列实现，不阻塞主线程
  • ✅ 回调函数是最原始的异步方案，但嵌套会导致"回调地狱"
  • ✅ Promise 有三种状态：pending → fulfilled / rejected，状态不可逆
  • ✅ .then() 返回新 Promise，必须 return 才能传递结果给下一个 .then()
  • ✅ Promise.all（缺一不可）、allSettled（宽容）、race（先到先得）、any（一胜即可）
  • ✅ async/await 是 Promise 的语法糖，让异步代码看起来像同步
  • ✅ async 函数始终返回 Promise，await 会暂停执行但不阻塞线程
  • ✅ 用 try/catch 包裹 await 处理错误；互不依赖的任务用 Promise.all 并行
  • ✅ 事件循环：同步 → 清空微任务（Promise.then）→ 一个宏任务（setTimeout）→ 渲染 → 循环
  • ✅ 并发控制：用计数器 + Promise.race 实现"池限制"；重试用循环 + 递增延迟
  • ✅ CesiumJS 中：场景初始化是串行 async/await；多图层加载用 allSettled 并行；资源加载用 race 做超时；实时数据用轮询
