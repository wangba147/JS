为什么地形与影像如此重要

动机

🌋 无地形 vs 有地形

默认情况下，CesiumJS 的 Globe 是一个光滑的数学椭球体——没有山峰、没有峡谷、没有海底起伏。这就像把一个地球仪涂上地图照片：表面是平的。

当你加载真实地形数据后，地球就有了高低起伏：珠穆朗玛峰真的会凸起来，马里亚纳海沟真的会凹下去。相机贴地飞行时，你会看到山脉、河谷——这才是真正的三维地球。

🖼 影像层：地球的"皮肤"

影像（Imagery）就是贴在地球表面的地图图片。没有影像，地球就是一个灰色的素模。有了影像，你能看到卫星图、街道图、行政区划、实时天气图——任何栅格地图数据都可以作为影像图层。

而且可以叠多层：底层放卫星图，上面叠加半透明行政区划，最上面标注道路。这就是 GIS 里的"图层叠加"概念。

💡 核心概念：Globe 对象有两个关键属性——terrainProvider（决定"有多高"）和 imageryLayers（决定"长什么样"）。地形是3D几何体，影像是2D贴图。两者独立管理，互不干扰。

Globe 对象：地球的"皮肤"与"骨架"

架构

🌐 Globe = TerrainProvider + ImageryLayers

Viewer.scene.globe 是 CesiumJS 中表示地球的核心对象。它内部管理两个子系统：

Viewer.scene.globe
地球对象
↘            ↙
terrainProvider
地形数据源（骨架）
imageryLayers
影像图层集合（皮肤）
↓
↓
地形瓦片 (QuantizedMesh)
每个瓦片携带高程数据
影像瓦片 (JPEG/PNG/WebP)
每层叠加不同地图

两者的共同目标：按需加载瓦片（Tiled），只加载当前视野范围内的数据，用金字塔结构实现 LOD（远近不同精度）。

🔍 globe 的核心配置属性

属性类型说明
terrainProvider TerrainProvider 地形数据源。默认 EllipsoidTerrainProvider（光滑椭球）
imageryLayers ImageryLayerCollection 影像图层集合，支持增删、排序、透明度
baseColor Color 无影像覆盖区域的底色（默认深蓝）
show Boolean 是否显示地球
depthTestAgainstTerrain Boolean 实体是否做地形深度测试（false 则浮在地形上方）
enableLighting Boolean 地形是否受光照影响（阴影）
showWaterEffect Boolean 是否显示水面效果
3

TerrainProvider：地形提供者体系

核心

🗺 TerrainProvider 是什么？

TerrainProvider 是一个抽象接口，负责按需向 Globe 提供地形高程数据。CesiumJS 内置了多种实现：

类名数据源精度需要 Token
EllipsoidTerrainProvider 数学椭球体 无真实高程 否（默认）
CesiumTerrainProvider Cesium ion 全球地形 ~30m (全球) / ~1m (部分地区) 需要
ArcGISTiledElevationTerrainProvider ArcGIS 高程服务 取决于服务配置 视情况
VRTheWorldTerrainProvider VRTheWorld 中等 否
GoogleEarthEnterpriseTerrainProvider Google Earth Enterprise 高 需要

🔨 地形数据格式：QuantizedMesh

CesiumJS 使用 QuantizedMesh 格式来编码地形瓦片。每个瓦片不仅包含高程值，还包含：

  • 顶点法线：用于光照计算，让山脊有明暗面
  • 水面遮罩：标记哪些区域是水体
  • 元数据：最小/最大高程、边界范围
  • 索引数据：用于高效查询高程

这是一种二进制格式，比传统 DEM（数字高程模型）更紧凑，专为 Web 传输优化。

4

EllipsoidTerrainProvider：无地形模式

默认

🔵 光滑椭球体

这是 CesiumJS 的默认地形提供者。它不加载任何真实高程数据，地球表面就是一个完美的数学椭球体（WGS84）。

// 默认情况下，就等于这样：
const viewer = new Cesium.Viewer('cesiumContainer', {
  terrainProvider: new Cesium.EllipsoidTerrainProvider()
});

// 等价于不传 terrainProvider，CesiumJS 自动创建

🤔 什么时候用？

  • ✅ 不需要地形起伏的场景（如只看全球数据分布）
  • ✅ 性能敏感场景——椭球体零开销
  • ✅ 快速原型开发阶段
  • ❌ 需要相机贴地飞行、分析地形剖面
  • ❌ 需要遮挡分析（山体会挡住后面的东西）
⚠️ 注意：使用 EllipsoidTerrainProvider 时，sampleTerrain() 返回的高程值始终为 0。所有实体都"浮"在椭球面上，即使它们应该在海底。
5

CesiumTerrainProvider：Cesium ion 全球地形

推荐

🌋 最常用的真实地形方案

Cesium ion 提供的全球地形服务，底层整合了多种数据源：

  • Cesium World Terrain：融合了 SRTM、ASTER GDEM、NASADEM 等多种全球 DEM 数据，默认精度约 30m
  • Cesium World Terrain + 高精度区域：部分地区（如美国、欧洲）可达 1m 精度
  • 海底地形：包含海洋测深数据，海底也有起伏

🔧 基本用法

// 方式 1：创建 Viewer 时指定（推荐）
const viewer = new Cesium.Viewer('cesiumContainer', {
  terrainProvider: new Cesium.CesiumTerrainProvider({
    url: Cesium.IonResource.fromAssetId(1),  // Asset ID 1 = Cesium World Terrain
    requestVertexNormals: true,  // 请求法线，用于光照
    requestWaterMask: true      // 请求水面遮罩，显示水面效果
  })
});

// 方式 2：使用 createWorldTerrain 快捷函数（v1.60+）
const viewer = new Cesium.Viewer('cesiumContainer', {
  terrainProvider: await Cesium.createWorldTerrain({
    requestVertexNormals: true,
    requestWaterMask: true
  })
});

⚙️ 关键配置选项

选项默认值说明
requestVertexNormals false 是否请求法线数据。开启后地形有光照阴影，立体感更强
requestWaterMask false 是否请求水面遮罩。开启后海洋区域显示水面波光效果
ellipsoid Ellipsoid.WGS84 参考椭球体。月球/火星需改为对应的椭球体
💡 最佳实践：生产环境建议同时开启 requestVertexNormals 和 requestWaterMask。视觉效果提升显著，性能影响很小——法线是瓦片自带的，不需要额外请求。

ArcGISTiledElevationTerrainProvider

ArcGIS

🌐 对接 ArcGIS 高程服务

如果你的企业使用 ArcGIS 生态，可以直接连接 ArcGIS 发布的高程服务：

const viewer = new Cesium.Viewer('cesiumContainer', {
  terrainProvider: new Cesium.ArcGISTiledElevationTerrainProvider({
    url: 'https://elevation3d.arcgis.com/arcgis/rest/services/WorldElevation3D/Terrain3D/ImageServer',
    token: 'YOUR_ARCGIS_TOKEN'  // 如果需要
  })
});

ArcGIS 高程服务使用LERC 压缩格式，压缩率极高，适合企业内网部署。

ImageryLayer：影像图层基础

核心

🎨 什么是 ImageryLayer？

ImageryLayer 是 CesiumJS 中的单个影像图层。每个图层包装一个 ImageryProvider（数据源），并额外控制：

  • 透明度（alpha）
  • 亮度/对比度/色相/饱和度/伽马
  • 显示范围（按矩形裁剪）
  • 层级顺序（在多图层中的叠放位置）

🔧 创建和添加图层

// 方式 1：创建 Viewer 时指定默认底图
const viewer = new Cesium.Viewer('cesiumContainer', {
  imageryProvider: new Cesium.ArcGisMapServerImageryProvider({
    url: 'https://services.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer'
  })
});

// 方式 2：通过 imageryLayers.add() 添加
const layer = viewer.imageryLayers.addImageryProvider(
  new Cesium.IonImageryProvider({ assetId: 3 })  // Bing Maps Aerial
);

// 方式 3：使用 addImageryProvider 快捷方法
const layer = viewer.imageryLayers.addImageryProvider(
  new Cesium.OpenStreetMapImageryProvider({
    url: 'https://tile.openstreetmap.org/'
  })
);

🎨 ImageryLayer 的属性控制

// 获取图层引用后，可以动态调整样式
layer.alpha = 0.5;              // 透明度 0~1
layer.brightness = 1.2;        // 亮度，默认 1.0
layer.contrast = 1.1;         // 对比度
layer.hue = 0.1;              // 色相偏移
layer.saturation = 1.5;      // 饱和度
layer.gamma = 1.2;            // 伽马校正
layer.show = true;            // 是否显示
layer.splitDirection = Cesium.SplitDirection.LEFT;  // 分屏方向（用于卷帘对比）
💡 实用技巧：alpha 属性非常适合做"图层对比"。让上层半透明，就能同时看到上下两层地图。配合 splitDirection 还能做左右分屏对比（卷帘效果）。

ImageryProvider：影像提供者大全

参考

🗃 内置 ImageryProvider 一览

类名数据源类型需要 Key
ArcGisMapServerImageryProvider ArcGIS MapServer / ImageServer 卫星 / 街道 / 地形 否（公用服务）
BingMapsImageryProvider Bing Maps 卫星 / 街道 / 混合 需要 Bing Key
IonImageryProvider Cesium ion 托管 多种（Bing/Mapbox等） 需要 ion Token
OpenStreetMapImageryProvider OpenStreetMap 街道地图 否
TileMapServiceImageryProvider 标准 TMS 服务 任意 否

WebMapTileServiceImageryProvider WMTS 标准服务 任意 否

GridImageryProvider 内置（程序生成） 网格线 否

TileCoordinatesImageryProvider 内置（程序生成） 瓦片坐标标签 否

🛠 最常用的几个

// 1. Bing Maps 卫星图（通过 Cesium ion）
new Cesium.IonImageryProvider({ assetId: 2 })  // Bing Maps Aerial

// 2. Bing Maps 街道图（通过 Cesium ion）
new Cesium.IonImageryProvider({ assetId: 4 })  // Bing Maps Road

// 3. ArcGIS 全球影像（免费，无需 key）
new Cesium.ArcGisMapServerImageryProvider({
  url: 'https://services.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer'
})

// 4. OpenStreetMap（免费，无需 key）
new Cesium.OpenStreetMapImageryProvider({
  url: 'https://tile.openstreetmap.org/'
})

// 5. 自定义 XYZ 瓦片（如天地图、高德、Google 等）
new Cesium.UrlTemplateImageryProvider({
  url: 'https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png',
  subdomains: ['a', 'b', 'c']
})
💡 UrlTemplateImageryProvider 是万能钥匙：只要你知道 XYZ 瓦片 URL 模板（{z}/{x}/{y}），几乎任何在线地图都能接入 CesiumJS。天地图、高德、谷歌地图、Mapbox——都支持这种标准格式。

图层叠加、排序与透明度控制

实战

📊 ImageryLayerCollection 操作

viewer.imageryLayers 是一个 ImageryLayerCollection，管理所有影像图层。核心操作：

方法说明
add(layer, index)添加图层，可指定插入位置（index 越大越上层）
addImageryProvider(provider, index)快捷方式：从 provider 创建图层并添加
remove(layer, destroy)移除图层
removeAll(destroy)移除所有图层
contains(layer)是否包含某图层
indexOf(layer)获取图层索引
lower/lowerToBottom下移一层 / 移到底层
raise/raiseToTop上移一层 / 移到顶层
pickImageryLayerFeatures拾取某位置的图层要素（WMS 支持）

🌐 实战：多图层叠加

// 构建一个完整的图层栈：底图卫星 + 中间行政区划 + 顶部标注

const viewer = new Cesium.Viewer('cesiumContainer', {
  baseLayerPicker: false  // 关闭默认底图选择器
});

// 第 0 层（最底层）：卫星底图
const satellite = viewer.imageryLayers.addImageryProvider(
  new Cesium.ArcGisMapServerImageryProvider({
    url: 'https://services.arcgisonline.com/.../World_Imagery/MapServer'
  }),
  0  // index = 0，最底层
);

// 第 1 层：行政区划（半透明叠加）
const admin = viewer.imageryLayers.addImageryProvider(
  new Cesium.ArcGisMapServerImageryProvider({
    url: 'https://.../Reference/World_Boundaries/MapServer'
  }),
  1
);
admin.alpha = 0.6;  // 60% 透明度，能看到底下的卫星图

// 第 2 层（最顶层）：道路标注
const labels = viewer.imageryLayers.addImageryProvider(
  new Cesium.ArcGisMapServerImageryProvider({
    url: 'https://.../Reference/World_Transportation/MapServer'
  }),
  2
);

🔍 动态控制图层

// 切换图层可见性
satellite.show = false;  // 隐藏卫星图

// 调整图层顺序
viewer.imageryLayers.raise(admin);      // 上移一层
viewer.imageryLayers.raiseToTop(admin);  // 移到最顶层
viewer.imageryLayers.lowerToBottom(satellite); // 移到底层

// 动态调整透明度（实现淡入淡出）
let alpha = 1.0;
const fadeOut = setInterval(() => {
  alpha -= 0.01;
  admin.alpha = alpha;
  if (alpha <= 0) clearInterval(fadeOut);
}, 50);

// 移除图层
viewer.imageryLayers.remove(admin);  // 不移除 provider
viewer.imageryLayers.remove(admin, true); // 同时销毁 provider

高程采样：sampleTerrain / sampleTerrainMostDetailed

高级

📏 为什么要采样高程？

在地理信息系统中，经常需要知道某个经纬度位置的实际海拔高度：

  • 放置实体时贴地（clamp to ground）
  • 计算飞行路径的最低安全高度
  • 地形剖面分析
  • 洪水淹没模拟
  • 视域分析（通视计算）

CesiumJS 提供了 sampleTerrain() 和 sampleTerrainMostDetailed() 两个异步方法。

🔧 sampleTerrain() 用法

// sampleTerrain(terrainProvider, level, positions)
// - terrainProvider: 地形提供者
// - level: 采样精度级别（数字越大越精确，但越慢）
// - positions: Cartographic[] 要采样的位置数组
// 返回 Promise<Cartographic[]>，每个 Cartographic 的 height 被填充

const positions = [
  Cesium.Cartographic.fromDegrees(86.9250, 27.9881),   // 珠穆朗玛峰
  Cesium.Cartographic.fromDegrees(116.3972, 39.9075),  // 北京
  Cesium.Cartographic.fromDegrees(-155.5319, 19.5981), // 夏威夷（海平面附近）
];

Cesium.sampleTerrainMostDetailed(viewer.terrainProvider, positions)
  .then(sampledPositions => {
    sampledPositions.forEach((pos, i) => {
      console.log(`位置 ${i}: 高程 = ${pos.height.toFixed(2)} 米`);
    });
    // 输出类似：
    // 位置 0: 高程 = 8848.86 米（珠峰）
    // 位置 1: 高程 = 43.50 米（北京）
    // 位置 2: 高程 = 0.00 米（海面）
  });

⚙ sampleTerrain vs sampleTerrainMostDetailed

方法精度速度适用场景
sampleTerrain(provider, level, positions) 取决于 level 参数 可控 需要控制精度/性能的场景
sampleTerrainMostDetailed(provider, positions) 最高可用精度 较慢 需要最精确高程的场景（如贴地放置）
⚠️ 重要：如果使用 EllipsoidTerrainProvider，所有位置的高程返回值都是 0。必须使用真实地形提供者（如 CesiumTerrainProvider）才能获取真实高程。另外，采样是异步操作——需要等待瓦片加载完成。

CesiumJS 关联场景

应用

🌋 场景 1：飞行仿真地形回避

无人机/飞行仿真中，需要根据前方地形高度动态调整飞行路径。使用 sampleTerrainMostDetailed() 批量查询航线沿路高程，确保飞机始终高于地形。

// 航线沿路高程检测
const waypoints = [
  [116, 39], [117, 38], [118, 37]
].map(([lon, lat]) => 
  Cesium.Cartographic.fromDegrees(lon, lat)
);

const sampled = await Cesium.sampleTerrainMostDetailed(
  viewer.terrainProvider, waypoints
);
const safeAltitudes = sampled.map(p => p.height + 500); // 地形上方 500m

🎨 场景 2：图层卷帘对比

对比不同时期的卫星影像（如灾害前后对比），使用 splitDirection 实现左右分屏：

// 左半边：灾害前影像
beforeLayer.splitDirection = Cesium.SplitDirection.LEFT;
// 右半边：灾害后影像
afterLayer.splitDirection = Cesium.SplitDirection.RIGHT;
// 然后拖动中间的分隔条实时对比

🌊 场景 3：洪水淹没分析

给定一个水位高度，使用地形高程数据计算淹没范围。结合 sampleTerrain 批量查询和可视化。

// 伪代码：淹没分析
const waterLevel = 10; // 水位 10m
const grid = generateGrid(area, 100); // 区域 100x100 网格
const heights = await Cesium.sampleTerrainMostDetailed(
  viewer.terrainProvider, grid
);
const flooded = heights.filter(p => p.height < waterLevel);
// 将 flooded 区域用半透明蓝色 polygon 渲染

🔭 场景 4：多源数据融合显示

同一场景同时显示多种数据源：底层 Bing 卫星图 + 中层气象雷达图（WMS，半透明）+ 顶层交通流量热力图（自定义瓦片）：

// 三层数据融合
viewer.imageryLayers.addImageryProvider(bingSatellite, 0); // 底层
const radar = viewer.imageryLayers.addImageryProvider(weatherRadarWMS, 1);
radar.alpha = 0.5;
const traffic = viewer.imageryLayers.addImageryProvider(trafficHeatmap, 2);
traffic.alpha = 0.7;

练习 1：地形提供者切换

编写代码，在 CesiumJS Viewer 中创建两个按钮：一个切换到 EllipsoidTerrainProvider（无地形），一个切换到 CesiumTerrainProvider（Cesium ion 全球地形）。提示：动态修改 viewer.scene.globe.terrainProvider。

练习 2：图层管理器

创建一个 LayerManager 类，管理多个影像图层：

// 期望 API：
const mgr = new LayerManager(viewer);
mgr.addLayer('satellite', arcgisProvider);   // 按名称添加
mgr.addLayer('labels', osmProvider);
mgr.setAlpha('labels', 0.5);                // 按名称设置透明度
mgr.toggleLayer('labels');                     // 按名称切换显隐
mgr.removeLayer('labels');                    // 按名称移除

提示：内部用 Map 存储名称到图层引用（复习 Day 07 的 Map），用 ImageryLayerCollection 的 API 操作。

练习 3：高程查询工具

编写一个函数 async function getElevation(viewer, lon, lat)，输入经纬度，返回该位置的地形高程（米）。要求：

  • 使用 sampleTerrainMostDetailed
  • 如果地形提供者没有加载完成，返回 null 并给出提示
  • 如果使用 EllipsoidTerrainProvider，给出警告"当前无真实地形数据"
