数组基础：创建、访问、length

ARRAYS

数组（Array）是一个有序的数据集合，用方括号 [] 表示。每个元素都有从 0 开始的索引。

属性/方法 说明 示例
length 数组中元素的个数 [1,2,3].length → 3
索引访问 通过数字下标访问元素 arr[0] → 第一个元素
构造函数 用 new Array() 创建数组 new Array(3) → [空,空,空]
Array.of() 可靠的数组创建（ES6） Array.of(3) → [3] 
// ① 创建数组（三种常用方式）
const cities = ['北京', '上海', '广州', '深圳'];
const emptyArr = [];                      // 空数组
const nums = Array.of(10, 20, 30);      // ES6：创建含3个元素的数组

// ② 访问元素（索引从 0 开始）
console.log(cities[0]);                 // "北京" — 第1个
console.log(cities[cities.length - 1]); // "深圳" — 最后一个
console.log(cities[99]);               // undefined（索引越界不报错）

// ③ length 属性——就是数组元素个数
console.log(cities.length);           // 4

// ④ 数组可混合存放任意类型
const mixed = [42, "hello", true, null, {x: 1}, [1,2]];

// ⑤ 修改元素
cities[1] = '重庆';               // 将"上海"改为"重庆"
console.log(cities);                // ["北京","重庆","广州","深圳"]
💡
注意：new Array(3) 创建的是长度为 3 的空槽数组，而 Array.of(3) 创建的是含一个元素 3 的数组。推荐用字面量 [] 或 Array.of()。

数组方法：增删改查

MUTATE

🔧 会修改原数组的方法（Mutating）

let arr = [1, 2, 3];

// 尾部操作
arr.push(4);          // 在尾部添加 → [1,2,3,4]，返回 4（新长度）
arr.pop();            // 删除尾部元素 → [1,2,3]，返回 4（被删元素）

// 头部操作
arr.unshift(0);      // 在头部添加 → [0,1,2,3]，返回 4
arr.shift();          // 删除头部元素 → [1,2,3]，返回 0

// splice — 万能增删（起始索引, 删除个数, ...新增元素）
arr.splice(1, 1);        // 删除索引1的元素 → [1,3]
arr.splice(1, 0, 10, 20); // 在索引1处插入 → [1,10,20,3]
arr.splice(2, 1, 'X');     // 替换索引2 → [1,10,'X',3]

// reverse — 反转
[1,2,3].reverse();     // → [3,2,1]

// sort — 排序（默认按字符串排序，需要传入比较函数）
[3, 1, 10, 2].sort();                     // ⚠️ [1,10,2,3] — 字符串序！
[3, 1, 10, 2].sort((a, b) => a - b);   // [1,2,3,10] ✅ 数字序

🔍 不会修改原数组的方法（非变异）

const a = [1, 2, 3, 4, 5];

// 查找
a.indexOf(3);              // 2（第一个匹配索引，找不到返回 -1）
a.lastIndexOf(2);          // 1（最后一个匹配索引）
a.includes(3);              // true（是否包含该值）
a.find(x => x > 3);           // 4（第一个满足条件的元素）
a.findIndex(x => x > 3);      // 3（第一个满足条件的索引）

// 截取 / 拼接
const sub = a.slice(1, 4);       // [2,3,4]（拷贝，不含末索引）a 不变
const merged = a.concat([6,7]); // [1,2,3,4,5,6,7] a 不变
const joined = a.join(' - ');    // "1 - 2 - 3 - 4 - 5"（转字符串）

// ES6+：展开运算符（不修改原数组）
const copy = [...a];             // 浅拷贝：[1,2,3,4,5]
const extended = [...a, 6, 7];  // [1,2,3,4,5,6,7]
⚠️
sort() 默认按字符串 Unicode 排序！数字排序必须传入比较函数 (a,b) => a - b（升序）或 (a,b) => b - a（降序）。

数组迭代：map / filter / reduce

ITERATION

🔄 forEach — 遍历每个元素

对每个元素执行一次回调，不返回新数组。

const city = ['北京', '上海', '广州'];

city.forEach((name, index) => {
  console.log(`${index}: ${name}`);
});
// 0: 北京
// 1: 上海
// 2: 广州

🗺️ map — 映射（返回新数组）

对每个元素应用函数，返回新数组，元素个数不变。

const nums = [1, 2, 3, 4, 5];

const doubled = nums.map(x => x * 2);
console.log(doubled);
// [2, 4, 6, 8, 10]

🔍 filter — 过滤（返回新数组）

通过条件函数过滤，返回符合条件的元素组成的数组。

const nums = [1, 2, 3, 4, 5];

const even = nums.filter(x => x % 2 === 0);
console.log(even);
// [2, 4]

const big = nums.filter(x => x > 2);
console.log(big);
// [3, 4, 5]

📉 reduce — 归约（单向累积）

将数组"缩减"为一个值——求和、求平均、分组等。

const nums = [1, 2, 3, 4, 5];

const sum = nums.reduce((acc, cur) => {
  return acc + cur;
}, 0);
console.log(sum); // 15

📊 迭代方法速查表

方法 返回值 典型用途 链式调用
forEach undefined 遍历执行副作用（打印、DOM修改等） ❌ 不支持
map 新数组（等长） 数据转换 / 提取字段 ✅
filter 新数组（变短或等长） 筛选数据 ✅
reduce 任意值 数据聚合（求和/分组/统计） ✅（通常在末尾）
find 第一个匹配元素 按条件查找单个项 ✅
some boolean 是否有元素满足条件 ✅
every boolean 是否所有元素都满足条件 ✅
sort 排序后的原数组 排序（会修改原数组） ⚠️ 注意变异

⛓️ 链式调用——组合多个方法

map、filter、sort 等返回数组的方法可以链式串联，写出清晰的数据处理管道：

const items = [
  { name: '北京', pop: 2154 },
  { name: '上海', pop: 2487 },
  { name: '广州', pop: 1868 },
  { name: '拉萨', pop: 86 },
  { name: '深圳', pop: 1756 },
];

// 需求：找出人口 > 1000 的城市名，按人口降序排列
const result = items
  .filter(c => c.pop > 1000)         // 过滤
  .sort((a, b) => b.pop - a.pop)       // 排序
  .map(c => c.name);                    // 提取名字

console.log(result); // ["上海", "北京", "广州", "深圳"]

对象基础：创建与属性操作

OBJECTS

对象（Object）是键值对（key-value）的集合，花括号 {} 表示。每个键（属性名）映射到一个值（可以是任意类型）。

🏗️ 创建对象

// ① 字面量（最常用）
const coordinate = {
  longitude: 116.4,
  latitude:  39.9,
  height:    50,
};

// ② 构造函数
const point = new Object();
point.x = 10;
point.y = 20;

// ③ ES6+ 属性简写
const lon = 116.4, lat = 39.9;
const pos = { lon, lat }; // 等价于 { lon: lon, lat: lat }

📝 属性访问

const city = {
  name: '杭州',
  'full name': '杭州市',  // 含空格的属性名要加引号
  population: 1220,
};

// 点号语法（.）——属性名必须为合法标识符
city.name          // "杭州"
city.population    // 1220

// 方括号语法（[]）——可包含任意字符、动态键
city['name']        // "杭州"
city['full name']   // "杭州市"

const key = 'population';
city[key]             // 1220 — 动态键访问！

✏️ 添加 / 修改 / 删除属性

const entity = { id: 'marker_1' };

// 添加属性（直接赋值）
entity.name = '天安门';
entity.type = 'poi';
console.log(entity);
// { id: 'marker_1', name: '天安门', type: 'poi' }

// 修改属性（和添加语法一样）
entity.name = '故宫';

// 删除属性
delete entity.type;
console.log(entity.type); // undefined

// 检查属性是否存在
console.log('name' in entity);      // true
console.log(entity.hasOwnProperty('id')); // true

🧊 const 对象与可变性

const 声明的对象引用不可变，但内部属性可变。

const obj = { x: 1 };
obj.x = 2;        // ✅ 可以修改属性
obj.y = 3;        // ✅ 可以添加属性
// obj = {};          ❌ 不能重新绑定引用
ℹ️
为什么 const 对象可以修改？ const绑定的是引用（内存地址），不是内容。对象的属性存储在堆内存中，修改它们不改变引用本身。就像你不能把门牌号改了，但可以重新装修房间。

对象方法与遍历

METHODS & THIS

🏃 对象方法

对象的属性值可以是函数——这种属性就叫"方法"（method）。

const camera = {
  x: 0,
  y: 0,
  z: 100,

  // ES6 方法简写（推荐）
  moveUp(distance) {
    this.z += distance;
  },

  getPosition() {
    return `(${this.x}, ${this.y}, ${this.z})`;
  },
};

camera.moveUp(50);
console.log(camera.getPosition());
// (0, 0, 150)

🎯 this 关键字

在对象方法中，this 指向调用该方法的对象（即方法前的那个对象）。

const user = {
  name: '开发者',
  greet() {
    // this 指 user 对象
    console.log(`你好，我是${this.name}`);
  },
};

user.greet(); // "你好，我是开发者"

// ⚠️ 箭头函数的 this 不同！它继承外层作用域，不绑定调用者

🔁 遍历对象的键和值

const mapConfig = {
  center: [116.4, 39.9],
  zoom:   12,
  theme:  'dark',
};

// ① for...in — 遍历键（含原型链，需过滤）
for (const key in mapConfig) {
  if (mapConfig.hasOwnProperty(key)) {
    console.log(key, '→', mapConfig[key]);
  }
}

// ② Object.keys() — 获取所有自身属性的键（数组）
const keys = Object.keys(mapConfig);
console.log(keys); // ["center", "zoom", "theme"]

// ③ Object.values() — 获取所有自身属性的值（数组）
const values = Object.values(mapConfig);
console.log(values); // [[116.4,39.9], 12, "dark"]

// ④ Object.entries() — 键值对数组
const entries = Object.entries(mapConfig);
console.log(entries);
// [["center", [...]], ["zoom", 12], ["theme", "dark"]]

// ⑤ forEach + entries 优雅遍历
entries.forEach(([k, v]) => console.log(`${k}: ${v}`));

🛡️ Object.freeze() 与 Object.seal()

方法 可修改现有属性 可添加新属性 可删除属性
Object.freeze() ❌ ❌ ❌
Object.seal() ✅ ❌ ❌
无（默认） ✅ ✅ ✅

解构赋值（数组 & 对象）

ES6+

解构赋值（Destructuring）是 ES6 最重要的语法糖之一——让你从数组或对象中快速提取值并赋给变量。

📦 数组解构

// 基本解构——按位置匹配
const coords = [116.4, 39.9, 50];
const [lon, lat, alt] = coords;
console.log(lon, lat, alt); // 116.4 39.9 50

// 跳过元素
const [, latOnly] = coords;
console.log(latOnly); // 39.9

// 默认值
const [a = 1, b = 2] = [10];
console.log(a, b); // 10 2（b 用了默认值）

// 交换变量（无需临时变量！）
let x = 1, y = 2;
[x, y] = [y, x];
console.log(x, y); // 2 1

// 剩余元素（rest）
const [first, ...rest] = [1, 2, 3, 4];
console.log(first); // 1
console.log(rest); // [2, 3, 4]

📋 对象解构

// 基本解构——按属性名匹配（非顺序！）
const city = { name: '北京', lat: 39.9, lon: 116.4 };
const { name, lat, lon } = city;
console.log(name); // "北京"

// 重命名
const { name: cityName } = city;
console.log(cityName); // "北京"

// 默认值
const { zoom = 12 } = city;
console.log(zoom); // 12（city 中没有 zoom）

// 剩余属性（rest）
const { name: n, ...coords } = city;
console.log(coords); // { lat: 39.9, lon: 116.4 }

// 函数参数解构（非常常用！）
const printCity = ({ name, lat, lon }) => {
  console.log(`${name}位于 (${lat}°, ${lon}°)`);
};
printCity(city); // 北京位于 (39.9°, 116.4°)
💡
关键区别：数组解构按位置匹配，对象解构按属性名匹配。这是两者最大的不同。

🌍 CesiumJS 关联场景

CESIUMJS

数组和对象是 CesiumJS 开发中最核心的数据结构。来看看将来的实际应用场景：

🗺️ 场景 1：管理实体集合（Entity Collection）

在 CesiumJS 中，你可以在地球上放置各种标记点（marker）、线条、多边形。这些实体通常存储为对象数组：

// CesiumJS 中的实体数据——对象数组
const landmarks = [
  { id: 'bj',    name: '天安门',   lon: 116.3975, lat: 39.9087 },
  { id: 'sh',    name: '东方明珠', lon: 121.4998, lat: 31.2397 },
  { id: 'gz',    name: '广州塔',   lon: 113.3245, lat: 23.1065 },
  { id: 'cd',    name: '宽窄巷子', lon: 104.0535, lat: 30.6698 },
];

// 用 map 将地标数据转换为 Cesium 实体配置
const entities = landmarks.map(place => ({
  id: place.id,
  name: place.name,
  position: Cesium.Cartesian3.fromDegrees(place.lon, place.lat),
  point: { pixelSize: 10, color: Cesium.Color.RED },
}));

// 用 filter 筛选特定区域的地标
const easternCities = landmarks.filter(p => p.lon > 110);
console.log(easternCities.length); // 3（北京、上海、广州）

🎨 场景 2：样式配置对象

CesiumJS 大量使用配置对象来定义实体的外观：

const polygonStyle = {

// 对象方法：设置相机飞行动画

};

📊 场景 3：轨迹数据处理

利用数组方法处理 GPS 轨迹数据——这是 CesiumJS 的典型任务：

// GPS 轨迹数组（每个点是一个对象）
const gpsTrack = [
  { time: '08:00', lon: 116.4, lat: 39.9, alt: 0 },
  { time: '08:05', lon: 116.5, lat: 40.0, alt: 200 },
  { time: '08:10', lon: 116.6, lat: 40.1, alt: 500 },
  { time: '08:15', lon: 116.7, lat: 40.2, alt: 800 },
  { time: '08:20', lon: 116.8, lat: 40.3, alt: 1200 },
];

// map：提取所有经度
const lons = gpsTrack.map(p => p.lon);
console.log(lons); // [116.4, 116.5, 116.6, 116.7, 116.8]

// reduce：计算平均海拔
const avgAlt = gpsTrack.reduce((sum, p) => sum + p.alt, 0) / gpsTrack.length;
console.log(avgAlt); // 540

// filter：找出高于 500m 的航点
const highAlt = gpsTrack.filter(p => p.alt > 500);
console.log(highAlt.length); // 3

// find：找到海拔超过 1000 的第一个点
const firstHigh = gpsTrack.find(p => p.alt > 1000);
console.log(firstHigh); // { time: '08:20', lon: 116.8, lat: 40.3, alt: 1200 }

🧩 场景 4：图层管理

CesiumJS 中的图层管理本质就是对象数组的增删操作：

// 模拟图层管理系统
const layers = [
  { id: 'base',     name: '底图',   visible: true,  opacity: 1.0 },
  { id: 'building', name: '建筑',   visible: true,  opacity: 0.8 },
  { id: 'marker',   name: '标记点', visible: false, opacity: 1.0 },
];

// 添加新图层
layers.push({ id: 'weather', name: '天气', visible: true, opacity: 0.5 });

// 删除指定图层
const idx = layers.findIndex(l => l.id === 'marker');
if (idx !== -1) layers.splice(idx, 1);

// 获取所有可见图层
const visibleLayers = layers.filter(l => l.visible);
console.log(visibleLayers.map(l => l.name).join(', '));
// "底图, 建筑, 天气"

// 切换图层可见性
const toggleLayer = (id) => {
  const layer = layers.find(l => l.id === id);
  if (layer) layer.visible = !layer.visible;
};

📋 今日知识点总结

  • ✅ 数组是有序元素集合，用 [] 创建；索引从 0 开始；length 表示元素个数
  • ✅ 变异方法（push/pop/shift/unshift/splice/sort/reverse）会修改原数组
  • ✅ 非变异方法（map/filter/reduce/slice/concat）返回新数组，不修改原数组
  • ✅ map 用于转换、filter 用于筛选、reduce 用于归约，三者可以链式调用
  • ✅ 对象是键值对集合，用 {} 创建；用 . 或 [] 访问属性
  • ✅ const 对象可以修改内部属性，但不能重新绑定引用
  • ✅ 对象方法中的 this 指向调用该方法的对象
  • ✅ 解构赋值是 ES6 的强大特性：数组按位置匹配，对象按属性名匹配
  • ✅ Object.keys() / values() / entries() 是遍历对象的推荐方式
