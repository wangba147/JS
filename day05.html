DOM 简介：文档对象模型

DOM

DOM（Document Object Model）是浏览器将 HTML 文档解析后形成的一棵树形结构。JavaScript 通过 DOM API 来读取和修改网页内容。

🌲 DOM 树结构

每个 HTML 标签都是一个元素节点（Element Node），标签内的文本是文本节点（Text Node），标签上的属性是属性节点（Attribute Node）。

<!-- HTML 源码 -->
<div id="map">
  <h1>CesiumJS 地球</h1>
  <p class="desc">三维地理可视化</p>
</div>

/* 对应的 DOM 树 */
        document
           |
         html
        /    \
     head    body
              |
             div#map
            /       \
         h1         p.desc
          |          |
    "CesiumJS 地球"  "三维地理可视化"
*/
ℹ️
DOM 是浏览器提供的 API，不是 JavaScript 语言本身的一部分。但在浏览器环境中，JS 通过全局对象 document 和 window 来访问 DOM。

DOM 选择器：找到元素

SELECTORS

操作 DOM 的第一步是选中目标元素。JavaScript 提供了多种选择器方法：

方法 选择依据 返回结果 推荐指数
getElementById(id) id 属性 单个元素 / null ✅ 快速
getElementsByClassName(cls) class 属性 HTMLCollection（实时集合） ⚠️ 旧式
getElementsByTagName(tag) 标签名 HTMLCollection（实时集合） ⚠️ 旧式
querySelector(selector) CSS 选择器 第一个匹配的元素 / null ✅ 推荐
querySelectorAll(selector) CSS 选择器 NodeList（静态集合，可 forEach） ✅ 推荐

🔍 选择器用法示例

// ① getElementById — 通过 id 查找（最快速）
const mapContainer = document.getElementById("cesiumContainer");

// ② querySelector — CSS 选择器，返回第一个匹配
const title = document.querySelector("h1");           // 第一个 h1
const btn   = document.querySelector(".btn-primary"); // 第一个 class="btn-primary"
const nav   = document.querySelector("nav ul li");    // 嵌套选择

// ③ querySelectorAll — 返回所有匹配（NodeList）
const items = document.querySelectorAll(".layer-item");
items.forEach((item, index) => {
  console.log(`图层 ${index}: ${item.textContent}`);
});

// ④ 在指定元素内查找（缩小范围）
const sidebar = document.querySelector("#sidebar");
const links   = sidebar.querySelectorAll("a"); // 只在 sidebar 内找 a 标签

// ⑤ 组合选择器
const activeItem = document.querySelector("li.active[data-type='satellite']");
💡
选择器性能：getElementById 最快，因为它直接通过哈希表查找。但对于大多数场景，querySelector 和 querySelectorAll 的性能差异可以忽略不计，且语法更灵活。建议统一使用 querySelector 系列，保持代码一致性。

读取与修改内容

CONTENT

选中元素后，可以读取或修改它的文本内容和HTML 结构。

📝 textContent

纯文本内容，不包含 HTML 标签，不解析 HTML，安全性高。

const box = document.querySelector("#info");

// 读取
console.log(box.textContent);

// 修改（文本原样写入）
box.textContent = "<b>警告</b>：前方高能";
// 页面显示：<b>警告</b>：前方高能（不会解析为加粗）

🌐 innerHTML

元素的完整 HTML 内容，会解析 HTML 标签，功能强大但要小心 XSS。

const box = document.querySelector("#info");

// 读取
console.log(box.innerHTML);

// 修改（HTML 会被解析渲染）
box.innerHTML = "<b>警告</b>：前方高能";
// 页面显示：警告：前方高能（加粗效果生效）

⚠️ innerHTML vs textContent vs innerText 对比

属性 解析 HTML 读取隐藏元素 XSS 风险
textContent ❌ 不解析 ✅ 读取 ✅ 安全
innerHTML ✅ 解析 ✅ 读取 ⚠️ 有 XSS 风险
innerText ❌ 不解析 ❌ 忽略隐藏元素 ✅ 安全
⚠️
XSS 警告：永远不要将用户输入直接赋值给 innerHTML！恶意用户可以注入 <script> 标签执行攻击代码。对于用户输入，始终使用 textContent。

🌍 CesiumJS 场景：动态更新坐标信息

// 假设页面上有一个信息面板显示当前相机位置
const lonDisplay = document.querySelector("#longitude");
const latDisplay = document.querySelector("#latitude");
const altDisplay = document.querySelector("#altitude");

function updatePosition(lon, lat, alt) {
  lonDisplay.textContent = lon.toFixed(4) + "°";
  latDisplay.textContent = lat.toFixed(4) + "°";
  altDisplay.textContent = alt.toFixed(0) + " m";
}

updatePosition(116.3974, 39.9093, 50000);
// 信息面板实时更新为：116.3974°, 39.9093°, 50000 m

属性与样式操作

ATTRIBUTES & STYLES

🏷️ HTML 属性操作

const img = document.querySelector("#map-thumb");

// 读取属性
console.log(img.getAttribute("src"));

// 设置属性
img.setAttribute("src", "beijing-satellite.png");
img.setAttribute("alt", "北京卫星影像");

// 检查是否存在某属性
console.log(img.hasAttribute("data-layer")); // true / false

// 删除属性
img.removeAttribute("title");

// 便捷属性访问（标准属性可直接用点号）
img.src = "new-image.png";
img.id = "updated-thumb";

🎨 样式操作

const panel = document.querySelector(".info-panel");

// 直接修改 style（内联样式，优先级最高）
panel.style.backgroundColor = "#1a1d27";
panel.style.width = "320px";
panel.style.display = "none";    // 隐藏
panel.style.display = "block";   // 显示

// 注意：CSS 属性名转 camelCase
// background-color → backgroundColor
// border-radius    → borderRadius

// 一次性设置多个样式
panel.style.cssText = `
  background: #1a1d27;
  border: 1px solid #2e3350;
  border-radius: 8px;
  padding: 16px;
`;

🎯 classList：优雅的类名操作

通过 classList 可以方便地添加、删除、切换 CSS 类，比直接操作 className 字符串更安全。

const btn = document.querySelector("#layer-toggle");

// 添加类名
btn.classList.add("active");

// 删除类名
btn.classList.remove("disabled");

// 切换类名（有则删，无则加）— 最常用的技巧
btn.classList.toggle("active");

// 检查是否包含某类名
if (btn.classList.contains("active")) {
  console.log("图层已激活");
}

// 替换类名
btn.classList.replace("btn-default", "btn-primary");
💡
最佳实践：尽量通过添加/删除 CSS 类来控制样式，而不是直接修改 element.style。这样样式集中在 CSS 文件中，便于维护和主题切换。直接改 style 只适用于需要动态计算的样式值（如鼠标跟随位置）。

DOM 遍历：在树中移动

TRAVERSAL

有时需要通过一个已知的元素，在 DOM 树中导航到相关的其他元素。

⬆️ 向上遍历

const item = document.querySelector(".layer-item");

item.parentNode        // 直接父节点（可能是文本节点）
item.parentElement     // 父元素节点（推荐）
item.closest(".sidebar") // 向上查找最近的匹配祖先（含自身）

⬇️ 向下遍历

const container = document.querySelector("#layer-list");

container.children        // 所有子元素（HTMLCollection）
container.firstElementChild  // 第一个子元素
container.lastElementChild   // 最后一个子元素
container.querySelector(".active") // 在子元素中搜索

➡️ 横向遍历

const item = document.querySelector(".layer-item");

item.nextElementSibling     // 下一个兄弟元素
item.previousElementSibling // 上一个兄弟元素

// 遍历所有兄弟元素
let sibling = item.nextElementSibling;
while (sibling) {
  console.log(sibling.textContent);
  sibling = sibling.nextElementSibling;
}

🧭 常用遍历技巧

// Array.from 将类数组转为真数组，使用数组方法
const children = Array.from(container.children);
const filtered = children.filter(el => el.classList.contains("visible"));

// [...] 展开运算符同样有效
const siblings = [...item.parentElement.children]
  .filter(el => el !== item);
ℹ️
注意区分 childNodes 和 children：childNodes 包含文本节点、注释节点等所有节点；children 只包含元素节点。在大多数实际开发中，使用 children 更安全。

创建、插入与删除元素

CREATE & MODIFY

DOM 的强大之处在于可以动态创建、移动和删除元素，实现完全交互式的网页。

✨ 创建元素

// ① createElement — 创建新元素
const div = document.createElement("div");
div.id = "tooltip";
div.className = "map-tooltip";
div.textContent = "点击选择位置";

// ② createTextNode — 创建文本节点（较少用）
const text = document.createTextNode("纯文本内容");
div.appendChild(text);

// ③ cloneNode — 克隆现有元素
const template = document.querySelector(".layer-item");
const clone = template.cloneNode(true); // true = 深克隆（包含子元素）
clone.textContent = "新图层";

📌 插入元素到 DOM

const list = document.querySelector("#layer-list");
const newItem = document.createElement("li");
newItem.textContent = "卫星影像层";

// ① append — 在末尾添加（现代推荐，可同时添加多个）
list.append(newItem);

// ② prepend — 在开头添加
list.prepend(newItem);

// ③ before / after — 在参考元素之前/之后插入
const ref = document.querySelector(".layer-item.active");
ref.after(newItem);  // 在 ref 后面插入

// ④ insertAdjacentElement — 精确控制位置
list.insertAdjacentElement("beforeend", newItem);
// 位置参数: "beforebegin" | "afterbegin" | "beforeend" | "afterend"

// ⑤ insertAdjacentHTML — 直接插入 HTML 字符串
list.insertAdjacentHTML("beforeend", `<li class="layer-item">地形层</li>`);

🗑️ 删除与替换元素

const item = document.querySelector(".layer-item.old");

// ① remove — 元素自我删除（现代推荐）
item.remove();

// ② 通过父元素删除子元素（兼容旧浏览器）
item.parentElement.removeChild(item);

// ③ replaceWith — 替换元素
const newItem = document.createElement("li");
newItem.textContent = "替换后的图层";
item.replaceWith(newItem);

// ④ replaceChild — 旧版替换方式
item.parentElement.replaceChild(newItem, item);
💡
性能提示：频繁操作 DOM 会引发重排（Reflow）和重绘（Repaint），影响性能。如果需要批量插入多个元素，可以先用 document.createDocumentFragment() 在内存中组装，最后一次性插入到页面中。

⚡ DocumentFragment 批量插入

// 高性能批量插入示例
const list = document.querySelector("#layer-list");
const fragment = document.createDocumentFragment();

const layers = ["卫星", "地形", "街道", "3D建筑"];
layers.forEach((name) => {
  const li = document.createElement("li");
  li.className = "layer-item";
  li.textContent = name;
  fragment.appendChild(li);
});

// 只触发一次 DOM 更新！
list.append(fragment);

CesiumJS 关联场景

CESIUMJS

在 CesiumJS 三维地球开发中，DOM 操作无处不在：自定义信息面板、图层控制 UI、加载指示器、坐标拾取显示等，全部依赖 DOM API。

🌍 场景 1：自定义图层控制面板

// CesiumJS 应用中的典型 DOM 操作：动态构建图层列表
const layerPanel = document.querySelector("#layer-panel");

const imageryLayers = [
  { id: "satellite", name: "卫星影像", active: true },
  { id: "terrain",   name: "地形晕渲", active: false },
  { id: "roads",     name: "道路网",   active: false },
];

function renderLayerList() {
  layerPanel.innerHTML = ""; // 清空

  imageryLayers.forEach((layer) => {
    const btn = document.createElement("button");
    btn.className = "layer-btn";
    btn.dataset.layerId = layer.id;

    if (layer.active) {
      btn.classList.add("active");
    }

    btn.textContent = layer.name;
    layerPanel.appendChild(btn);
  });
}

renderLayerList();

🌍 场景 2：CesiumJS 信息框（InfoBox）动态内容

// CesiumJS 的 InfoBox 本质上是 DOM 元素
// 我们可以通过 DOM API 自定义其内容和样式

// 获取 Cesium Viewer 的 infoBox 容器
const infoBox = document.querySelector(".cesium-infoBox");

// 自定义信息框标题样式
const title = infoBox.querySelector(".cesium-infoBox-title");
title.style.backgroundColor = "#1a3a5f";
title.style.color = "#61dafb";

// 动态更新信息框内容（响应实体选择事件）
function updateInfoBox(entity) {
  const content = infoBox.querySelector(".cesium-infoBox-description");
  content.innerHTML = `
    <h3>${entity.name}</h3>
    <p>经度: ${entity.position.longitude.toFixed(4)}°</p>
    <p>纬度: ${entity.position.latitude.toFixed(4)}°</p>
    <p>高度: ${entity.position.height.toFixed(0)} m</p>
  `;
}

🌍 场景 3：加载状态指示器

// CesiumJS 加载地形或影像时显示进度
const loader = document.createElement("div");
loader.id = "cesium-loader";
loader.className = "loading-overlay";
loader.innerHTML = `<div class="spinner"></div><p>加载地形数据中...</p>`;
document.body.appendChild(loader);

// 地形加载完成后隐藏
function hideLoader() {
  loader.style.opacity = "0";
  loader.style.transition = "opacity 0.5s";
  setTimeout(() => loader.remove(), 500);
}
ℹ️
CesiumJS 与 DOM 的关系：CesiumJS 核心渲染在 <canvas> 上进行，但所有周边 UI（工具栏、时间轴、图层列表、属性面板、搜索框）都是标准 DOM 元素。掌握 DOM 操作，才能构建完整的 CesiumJS 应用界面。

📋 今日知识点总结

  • ✅ DOM 是浏览器将 HTML 解析成的树形结构，通过 document 对象访问
  • ✅ querySelector / querySelectorAll 是最灵活、最推荐的选择器方法
  • ✅ textContent 操作纯文本（安全），innerHTML 操作 HTML（注意 XSS）
  • ✅ classList.add/remove/toggle/contains 是操作 CSS 类的最佳方式
  • ✅ createElement + append/prepend/before/after 实现动态元素创建和插入
  • ✅ DocumentFragment 可以批量插入元素而只触发一次重排，性能更优
  • ✅ CesiumJS 的 InfoBox、工具栏、图层控制面板等 UI 全部依赖 DOM 操作构建
