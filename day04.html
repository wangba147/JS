if / else if / else 条件判断

CONDITIONAL

条件判断是控制流的基石。if 语句根据条件的真假（true/false）决定程序的执行路径。

🔀 基本语法结构

// ① 单分支
if (condition) {
  // 条件为 true 时执行
}

// ② 双分支
if (condition) {
  // 条件为 true 时执行
} else {
  // 条件为 false 时执行
}

// ③ 多分支
if (condition1) {
  // condition1 为 true
} else if (condition2) {
  // condition1 为 false，condition2 为 true
} else if (condition3) {
  // 前面的都为 false，condition3 为 true
} else {
  // 以上条件全部不满足
}

🌍 实战示例：根据缩放等级显示不同信息

const zoomLevel = 8;

if (zoomLevel >= 15) {
  console.log('街道级别 — 可以看到建筑细节');
} else if (zoomLevel >= 10) {
  console.log('城市级别 — 可以看到城区分布');
} else if (zoomLevel >= 5) {
  console.log('省级级别 — 可以看到城市分布');
} else {
  console.log('国家级别 — 鸟瞰全局');
}
// 输出："省级级别 — 可以看到城市分布"

⚡ 真值与假值（Truthy & Falsy）

if 的条件并不要求一定是布尔值——JavaScript 会自动将值转换为布尔。以下值是 falsy（视为 false），其余全是 truthy：

值 类型 布尔转换
false Boolean false
0 Number false
-0 Number false
0n BigInt false
"" String（空字符串） false
null Null false
undefined Undefined false
NaN Number false
"0" / [] / {} / -1 其他 true ✅ 
// 实际应用：检查变量是否有有效值
let cityName = '';

if (cityName) {
  console.log('城市名：', cityName);
} else {
  console.log('城市名为空，请提供名称');
}
// 输出："城市名为空，请提供名称"（空字符串是 falsy）

// ⚠️ 注意：0 是 falsy！
const altitude = 0;
if (altitude) {
  console.log('海拔不为零');  // 不会执行！
}
if (altitude >= 0) {
  console.log('海平面或以上');  // ✅ 更严谨的写法
}
⚠️
常见陷阱："0"（字符串）、[]（空数组）、{}（空对象）都是 truthy！判断数组是否为空应使用 arr.length > 0，判断对象是否为空应使用 Object.keys(obj).length > 0。

三元运算符与短路求值

SHORTCUT

❓ 三元运算符 ? :

条件表达式的简写形式——适合简单的二选一：

// 语法：条件 ? 真值 : 假值
const altitude = 150;
const status = altitude > 100
  ? '高空飞行'
  : '低空巡航';

console.log(status); // "高空飞行"

// 嵌套三元（不推荐超过 2 层）
const level = altitude > 1000 ? '巡航'
  : altitude > 100 ? '飞行'
  : '地面';

🔗 短路求值 && / ||

逻辑运算符的特性可以用于条件赋值：

// || 返回第一个 truthy 值（OR 短路）
const name = '' || '未命名标注';
console.log(name); // "未命名标注"

// && 返回第一个 falsy 值（AND 短路）
const config = null;
config && console.log(config.zoom);
// config 为 falsy，右侧不执行——安全访问！

// ?? 空值合并（ES2020）——只对 null/undefined 生效
const count = 0 ?? 10;
console.log(count); // 0（0 不是 null/undefined）

const val = null ?? '默认值';
console.log(val); // "默认值"

📊 三种方式对比

方式 语法 适用场景 返回值
if...else 语句 多行逻辑、副作用操作 无返回值
三元 ? : 表达式 简单的二选一赋值 有返回值
|| / ?? 表达式 提供默认值、守卫 有返回值

switch 语句

SWITCH

switch 语句将一个表达式与多个 case 值匹配，适合固定值的多分支判断。

📋 基本语法

switch (expression) {
  case value1:
    // expression === value1 时执行
    break;       // ⚠️ 别忘了 break！
  case value2:
    // expression === value2 时执行
    break;
  default:
    // 所有 case 都不匹配时执行
    break;
}

🗺️ 实战：根据地图类型选择渲染方式

const mapType = 'satellite';

switch (mapType) {
  case 'streets':
    console.log('加载街道底图…');
    break;
  case 'satellite':
    console.log('加载卫星影像…');
    break;
  case 'terrain':
    console.log('加载地形图…');
    break;
  case 'hybrid':
    console.log('加载混合底图…');
    break;
  default:
    console.log('未知底图类型，使用默认');
    break;
}
// 输出："加载卫星影像…"

🔗 case 合并（利用穿透）

// 多个 case 执行同一段代码
const direction = 'N';

switch (direction) {
  case 'N':
  case 'S':
    console.log('南北方向');
    break;
  case 'E':
  case 'W':
    console.log('东西方向');
    break;
}
// 输出："南北方向"

⚠️ 忘记 break 的后果

const x = 1;
switch (x) {
  case 1:
    console.log('一');
    // 忘记 break！
  case 2:
    console.log('二');
    // 忘记 break！
  case 3:
    console.log('三');
    break;
}
// 输出：一 / 二 / 三
// 这就是"穿透"——非预期的行为！
ℹ️
switch vs if...else if：switch 适合固定值匹配（字符串、数字），if...else if 适合范围判断和复杂条件。switch 使用严格相等 === 比较。
4

for 循环

LOOP

for 循环是最经典的循环结构——当你明确知道循环次数时使用。

🔁 经典 for 循环

// 语法：for (初始化; 条件; 更新) { 循环体 }
for (let i = 0; i < 5; i++) {
  console.log(`第 ${i} 次循环`);
}
// 第 0 次循环
// 第 1 次循环
// 第 2 次循环
// 第 3 次循环
// 第 4 次循环

🌍 实战：批量创建坐标点

// 生成一条沿经线均匀分布的 10 个点
const waypoints = [];
const startLon = 116.0;
const step = 0.5;  // 每隔 0.5 度

for (let i = 0; i < 10; i++) {
  waypoints.push({
    id: `wp_${i}`,
    lon: startLon + i * step,
    lat: 39.9,
  });
}

console.log(waypoints);
// [{id:"wp_0", lon:116.0, lat:39.9}, {id:"wp_1", lon:116.5, lat:39.9}, ...]

🔄 嵌套循环——二维遍历

// 生成 3×3 网格坐标点
const grid = [];
const baseLon = 116.0;
const baseLat = 39.0;

for (let row = 0; row < 3; row++) {
  for (let col = 0; col < 3; col++) {
    grid.push({
      row, col,
      lon: baseLon + col * 0.5,
      lat: baseLat + row * 0.5,
    });
  }
}
console.log(`共 ${grid.length} 个网格点`); // 共 9 个网格点
💡
最佳实践：循环变量务必使用 let（不是 var），确保每次迭代有独立作用域。避免在循环中创建闭包陷阱。

while / do...while 循环

CONDITION-LOOP

当你不确定循环次数，只知道循环的终止条件时，使用 while 或 do...while。

🔄 while — 先判断后执行

// 语法：while (条件) { 循环体 }
// 条件为 true 才进入循环

let distance = 1000;
let steps = 0;

// 每步走 200 米，走完为止
while (distance > 0) {
  distance -= 200;
  steps++;
  console.log(`走了 ${steps} 步，剩余 ${distance} 米`);
}
console.log('到达！');

🔁 do...while — 先执行后判断

// 至少执行一次，然后判断
let retryCount = 0;
const maxRetries = 3;

do {
  retryCount++;
  console.log(`尝试加载地图数据 (第${retryCount}次)`);
  // 模拟加载成功/失败
  const success = retryCount >= 2;
  if (success) {
    console.log('加载成功！');
    break;
  }
} while (retryCount < maxRetries);

📊 for vs while 选择指南

特征 for while do...while
循环次数 明确已知 未知/按条件 未知/按条件
首次判断 执行前判断 执行前判断 执行后判断（至少执行1次）
迭代变量 内置在 for 头部 需外部声明 需外部声明
典型场景 遍历数组、计数 等待条件满足 用户输入、重试

for...of / for...in 迭代

ITERATION

ES6 引入了更优雅的迭代语法：for...of 遍历值，for...in 遍历键。

📦 for...of — 遍历值

适用于可迭代对象：数组、字符串、Map、Set 等。

// 遍历数组
const cities = ['北京', '上海', '广州'];

for (const city of cities) {
  console.log(city);
}
// 北京 / 上海 / 广州

// 遍历字符串
for (const char of 'Cesium') {
  console.log(char);
}
// C / e / s / i / u / m

// entries() 同时获取索引和值
for (const [i, city] of cities.entries()) {
  console.log(`${i}: ${city}`);
}
// 0: 北京 / 1: 上海 / 2: 广州

🔑 for...in — 遍历键

适用于对象，遍历所有可枚举属性（含原型链）。

const camera = {
  lon: 116.4,
  lat: 39.9,
  height: 500,
};

// 遍历对象的键
for (const key in camera) {
  console.log(`${key} = ${camera[key]}`);
}
// lon = 116.4
// lat = 39.9
// height = 500

// ⚠️ for...in 不推荐用于数组！
const arr = ['a', 'b'];
for (const k in arr) {
  console.log(typeof k); // "string" — 索引变成了字符串！
}

📊 四种遍历方式对比

方式 适用对象 获得内容 推荐度
for...of 数组、字符串、Map、Set 值 ⭐⭐⭐ 遍历数组首选
for...in 对象 键（字符串） ⭐⭐ 对象可用，数组慎用
forEach 数组 值 + 索引 ⭐⭐⭐ 但不能 break
经典 for 所有 索引 ⭐⭐⭐ 最灵活
⚠️
for...in 的陷阱：它遍历原型链上的可枚举属性，不仅遍历自身属性。建议用 hasOwnProperty() 过滤，或直接使用 Object.keys() + for...of。

break / continue 与标签

FLOW CONTROL

🛑 break — 完全跳出循环

// 找到第一个海拔超过 3000m 的点就停止
const points = [
  { alt: 500 },
  { alt: 2000 },
  { alt: 3500 },
  { alt: 4000 },
];

for (const p of points) {
  if (p.alt > 3000) {
    console.log(`找到：${p.alt}m`);
    break;  // 跳出整个循环
  }
}
// 输出："找到：3500m"

⏭️ continue — 跳过本次迭代

// 只处理海拔大于 1000m 的点
const data = [100, 500, 1500, 800, 3000];

for (const alt of data) {
  if (alt < 1000) {
    continue; // 跳过，进入下一轮
  }
  console.log(`高海拔：${alt}m`);
}
// 高海拔：1500m
// 高海拔：3000m

🏷️ 标签语句——跳出多层循环

// 在二维网格中查找第一个特定条件的点
const grid = [
  [1, 2, 3],
  [4, 0, 6],   // 0 是我们要找的目标
  [7, 8, 9],
];

outerLoop:  // 标签名可以自定义
for (let r = 0; r < grid.length; r++) {
  for (let c = 0; c < grid[r].length; c++) {
    if (grid[r][c] === 0) {
      console.log(`找到 0 在行${r}列${c}`);
      break outerLoop;  // 跳出外层循环！
    }
  }
}
// 输出："找到 0 在行1列1"
ℹ️
替代方案：标签语句在实际项目中较少使用。更推荐使用函数 return 或数组的 some()/find() 方法来替代嵌套循环 + break 的场景。

🌍 CesiumJS 关联场景

CESIUMJS

控制流是 CesiumJS 开发中最常用的编程结构。以下是真实开发中高频出现的场景：

🗺️ 场景 1：根据实体类型选择渲染样式

在 CesiumJS 中，不同类型的地理实体需要不同的渲染配置：

// 根据标注类型选择不同的点样式
function getMarkerStyle(type) {
  switch (type) {
    case 'capital':
      return {
        pixelSize: 14,
        color: 'RED',
        outlineColor: 'WHITE',
        outlineWidth: 2,
      };
    case 'city':
      return {
        pixelSize: 10,
        color: 'ORANGE',
        outlineColor: 'WHITE',
        outlineWidth: 1,
      };
    case 'town':
      return {
        pixelSize: 6,
        color: 'YELLOW',
      };
    default:
      return {
        pixelSize: 4,
        color: 'CYAN',
      };
  }
}

const style = getMarkerStyle('capital');
console.log(style);
// { pixelSize: 14, color: 'RED', ... }

✈️ 场景 2：航线距离计算（循环 + 条件）

计算航线途经点的总距离，并筛选出需要加油的中转站：

// 航线途经点
const route = [
  { name: '北京', lon: 116.4, lat: 39.9 },
  { name: '西安', lon: 108.9, lat: 34.3 },
  { name: '成都', lon: 104.1, lat: 30.6 },
  { name: '昆明', lon: 102.7, lat: 25.0 },
];

// 简化距离计算（实际项目用 Cesium.Cartesian3.distance）
function simpleDistance(a, b) {
  const dlon = b.lon - a.lon;
  const dlat = b.lat - a.lat;
  return Math.sqrt(dlon * dlon + dlat * dlat) * 111; // 粗略换算为 km
}

// 计算总距离 + 分段距离
let totalDist = 0;
const segments = [];

for (let i = 0; i < route.length - 1; i++) {
  const dist = simpleDistance(route[i], route[i + 1]);
  totalDist += dist;
  segments.push({
    from: route[i].name,
    to: route[i + 1].name,
    distance: dist,
  });
}

// 筛选超过 800km 的长距离航段（需要中途加油）
const refuelNeeded = segments.filter(s => s.distance > 800);

console.log(`总距离：${totalDist.toFixed(0)} km`);
console.log('需要加油的航段：', refuelNeeded.map(s => `${s.from}→${s.to}`));

📡 场景 3：数据加载重试机制（while + 条件）

在 CesiumJS 中加载远程 GeoJSON 数据时，网络可能不稳定，需要重试：

// 模拟数据加载重试
function loadGeoJSON(url) {
  // 模拟：第 2 次才成功
  const attempts = loadGeoJSON.attempts || 0;
  loadGeoJSON.attempts = attempts + 1;
  return attempts >= 2;
}

const maxRetries = 3;
let retries = 0;
let loaded = false;

while (!loaded && retries < maxRetries) {
  retries++;
  console.log(`尝试加载 (第${retries}次)...`);

  loaded = loadGeoJSON('https://example.com/data.geojson');

  if (!loaded) {
    console.log('加载失败，准备重试…');
  }
}

if (loaded) {
  console.log('✅ GeoJSON 数据加载成功！');
} else {
  console.log('❌ 加载失败，请检查网络');
}

🏗️ 场景 4：批量构建 CesiumJS 实体

用 for 循环批量创建 CesiumJS 地球上的标注点：

// 中国主要城市数据
const majorCities = [
  { name: '北京', lon: 116.41, lat: 39.90, pop: 2154 },
  { name: '上海', lon: 121.47, lat: 31.23, pop: 2487 },
  { name: '广州', lon: 113.26, lat: 23.13, pop: 1868 },
  { name: '深圳', lon: 114.07, lat: 22.55, pop: 1756 },
  { name: '成都', lon: 104.07, lat: 30.57, pop: 2094 },
];

// 用 for...of 批量构建实体配置
const cesiumEntities = [];

for (const city of majorCities) {
  // 根据人口决定标记大小
  const size = city.pop > 2000 ? 14 : 8;

  // 根据人口规模选择颜色
  const color = city.pop > 2000 ? 'RED' : 'ORANGE';

  cesiumEntities.push({
    name: city.name,
    position: [city.lon, city.lat],
    point: { pixelSize: size, color: color },
  });
}

console.log(`共创建 ${cesiumEntities.length} 个实体`);
console.log(cesiumEntities);

题 2：循环 + 条件——筛选航线

给定一组城市坐标，用 for...of 遍历并找出所有经度大于 110° 且纬度小于 35° 的城市名称。

const cities = [
  { name: '北京', lon: 116.4, lat: 39.9 },
  { name: '上海', lon: 121.5, lat: 31.2 },
  { name: '广州', lon: 113.3, lat: 23.1 },
  { name: '成都', lon: 104.1, lat: 30.6 },
  { name: '深圳', lon: 114.1, lat: 22.5 },
];

// 你的代码：
const result = [];
for (const city of cities) {
  // 条件：lon > 110 且 lat < 35
  if (city.lon > 110 && city.lat < 35) {
    result.push(city.name);
  }
}
console.log(result); // 期望：["上海", "广州", "深圳"]
题 3：switch + 循环——地图图例生成器（CesiumJS 关联）

写一个函数 getLegendColor(level)，用 switch 返回热力图等级对应的颜色。然后用 for 循环生成 1~5 级的图例数据数组。

function getLegendColor(level) {
  switch (level) {
    case 1: return '#00ff00'; // 绿色-低
    case 2: return '#ffff00'; // 黄色-中低
    case 3: return '#ff8800'; // 橙色-中
    case 4: return '#ff0000'; // 红色-高
    case 5: return '#8b0000'; // 深红-极高
    default: return '#cccccc';
  }
}

// 生成图例数组
const legend = [];
for (let i = 1; i <= 5; i++) {
  legend.push({ level: i, color: getLegendColor(i) });
}
console.log(legend);

  • ✅ switch 适合固定值的多分支匹配，使用严格相等 ===，别忘了 break
  • ✅ for 循环适合已知次数的迭代，while 适合条件驱动的循环
  • ✅ for...of 遍历值（数组首选），for...in 遍历键（对象可用，数组慎用）
  • ✅ break 完全跳出循环，continue 跳过当前迭代
  • ✅ 标签语句可以跳出多层循环，但更推荐用函数 return 或数组方法替代
