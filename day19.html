为什么需要空间量算

动机

🤔 从"展示数据"到"分析数据"

前几节课，你已经能在地球上加载城市点、绘制多边形、叠加影像、响应点击事件。但真实 GIS 应用常常会问：

  • 这两点之间的地面距离是多少公里？
  • 这个地块的面积有多少平方米？
  • 飞机当前位置到跑道的方位角是多少？
  • 用户画的多边形是否包含某个目标点？
  • 建筑物顶部到地面的高度如何计算？

这些问题的共同点是：都需要在椭球三维空间中进行精确计算，而不是在平面屏幕上简单量像素。

⚡ 常见误区

误区问题正确做法
用平面距离公式算经纬度 经度 1° 在不同纬度代表的距离不同 使用椭球面距离（Haversine / Geodesic）
忽略高度分量 飞机与山峰的直线距离会出错 使用三维笛卡尔距离（Cartesian3.distance）
角度单位混用 Math.sin(45) 不是 sin(45°) 统一用弧度，或用 Cesium.Math.toRadians 转换
认为地球是正球 长距离测量误差可达 0.5% 使用 WGS84 椭球模型

所有空间计算的第一步：先把数据统一到合适的坐标系。

类型类单位适用场景
经纬度 Number 数组 / 对象 度（°） 用户输入、GeoJSON、配置文件
地理坐标（弧度） Cesium.Cartographic 弧度（rad） 椭球面计算、高程采样
三维世界坐标 Cesium.Cartesian3 米（m） 渲染、距离、碰撞检测
屏幕坐标 Cesium.Cartesian2 像素（px） 鼠标位置、屏幕拾取

💡 坐标系关系图

// 1. 用户最熟悉的：经纬度（度）
const beijingDeg = { lon: 116.4074, lat: 39.9042, height: 50 };

// 2. 椭球面计算用：Cartographic（弧度）
const beijingRad = Cesium.Cartographic.fromDegrees(
  116.4074, 39.9042, 50
);
// beijingRad.longitude ≈ 2.0316 rad, latitude ≈ 0.6965 rad

// 3. 三维渲染用：Cartesian3（米，地心坐标系）
const beijingCar = Cesium.Cartesian3.fromDegrees(
  116.4074, 39.9042, 50
);
// x, y, z 以地球质心为原点，Z 轴指向北极
ℹ️
地心惯性系 vs 地固系：Cesium 默认使用地固坐标系（ECEF），即随地球一起转动。做卫星轨道模拟时才需要切换到惯性系（ICRF）。

核心

📝 常用转换代码

// ===== 度数 → Cartesian3（最常用）=====
const position = Cesium.Cartesian3.fromDegrees(
  116.4074, 39.9042, 100
);

// ===== 度数数组 → Cartesian3 数组 =====
const positions = Cesium.Cartesian3.fromDegreesArrayHeights([
  116.40, 39.90, 0,
  116.41, 39.91, 0,
  116.42, 39.90, 0
]);

// ===== Cartesian3 → Cartographic =====
const carto = Cesium.Cartographic.fromCartesian(position);
console.log(
  Cesium.Math.toDegrees(carto.longitude),
  Cesium.Math.toDegrees(carto.latitude),
  carto.height
);

// ===== Cartographic → Cartesian3 =====
const carAgain = Cesium.Ellipsoid.WGS84.cartographicToCartesian(carto);

// ===== 批量转换 =====
const carts = Cesium.Ellipsoid.WGS84.cartographicArrayToCartesianArray([carto1, carto2]);
💡
性能提示：批量转换（带 Array 后缀的方法）比循环单次转换快得多，因为内部可以复用临时对象，减少 GC 压力。

🎯 屏幕坐标 ↔ 世界坐标

点击地球时拿到的是屏幕像素，需要转成三维世界坐标才能做量算：

handler.setInputAction(function onClick(movement) {
  const cartesian = viewer.scene.pickPosition(movement.position);
  if (Cesium.defined(cartesian)) {
    const carto = Cesium.Cartographic.fromCartesian(cartesian);
    console.log(
      '经度:', Cesium.Math.toDegrees(carto.longitude).toFixed(4),
      '纬度:', Cesium.Math.toDegrees(carto.latitude).toFixed(4),
      '高度:', carto.height.toFixed(1)
    );
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

🧮 向量运算示例

const a = new Cesium.Cartesian3(1, 2, 3);
const b = new Cesium.Cartesian3(4, 5, 6);
const result = new Cesium.Cartesian3();

// 加法：result = a + b（避免创建新对象，高性能）
Cesium.Cartesian3.add(a, b, result);

// 距离
const dist = Cesium.Cartesian3.distance(a, b);

// 中点
Cesium.Cartesian3.add(a, b, result);
Cesium.Cartesian3.multiplyByScalar(result, 0.5, result);

// 判断两个浮点数是否近似相等（三维坐标比较必用）
const same = Cesium.Cartesian3.equalsEpsilon(a, b, 1e-6);
⚠️
注意 result 参数：Cesium 大量 API 采用"结果对象"模式，传入 result 可避免每次创建新对象。在渲染循环或大量计算时，这能显著减少 GC 卡顿。

距离量算：三种场景

量算

📍 场景 A：椭球面距离（最常用）

地面点之间沿地球表面的最短路径。Cesium 提供 EllipsoidGeodesic，精度最高：

const start = Cesium.Cartographic.fromDegrees(116.4074, 39.9042);
const end   = Cesium.Cartographic.fromDegrees(121.4737, 31.2304);

const geodesic = new Cesium.EllipsoidGeodesic(start, end);
const surfaceDistance = geodesic.surfaceDistance;
console.log('北京-上海地面距离:', (surfaceDistance / 1000).toFixed(2), 'km');
// ≈ 1067 km

如果你想在地球表面画一条测地线（大圆航线），可以用 geodesic.interpolateUsingFraction：

const linePositions = [];
for (let i = 0; i <= 100; i++) {
  const c = geodesic.interpolateUsingFraction(i / 100);
  linePositions.push(Cesium.Cartesian3.fromRadians(c.longitude, c.latitude));
}
viewer.entities.add({
  polyline: {
    positions: linePositions,
    width: 2,
    material: Cesium.Color.CYAN
  }
});

✈️ 场景 B：三维空间直线距离

飞机、卫星、高楼等需要考虑高度时，用三维欧氏距离：

const p1 = Cesium.Cartesian3.fromDegrees(116.40, 39.90, 10000);
const p2 = Cesium.Cartesian3.fromDegrees(121.47, 31.23, 10000);

const straightDistance = Cesium.Cartesian3.distance(p1, p2);
console.log('三维直线距离:', (straightDistance / 1000).toFixed(2), 'km');

📐 场景 C：快速估算（Haversine）

不需要引入 Cesium 时的纯 JS 实现，适合服务端或轻量场景：

function haversine(lon1, lat1, lon2, lat2) {
  const R = 6371000; // 地球平均半径（米）
  const toRad = x => x * Math.PI / 180;
  const dLat = toRad(lat2 - lat1);
  const dLon = toRad(lon2 - lon1);
  const a = Math.sin(dLat / 2) ** 2 +
          Math.cos(toRad(lat1)) * Math.cos(toRad(lat2)) *
          Math.sin(dLon / 2) ** 2;
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  return R * c;
}

console.log(haversine(116.4074, 39.9042, 121.4737, 31.2304) / 1000);
// ≈ 1067 km（与 EllipsoidGeodesic 接近，但长距离误差稍大）
💡
选型建议：视觉展示或几百公里内用 Haversine 足够；公里级精密测量、导航、航线规划用 EllipsoidGeodesic。

面积量算与多边形处理

量算

📏 使用 PolygonGeometry 与 PolygonHierarchy

Cesium 没有直接返回"面积"的 API，但提供了 PolygonGeometry，你可以自己基于顶点计算。思路：把多边形投影到局部切平面，再用二维鞋带公式。

// 1. 定义多边形顶点（Cartesian3）
const positions = Cesium.Cartesian3.fromDegreesArray([
  116.39, 39.92,
  116.41, 39.92,
  116.41, 39.90,
  116.39, 39.90
]);

// 2. 创建 PolygonGeometry 提取三角网
const geometry = Cesium.PolygonGeometry.createGeometry(
  new Cesium.PolygonGeometry({
    polygonHierarchy: new Cesium.PolygonHierarchy(positions)
  })
);

// 3. 累加每个三角形的面积（局部平面投影后计算）
let area = 0;
const indices = geometry.indices;
const va = geometry.attributes.position.values;
for (let i = 0; i < indices.length; i += 3) {
  const idx = [indices[i], indices[i + 1], indices[i + 2]];
  const tri = idx.map(j => new Cesium.Cartesian3(va[j * 3], va[j * 3 + 1], va[j * 3 + 2]));
  area += triangleArea(tri[0], tri[1], tri[2]);
}

📐 投影到局部平面计算面积

更简洁的做法：取多边形中心为原点，建立局部东北天（ENU）坐标系，再用二维多边形面积公式：

function polygonAreaOnEllipsoid(cartesians) {
  // 1. 计算多边形中心（近似质心）
  const center = Cesium.Cartesian3.ZERO;
  Cesium.Cartesian3.add(...); // 累加后求平均
  Cesium.Cartesian3.multiplyByScalar(center, 1 / cartesians.length, center);

  // 2. 局部 ENU 基向量
  const centerCarto = Cesium.Cartographic.fromCartesian(center);
  const north = Cesium.Cartesian3.fromRadians(centerCarto.longitude, centerCarto.latitude + 0.0001);
  const east  = Cesium.Cartesian3.fromRadians(centerCarto.longitude + 0.0001, centerCarto.latitude);

  const up    = Cesium.Cartesian3.normalize(center, new Cesium.Cartesian3());
  const e     = Cesium.Cartesian3.normalize(Cesium.Cartesian3.subtract(east, center, new Cesium.Cartesian3()), new Cesium.Cartesian3());
  const n     = Cesium.Cartesian3.normalize(Cesium.Cartesian3.subtract(north, center, new Cesium.Cartesian3()), new Cesium.Cartesian3());

  // 3. 投影到 ENU 平面并算鞋带公式面积
  let sum = 0;
  for (let i = 0; i < cartesians.length; i++) {
    const v = Cesium.Cartesian3.subtract(cartesians[i], center, new Cesium.Cartesian3());
    const vx = Cesium.Cartesian3.dot(v, e);
    const vy = Cesium.Cartesian3.dot(v, n);
    const next = cartesians[(i + 1) % cartesians.length];
    const vn = Cesium.Cartesian3.subtract(next, center, new Cesium.Cartesian3());
    const vnx = Cesium.Cartesian3.dot(vn, e);
    const vny = Cesium.Cartesian3.dot(vn, n);
    sum += vx * vny - vnx * vy;
  }
  return Math.abs(sum) / 2;
}
ℹ️
小范围适用：局部平面投影法在几千米范围内误差很小；跨城市或跨省的大范围多边形，应使用球面多边形面积公式。

高度与高程采样

量算

📈 三种"高度"概念

概念含义获取方式
椭球高度（ellipsoidal） 相对于 WGS84 椭球面的高度 Cartographic.height
地形高度（terrain height） 相对于真实地面的高度 Cesium.sampleTerrain
海拔高度（orthometric） 相对于大地水准面的高度（常说"海拔"） 需结合大地水准面模型转换

📍 sampleTerrain 与 clampToGround

// 查询多个坐标点的地形高度
const positions = [
  Cesium.Cartographic.fromDegrees(116.40, 39.90),
  Cesium.Cartographic.fromDegrees(116.41, 39.91)
];

const updated = await Cesium.sampleTerrainMostDetailed(
  viewer.terrainProvider,
  positions
);

updated.forEach(c => {
  console.log(
    'lon:', Cesium.Math.toDegrees(c.longitude).toFixed(4),
    'height:', c.height.toFixed(1)
  );
});

// 让 Entity 贴地
viewer.entities.add({
  position: Cesium.Cartesian3.fromDegrees(116.40, 39.90),
  point: { pixelSize: 10, color: Cesium.Color.RED },
  heightReference: Cesium.HeightReference.CLAMP_TO_GROUND
});
💡
高度分析：建筑高度 = 建筑顶部椭球高度 - 地面地形高度。先用 sampleTerrain 取地面高程，再减去 Entity 的底部高度即可得净高。

方位角与目标点计算

导航

🧭 计算两点方位角

方位角（Bearing / Azimuth）是从正北方向顺时针旋转到目标方向的角度：

function bearing(lon1, lat1, lon2, lat2) {
  const toRad = x => x * Math.PI / 180;
  const dLon = toRad(lon2 - lon1);
  const y = Math.sin(dLon) * Math.cos(toRad(lat2));
  const x = Math.cos(toRad(lat1)) * Math.sin(toRad(lat2)) -
          Math.sin(toRad(lat1)) * Math.cos(toRad(lat2)) * Math.cos(dLon);
  let brng = Math.atan2(y, x) * 180 / Math.PI;
  return (brng + 360) % 360;
}

console.log(bearing(116.4074, 39.9042, 121.4737, 31.2304));
// ≈ 157°（北京看上海，东南方向）

🌟 已知起点、方位角、距离，求目标点

function destinationPoint(lon, lat, distance, bearingDeg) {
  const R = 6371000;
  const toRad = x => x * Math.PI / 180;
  const toDeg = x => x * 180 / Math.PI;
  const lat1 = toRad(lat);
  const lon1 = toRad(lon);
  const brng = toRad(bearingDeg);

  const lat2 = Math.asin(
    Math.sin(lat1) * Math.cos(distance / R) +
    Math.cos(lat1) * Math.sin(distance / R) * Math.cos(brng)
  );
  const lon2 = lon1 + Math.atan2(
    Math.sin(brng) * Math.sin(distance / R) * Math.cos(lat1),
    Math.cos(distance / R) - Math.sin(lat1) * Math.sin(lat2)
  );
  return { lon: toDeg(lon2), lat: toDeg(lat2) };
}

// 从北京出发，向东南 157° 走 100 km
console.log(destinationPoint(116.4074, 39.9042, 100000, 157));
💡
Cesium 原生方式：EllipsoidGeodesic 的 interpolateUsingSurfaceDistance 方法可以做同样的事，且精度更高，无需自己实现球面三角。

包围盒与可见性判断

优化

📏 AxisAlignedBoundingBox / BoundingSphere

判断一个 Entity 是否在相机视野内，或两个物体是否可能相交，先用包围盒/包围球做粗判：

// 计算一组点的包围盒
const positions = Cesium.Cartesian3.fromDegreesArrayHeights([...]);
const aabb = Cesium.AxisAlignedBoundingBox.fromPoints(positions);
console.log('中心:', aabb.center);
console.log('最小角:', aabb.minimum);
console.log('最大角:', aabb.maximum);

// 判断相机是否能看到该包围球
const sphere = Cesium.BoundingSphere.fromPoints(positions);
const cullingVolume = viewer.camera.frustum.computeCullingVolume(
  viewer.camera.position,
  viewer.camera.direction,
  viewer.camera.up
);
const visible = cullingVolume.computeVisibility(sphere) !== Cesium.Intersect.OUTSIDE;

🔍 点是否在多边形内

地理围栏、电子围栏常用。先把点与多边形投影到局部平面，再用射线法：

function pointInPolygon(point, polygon) {
  // polygon: Cartesian3[]，闭合多边形
  // 先投影到 ENU 平面得到二维坐标（省略投影代码，参考第 6 节）
  const { x, y } = projectToLocal(point);
  const pts = polygon.map(p => projectToLocal(p));

  let inside = false;
  for (let i = 0, j = pts.length - 1; i < pts.length; j = i++) {
    const xi = pts[i].x, yi = pts[i].y;
    const xj = pts[j].x, yj = pts[j].y;
    const intersect = ((yi > y) !== (yj > y)) &&
      (x < (xj - xi) * (y - yi) / (yj - yi) + xi);
    if (intersect) inside = !inside;
  }
  return inside;
}

构建交互式测量工具

实战

📐 测量工具设计思路

监听点击：用 ScreenSpaceEventHandler 获取鼠标点击的 Cartesian3。
存储节点：每点击一次，把坐标加入当前测量路径数组。
动态连线：用 CallbackProperty 实时更新 polyline positions。
计算并显示：每增加一个节点，累加各段 EllipsoidGeodesic 距离，用 Label 显示在场景里。
结束测量：双击或按 ESC 清空临时状态，保留结果 Entity。

📝 完整测量类实现

class MeasureTool {
  constructor(viewer) {
    this.viewer = viewer;
    this.active = false;
    this.points = [];
    this.entities = [];
    this.handler = new Cesium.ScreenSpaceEventHandler(viewer.scene.canvas);
  }

  start() {
    this.active = true;
    this.points = [];
    this.handler.setInputAction(e => this.onClick(e), Cesium.ScreenSpaceEventType.LEFT_CLICK);
    this.handler.setInputAction(e => this.onMove(e), Cesium.ScreenSpaceEventType.MOUSE_MOVE);
  }

  onClick(movement) {
    const cartesian = this.viewer.scene.pickPosition(movement.position);
    if (!cartesian) return;

    this.points.push(cartesian);

    // 放置节点
    this.entities.push(this.viewer.entities.add({
      position: cartesian,
      point: { pixelSize: 8, color: Cesium.Color.YELLOW }
    }));

    // 更新总距离标签
    this.updateLabel();
  }

  updateLabel() {
    let total = 0;
    for (let i = 1; i < this.points.length; i++) {
      const g = new Cesium.EllipsoidGeodesic(
        Cesium.Cartographic.fromCartesian(this.points[i - 1]),
        Cesium.Cartographic.fromCartesian(this.points[i])
      );
      total += g.surfaceDistance;
    }

    const last = this.points[this.points.length - 1];
    const text = total > 1000
      ? `总距离: ${(total/1000).toFixed(2)} km`
      : `总距离: ${total.toFixed(1)} m`;

    if (this.labelEntity) {
      this.labelEntity.position = last;
      this.labelEntity.label.text = text;
    } else {
      this.labelEntity = this.viewer.entities.add({
        position: last,
        label: {
          text: text,
          font: '14px sans-serif',
          fillColor: Cesium.Color.WHITE,
          pixelOffset: new Cesium.Cartesian2(0, -20)
        }
      });
    }
  }

  onMove(movement) {
    if (this.points.length === 0) return;
    const cartesian = this.viewer.scene.pickPosition(movement.endPosition);
    if (!cartesian) return;

    // 鼠标跟随的临时线
    const dynamicPositions = new Cesium.CallbackProperty(() => {
      return [this.points[this.points.length - 1], cartesian];
    }, false);

    if (!this.mouseLine) {
      this.mouseLine = this.viewer.entities.add({ polyline: { positions: dynamicPositions, width: 2 } });
    }
  }

  clear() {
    this.entities.forEach(e => this.viewer.entities.remove(e));
    this.viewer.entities.remove(this.labelEntity);
    this.viewer.entities.remove(this.mouseLine);
    this.points = [];
    this.entities = [];
    this.labelEntity = null;
    this.mouseLine = null;
  }
}

// 使用
const tool = new MeasureTool(viewer);
tool.start();
