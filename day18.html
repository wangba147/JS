为什么需要 DataSource

动机

🤔 两种加载数据的方式

在 CesiumJS 中，把数据放到地球上有两种方式。理解两者的区别是今天所有内容的基础：

❌ 方式 A：手动逐条 add

// 加载 100 个城市点...
for (const city of cities) {
  viewer.entities.add({
    position: Cesium.Cartesian3.fromDegrees(city.lon, city.lat),
    point: { pixelSize: 10, color: Cesium.Color.RED }
  });
}
// 问题：无法统一删除、无法控制加载状态、
// 手动循环效率低、没有"数据源"概念

✅ 方式 B：DataSource 批量加载

// 一行加载整个 GeoJSON 文件
const ds = await Cesium.GeoJsonDataSource.load(
  'data/cities.geojson',
  { stroke: Cesium.Color.WHITE }
);
viewer.dataSources.add(ds);
// 优势：统一管理、自动解析、支持异步、
// 一键移除、支持 GeoJSON/CZML/KML 等格式

💡 DataSource 的核心价值

特性手动 Entity.add()DataSource
批量加载 手动循环 自动解析，一行代码
标准格式 需自行解析 原生支持 GeoJSON/CZML/KML
统一移除 逐条 remove dataSources.remove(ds) 一键清
加载状态 手动管理 isLoading / loadingEvent
错误处理 需自行 try/catch errorEvent 事件
动态更新 手动修改属性 CZML 原生支持时间序列
💡 一句话理解：DataSource 是 Entity 的批量工厂。它读取标准格式数据文件，自动创建并管理一系列 Entity。你不需要关心每个 Entity 怎么创建，只需要告诉 DataSource "加载这个文件"，它会自动完成所有事情。

3

GeoJsonDataSource：矢量数据之王

核心

🌐 什么是 GeoJSON？

GeoJSON 是最通用的地理空间数据交换格式。它是一个 JSON 对象，用标准化的方式描述点、线、面和它们的属性。几乎所有 GIS 软件和 Web 地图库都支持 GeoJSON。

// 一个 GeoJSON FeatureCollection 示例
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": {
        "type": "Point",
        "coordinates": [116.3972, 39.9075]  // [经度, 纬度]
      },
      "properties": {
        "name": "北京",
        "population": 21540000
      }
    }
  ]
}

注意：GeoJSON 的坐标顺序是 [经度, 纬度]（和 CesiumJS 内部一样），但部分工具（如 Leaflet）使用 [纬度, 经度]，容易混淆。

🔧 加载 GeoJSON 数据

// 方式 1：从 URL 加载（最常用）
const geoJsonDs = await Cesium.GeoJsonDataSource.load(
  'https://example.com/china-provinces.geojson',
  {
    stroke: Cesium.Color.WHITE,          // 线条颜色
    fill: Cesium.Color.BLUE.withAlpha(0.3), // 填充颜色（半透明）
    strokeWidth: 2,
    markerSymbol: '⬤',              // 点的标记符号
    markerColor: Cesium.Color.RED,
    markerSize: 24
  }
);
viewer.dataSources.add(geoJsonDs);

// 方式 2：从 JS 对象加载
const data = {
  type: 'FeatureCollection',
  features: [/* ... */]
};
const ds = await Cesium.GeoJsonDataSource.load(data, {
  stroke: Cesium.Color.YELLOW
});
viewer.dataSources.add(ds);

// 方式 3：控制相机飞到数据范围
const ds = await Cesium.GeoJsonDataSource.load(url, options);
viewer.dataSources.add(ds);
await viewer.flyTo(ds);  // 飞到数据源范围

🎨 GeoJSON 几何类型 → CesiumJS Entity 映射

GeoJSON 类型CesiumJS Entity 图形说明
Pointpoint + billboard自动添加文本标签（从 properties.name 读取）
MultiPoint多个 point每个点独立为一个 Entity
LineStringpolyline折线
MultiLineString多个 polyline多条折线

🖌 properties → Entity 属性映射

GeoJSON 的 properties 对象会自动映射到 Entity 的描述信息：

// GeoJSON properties 中的特殊字段：
{
  "properties": {
    "name": "北京市",           // → Entity.name（显示为标签）
    "description": "中国首都",  // → Entity.description（点击弹窗）
    "population": 21540000      // → Entity.properties.population（自定义属性）
  }
}

name 和 description 是 GeoJsonDataSource 自动识别的两个特殊字段。其他所有字段都会存入 Entity.properties，可以在交互时读取。

💡 性能提示：GeoJSON 文件可能非常大（省级行政区划可达几十 MB）。大文件建议（1）先简化几何（如用 MapShaper 降低精度），（2）转为 TopoJSON 减小体积，（3）或使用 3D Tiles / 矢量瓦片替代——GeoJSON 适合中小规模数据（< 10MB）。

CzmlDataSource：时间动态数据

核心

⏰ 什么是 CZML？

CZML（Cesium Language）是 Cesium 团队设计的JSON 格式，专门用于描述随时间变化的地理空间数据。它是 GeoJSON 的"时间增强版"——不仅有空间坐标，还有时间戳。

典型应用：卫星轨道、飞机航线、导弹轨迹、无人机编队、传感器实时数据——任何"会动的东西"都适合用 CZML。

🔧 CZML 基本结构

// 一个描述飞行轨迹的 CZML 文档
[
  {
    "id": "document",
    "name": "CZML 示例",
    "version": "1.0",
    "clock": {                        // 时间范围定义
      "interval": "2025-01-01T00:00:00Z/2025-01-01T01:00:00Z",
      "currentTime": "2025-01-01T00:00:00Z",
      "multiplier": 60               // 时间倍速（60倍）
    }
  },
  {
    "id": "flight001",
    "name": "航班 CA1234",
    "position": {
      "interpolationAlgorithm": "LAGRANGE",
      "interpolationDegree": 5,
      "referenceFrame": "INERTIAL",
      "cartesian": [
        0,  x1, y1, z1,       // time=0 时的位置
        300, x2, y2, z2,       // time=300s 时的位置
        600, x3, y3, z3,       // time=600s 时的位置
        900, x4, y4, z4        // time=900s 时的位置
      ]
    },
    "point": {
      "color": { "rgba": [255, 0, 0, 255] },
      "pixelSize": 10
    },
    "path": {
      "material": { "solidColor": { "color": { "rgba": [255, 140, 0, 200] } } },
      "width": 2,
      "leadTime": 60,               // 轨迹前导时间（显示最近 60 秒的轨迹）
      "trailTime": 300              // 轨迹尾迹时间（保留 300 秒的轨迹）
    }
  }
]

🔧 加载 CZML 数据

// 从 URL 加载 CZML
const czmlDs = await Cesium.CzmlDataSource.load(
  'https://example.com/satellite-orbit.czml'
);
viewer.dataSources.add(czmlDs);

// 加载后，CZML 可能定义了自己的时钟设置
// 检查是否需要设置时钟范围
viewer.timeline.zoomTo = viewer.clock.startTime;
viewer.timeline.zoomTo = viewer.clock.stopTime;

⚙ CZML 核心特性

特性说明
时间插值LAGRANGE（拉格朗日）或 HERMITE（埃尔米特）插值算法，在关键帧之间平滑过渡
参考系INERTIAL（惯性系）或 FIXED（地固系），影响位置计算方式
path.leadTime轨迹前导时间（秒），从当前位置向前延伸
path.trailTime轨迹尾迹时间（秒），保留历史轨迹
availability实体存在的时间范围（在这个时间之外不显示）
orientation随时间变化的姿态/朝向（如卫星天线指向）
model可以绑定随时间变化的 glTF 3D 模型
⚠️ 注意：CZML 中的时间用 Unix 时间（ISO 8601） 或 相对秒数 表示。坐标用 Cartesian3（ECEF 地心地固坐标），不是经纬度！需要用 Cesium.Cartesian3.fromDegrees() 转换后再写入 CZML。

KmlDataSource：Google Earth 数据

兼容

🌎 KML/KMZ：Google Earth 的遗产

KML（Keyhole Markup Language）是 Google Earth 使用的XML 格式地理数据文件。KMZ 是 KML 的压缩版本（ZIP 压缩）。

如果你有大量 Google Earth 的历史数据需要迁移到 CesiumJS，KmlDataSource 是最直接的途径。

🔧 加载 KML/KMZ

// 加载 KML 文件
const kmlDs = await Cesium.KmlDataSource.load(
  'data/sites.kml',
  {
    camera: viewer.scene.camera,
    canvas: viewer.scene.canvas,
    clampToGround: true     // 贴地显示
  }
);
viewer.dataSources.add(kmlDs);

// KML 支持的特性（部分）
// Placemark → Entity (point/billboard/label/polygon)
// LineString → polyline
// Polygon → polygon
// NetworkLink → 动态刷新（时间驱动）
// ScreenOverlay → 受限支持
// GroundOverlay → 受限支持
💡 实用建议：KML 格式历史悠久，很多企业仍有大量 KML 数据。但新项目建议优先使用 GeoJSON 或 CZML——它们更简单、更适合 Web 传输且支持更完整。KmlDataSource 主要用于兼容旧系统。

CustomDataSource：自定义数据加载

灵活

🛠 什么时候用 CustomDataSource？

当你的数据不是标准格式——比如从 REST API 获取的 JSON、WebSocket 实时推送、数据库查询结果——CustomDataSource 就是最好的容器。

它本质上是一个命名的 EntityCollection，提供统一的数据源管理接口。

🔧 基本用法

// 创建自定义数据源
const myDs = new Cesium.CustomDataSource('实时传感器');
viewer.dataSources.add(myDs);

// 从 API 获取数据并添加到数据源
async function loadSensorData() {
  const response = await fetch('https://api.example.com/sensors');
  const sensors = await response.json();

  sensors.forEach(s => {
    myDs.entities.add({
      name: s.name,
      position: Cesium.Cartesian3.fromDegrees(s.lon, s.lat, s.alt),
      point: {
        pixelSize: 8,
        color: s.active ? Cesium.Color.GREEN : Cesium.Color.RED
      },
      label: {
        text: s.name,
        font: '12px sans-serif'
      },
      properties: s  // 保存原始数据，供交互使用
    });
  });
}

// 一键关闭/删除整个数据源
myDs.show = false;                          // 隐藏
viewer.dataSources.remove(myDs, true);        // 移除并销毁
💡 最佳实践：即使你的数据很简单，也建议用 CustomDataSource 包装而不是直接操作 viewer.entities。因为它让你能按"数据源"维度管理——显隐、移除、状态监听都在一个对象上完成。这在大项目中至关重要。

DataSourceCollection：统一管理

管理

📊 viewer.dataSources — 所有数据源的家

viewer.dataSources 是一个 DataSourceCollection，管理着场景中所有的数据源。每个数据源在这里都是平等的——无论它是 GeoJSON、CZML、KML 还是 CustomDataSource。

🔧 DataSourceCollection 核心操作

方法 / 属性说明
add(dataSource)添加一个数据源到场景中
remove(dataSource, destroy)移除数据源，可选是否销毁
removeAll(destroy)移除所有数据源
contains(dataSource)判断是否包含某数据源
indexOf(dataSource)获取数据源索引
get(index)按索引获取数据源
length数据源数量
dataSourceAdded数据源添加事件（Event）
dataSourceRemoved数据源移除事件（Event）
dataSourceMoved数据源顺序变化事件（Event）

🔧 实战：数据源管理器

// 封装一个简单的数据源管理器
class DataSourceManager {
  constructor(viewer) {
    this._viewer = viewer;
    this._sources = new Map();  // name → DataSource
  }

  async loadGeoJSON(name, url, options = {}) {
    const ds = await Cesium.GeoJsonDataSource.load(url, options);
    ds.name = name;
    this._viewer.dataSources.add(ds);
    this._sources.set(name, ds);
    return ds;
  }

  toggle(name) {
    const ds = this._sources.get(name);
    if (ds) ds.show = !ds.show;
    return ds?.show;
  }

  remove(name) {
    const ds = this._sources.get(name);
    if (ds) {
      this._viewer.dataSources.remove(ds, true);
      this._sources.delete(name);
    }
  }

  list() {
    return [...this._sources.entries()].map(([name, ds]) => ({
      name, show: ds.show,
      entityCount: ds.entities.values.length
    }));
  }
}
8

交互事件：ScreenSpaceEventHandler

核心

👆 从"看"到"交互"

前面 5 天我们一直在"看"地球。今天开始学习"点"地球——CesiumJS 提供了强大的事件系统来响应用户的鼠标和触摸操作。

ScreenSpaceEventHandler 是 CesiumJS 中处理屏幕空间事件的核心类。它监听鼠标、触摸、键盘事件，并让你对屏幕上的位置进行操作。

🗃 ScreenSpaceEventType 事件类型大全

事件类型触发时机常用度
LEFT_CLICK鼠标左键单击★★★★★
LEFT_DOUBLE_CLICK鼠标左键双击★★★
LEFT_DOWN / LEFT_UP鼠标左键按下/释放★★
RIGHT_CLICK鼠标右键单击★★★★
MOUSE_MOVE鼠标移动★★★★
WHEEL鼠标滚轮★★
PINCH_START / PINCH_END / PINCH_MOVE触摸捏合（移动端）★★

🔧 基本用法

// 创建事件处理器
const handler = new Cesium.ScreenSpaceEventHandler(
  viewer.scene.canvas  // 绑定到 Canvas 元素
);

// 监听鼠标左键点击
handler.setInputAction(function(movement) {
  // movement.position: {x, y} — 屏幕坐标
  console.log('点击位置:', movement.position);
  console.log('屏幕坐标: x=' + movement.position.x + ', y=' + movement.position.y);
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

// 监听鼠标移动
handler.setInputAction(function(movement) {
  // movement.endPosition: 鼠标当前位置
  // movement.startPosition: 鼠标起始位置
}, Cesium.ScreenSpaceEventType.MOUSE_MOVE);

// 移除特定事件监听
handler.removeInputAction(Cesium.ScreenSpaceEventType.LEFT_CLICK);

// 销毁整个处理器
handler.destroy();
⚠️ 重要：ScreenSpaceEventHandler 绑定的位置是像素坐标（屏幕 2D 坐标），左上角为 (0,0)。要获取 3D 地理坐标，需要使用 Picking（拾取）机制，见下一节。

拾取（Picking）：scene.pick / drillPick

核心

🎯 什么是 Picking？

Picking 就是"射线检测"——从屏幕上鼠标的位置发射一条射线，穿过 3D 场景，看这条射线击中了什么。这是所有 3D 交互的基础。

CesiumJS 提供了两个核心拾取方法：

  • scene.pick(position) — 拾取第一个被击中的对象（最前面的）
  • scene.drillPick(position) — 拾取所有被射线穿过的对象（按从前到后排序）

🔧 scene.pick() — 拾取最前面的对象

// scene.pick(windowPosition, width, height)
// 返回 Cesium.PickedObject 或 undefined

handler.setInputAction(function(movement) {
  const picked = viewer.scene.pick(movement.position);
  if (Cesium.defined(picked)) {
    // picked.id       → Entity 对象（如果有）
    // picked.primitive → Primitive 对象
    console.log('拾取到: ', picked.id?.name);
  } else {
    console.log('未拾取到任何对象');
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

🔧 scene.drillPick() — 穿透拾取

// 当多个对象重叠时，drillPick 能找到所有被射线穿过的对象
handler.setInputAction(function(movement) {
  const pickedObjects = viewer.scene.drillPick(movement.position);
  if (pickedObjects.length > 0) {
    console.log('拾取到 ' + pickedObjects.length + ' 个对象:');
    pickedObjects.forEach((p, i) => {
      console.log(`  ${i+1}. ${p.id?.name || '(未命名)'}`);
    });
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

🌐 屏幕坐标 → 地理坐标

// 除了拾取对象，还可以获取地球表面的坐标

// 1. 获取椭球体表面坐标（不考虑地形）
const cartesian = viewer.scene.pickEllipsoid(movement.position);
if (cartesian) {
  const cartographic = Cesium.Cartographic.fromCartesian(cartesian);
  const lon = Cesium.Math.toDegrees(cartographic.longitude);
  const lat = Cesium.Math.toDegrees(cartographic.latitude);
  console.log(`点击位置: 经度=${lon.toFixed(4)}, 纬度=${lat.toFixed(4)}`);
}

// 2. 获取地表坐标（考虑地形，需要地形数据已加载）
const cartesianOnTerrain = viewer.scene.pickPosition(movement.position);
// 如果该位置地形数据尚未加载，返回 undefined
💡 区分三个 pick 方法：scene.pick() 拾取 3D 对象（Entity、Primitive），pickEllipsoid() 拾取椭球体表面坐标（忽略地形），pickPosition() 拾取实际地表坐标（含地形高程）。大多数情况下用 pick() 检测点击了什么，用 pickEllipsoid() 计算经纬度。

Entity 点击与悬停交互

实战

👆 点击 Entity 显示信息

// 完整示例：点击城市标注显示详细信息
const handler = new Cesium.ScreenSpaceEventHandler(viewer.canvas);

handler.setInputAction(function(movement) {
  const picked = viewer.scene.pick(movement.position);

  if (Cesium.defined(picked) && picked.id instanceof Cesium.Entity) {
    const entity = picked.id;

    // 弹出 InfoBox（CesiumJS 内置信息面板）
    viewer.selectedEntity = entity;

    // 或者自定义处理
    console.log('点击了:', entity.name);
    console.log('属性:', entity.properties);
  } else {
    viewer.selectedEntity = undefined; // 取消选中
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

设置 viewer.selectedEntity 后，CesiumJS 会自动在 InfoBox 面板中显示 Entity 的 name 和 description。你可以自定义 description 为 HTML 来展示丰富的属性信息。

👀 鼠标悬停高亮

// 悬停时改变 Entity 的颜色（高亮效果）
let hoveredEntity = null;

handler.setInputAction(function(movement) {
  const picked = viewer.scene.pick(movement.endPosition);
  const entity = Cesium.defined(picked) ? picked.id : null;

  if (hoveredEntity !== entity) {
    // 还原上一个高亮的 Entity
    if (hoveredEntity) {
      hoveredEntity.point.color = Cesium.Color.WHITE.clone();
    }
    // 高亮新的 Entity
    if (entity && entity.point) {
      entity.point.color = Cesium.Color.YELLOW.clone();
    }
    hoveredEntity = entity;
  }
}, Cesium.ScreenSpaceEventType.MOUSE_MOVE);

综合实战：交互式数据浏览器

实战

🌐 场景描述

构建一个完整的交互式数据浏览器：从 GeoJSON 加载城市数据，支持点击查看详情，支持鼠标悬停高亮，提供图层面板控制显隐。

🔧 完整代码

// ==========================================
// 交互式城市数据浏览器
// ==========================================

const viewer = new Cesium.Viewer('cesiumContainer', {
  infoBox: true,  // 启用内置信息面板
  selectionIndicator: true
});

// --- 1. 加载城市数据 ---
const citiesGeoJSON = {
  type: 'FeatureCollection',
  features: [
    {
      type: 'Feature',
      geometry: { type: 'Point', coordinates: [116.3972, 39.9075] },
      properties: { name: '北京', population: 21540000, rank: '首都' }
    },
    {
      type: 'Feature',
      geometry: { type: 'Point', coordinates: [121.4737, 31.2304] },
      properties: { name: '上海', population: 24870000, rank: '直辖市' }
    },
    {
      type: 'Feature',
      geometry: { type: 'Point', coordinates: [113.2644, 23.1291] },
      properties: { name: '广州', population: 18680000, rank: '省会' }
    }
  ]
};

// --- 2. 加载并添加数据源 ---
let cityDs;
Cesium.GeoJsonDataSource.load(citiesGeoJSON, {
  markerColor: Cesium.Color.DODGERBLUE,
  markerSize: 30
}).then(ds => {
  cityDs = ds;
  viewer.dataSources.add(ds);
  return viewer.flyTo(ds);
});

// --- 3. 交互处理 ---
const handler = new Cesium.ScreenSpaceEventHandler(viewer.canvas);
let lastHovered = null;

// 点击事件：选中并显示详情
handler.setInputAction(function(movement) {
  const picked = viewer.scene.pick(movement.position);
  if (Cesium.defined(picked)) {
    const entity = picked.id;
    // 自定义 description
    entity.description = `
      <h3>${entity.name}</h3>
      <p>人口: ${(entity.properties.population / 10000).toFixed(0)} 万</p>
      <p>级别: ${entity.properties.rank}</p>
    `;
    viewer.selectedEntity = entity;
  } else {
    viewer.selectedEntity = undefined;
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

// 悬停事件：高亮效果
handler.setInputAction(function(movement) {
  const picked = viewer.scene.pick(movement.endPosition);
  const entity = Cesium.defined(picked) ? picked.id : null;

  if (lastHovered && lastHovered.billboard) {
    lastHovered.billboard.color = Cesium.Color.DODGERBLUE;
  }
  if (entity && entity.billboard) {
    entity.billboard.color = Cesium.Color.YELLOW;
  }
  lastHovered = entity;
}, Cesium.ScreenSpaceEventType.MOUSE_MOVE);
