ES6 是什么？版本一览

OVERVIEW

ES（ECMAScript）是 JavaScript 的语言规范。ES6（又名 ES2015） 发布于 2015 年，是 JS 历史上最大的一次更新，引入了类、模块、箭头函数、Promise、解构等大量特性。如今每年发布一个新版本。

📅 JavaScript 重要版本时间线

版本年份关键特性
ES52009 严格模式 (strict mode)、JSON、数组方法 (forEach/map/filter/reduce)
ES6 (ES2015)2015 let/const、箭头函数、模板字符串、解构、类、模块、Promise、Symbol、Map/Set、展开运算符
ES20162016 Array.includes()、指数运算符 (**)
ES20172017 async/await、Object.values/entries、字符串填充
ES20182018 对象展开运算符、Promise.finally、异步迭代
ES20202020 可选链 (?.）、空值合并 (??)、BigInt、动态 import
ES20212021 逻辑赋值运算符 (&&=/||=/??=)、String.replaceAll
💡
实用原则：今天着重学习日常开发最常用的 ES6+ 特性——let/const、箭头函数、模板字符串、解构、展开运算符、可选链和空值合并。这些都是 CesiumJS 源码和现代前端项目中随处可见的写法。

let 与 const：告别 var

VARIABLES

var 是 ES5 时代的声明方式，有变量提升和函数作用域两个"坑"。let 和 const 解决了这些问题。

❌ var 的三大问题

🔴 问题 1：变量提升

console.log(name); // undefined（不是报错！）
var name = "张三";

// var 声明被"提升"到顶部
// 等价于:
var name;           // 提升
console.log(name);  // undefined
name = "张三";

🟢 let/const 不会提升

console.log(name); // ❌ ReferenceError!
let name = "张三";

// let/const 存在"暂时性死区"
// 声明前访问会报错，更安全

🔴 问题 2：函数作用域

// var 是函数作用域
if (true) {
  var x = 10;
}
console.log(x); // 10（泄漏到外部！）

for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 输出: 3, 3, 3（共享同一个 i）

🟢 let 是块级作用域

// let 是块级作用域
if (true) {
  let y = 10;
}
// console.log(y);  // ❌ ReferenceError

for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 输出: 0, 1, 2（每轮独立 i）

🔴 问题 3：重复声明

var a = 1;
var a = 2; // 不报错，很难排查
var a = 3; // 覆盖了前面的值

🟢 let/const 禁止重复声明

let a = 1;
// let a = 2;  // ❌ SyntaxError
// 编译时就报错，bug 无处藏身

🔒 const：声明常量

const PI = 3.14159;
// PI = 3;  // ❌ TypeError: Assignment to constant variable

// const 必须初始化
// const x;  // ❌ SyntaxError

// ⚠️ 注意：const 不能重新赋值，但对象的属性可以修改
const user = { name: "张三", age: 28 };
user.age = 29;    // ✅ 允许！修改属性不是重新赋值
user.email = "zhang@test.com"; // ✅ 允许！添加属性
// user = {};  // ❌ 不允许！重新赋值整个对象

// 真正"冻结"对象用 Object.freeze()
const frozen = Object.freeze({ x: 1 });
// frozen.x = 2;  // ❌ 静默失败（严格模式下报错）

📋 使用建议

声明何时使用示例
const 默认首选——不会变的变量 配置常量、函数引用、API 地址
let 需要重新赋值时 循环计数器、累加值、状态变量
var 永远别用——除非维护旧代码 历史遗留项目
💡
CesiumJS 源码风格：CesiumJS 几乎全部使用 const 和 let。例如定义地球椭球体常量 const WGS84 = new Ellipsoid(...)，定义变量数据 let vertices = []。这是现代项目的标配。

箭头函数（Arrow Functions）

SYNTAX

箭头函数是 ES6 最受欢迎的语法之一，更简洁的函数写法，且自动绑定 this。

📝 基本语法

// 传统函数
function add(a, b) { return a + b; }

// 箭头函数（等价写法）
const add = (a, b) => { return a + b; };

// 只有一个表达式时可省略 {} 和 return
const add = (a, b) => a + b;

// 只有一个参数时可省略 ()
const double = n => n * 2;

// 没有参数时必须写 ()
const sayHi = () => "Hello!";

// 返回对象字面量需要用 () 包裹
const createUser = name => ({ name, age: 18 });
// 不包裹的话 {} 会被当成函数体

🎯 核心区别：this 绑定

这是箭头函数和普通函数最关键的区别——箭头函数没有自己的 this，它继承自外层作用域（词法作用域）。

🔴 普通函数：this 会变

const obj = {
  name: "地球",
  planets: ["水星", "金星"],
  list() {
    this.planets.forEach(function(p) {
      console.log(this.name, p);
      // this 指向 window/undefined
      // 输出: undefined 水星...
    });
  }
};
obj.list();

🟢 箭头函数：this 不变

const obj = {
  name: "地球",
  planets: ["水星", "金星"],
  list() {
    this.planets.forEach(p => {
      console.log(this.name, p);
      // this 指向 obj
      // 输出: 地球 水星, 地球 金星
    });
  }
};
obj.list();

⚠️ 箭头函数的限制

限制说明
没有自己的 this不能用作构造函数（不能 new）
没有 arguments需用剩余参数 ...args 代替
不能用作生成器不能用 yield
没有 prototype箭头函数没有原型属性
ℹ️
使用场景：在回调、数组方法（map/filter/reduce）、事件监听器中大量使用箭头函数。但定义对象方法时用方法简写（method() {}）更合适。

模板字符串（Template Literals）

STRINGS

用反引号 ` 替代引号，支持多行字符串和内嵌表达式。

🔤 三大核心能力

① 字符串插值

const name = "张三";
const age = 28;
const arr = [1, 2, 3];

// 旧方式
const old = "我叫 " + name + ", 今年 " + age + " 岁";

// 模板字符串：${} 里可以写任意 JS 表达式
const msg = `我叫 ${name}, 明年 ${age + 1} 岁`;
const calc = `数组中最大: ${Math.max(...arr)}`;
const ternary = `${age >= 18 ? "成年" : "未成年"}`;

② 多行字符串

// 旧方式：用 \n 拼接
const oldWay = "第一行\n第二行\n第三行";

// 新方式：换行直接写
const html = `
  <div class="card">
    <h3>${name}</h3>
    <p>${age} 岁</p>
  </div>
`;

③ 标签模板（Tagged Template）

// 自定义标签函数处理模板字符串
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) =>
    result + str + (values[i] ? `<strong>${values[i]}</strong>` : "")
  , "");
}

const result = highlight`价格: ${99}, 数量: ${3}`;
// "价格: <strong>99</strong>, 数量: <strong>3</strong>"
ℹ️
CesiumJS 中广泛应用：构建 HTML 信息框、生成 Tile URL、拼接 shader 代码等场景都用到模板字符串，比传统拼接清晰得多。

解构赋值（Destructuring）

PATTERN

从数组或对象中"解包"值，一次性赋给多个变量。代码极简且可读性强。

📦 对象解构

const entity = {
  name: "北京塔",
  position: { lon: 116.40, lat: 39.90, alt: 500 },
  color: "red"
};

// 基本解构
const { name, color } = entity;
console.log(name, color);  // 北京塔 red

// 重命名（: 取别名）
const { name: title, color: hue } = entity;
console.log(title, hue);  // 北京塔 red

// 默认值（= 设置默认）
const { visible = true, opacity = 1.0 } = entity;
console.log(visible, opacity);  // true 1.0

// 嵌套解构
const { position: { lon, lat } } = entity;
console.log(lon, lat);  // 116.40 39.90

// 剩余属性（...rest）
const { name: n, ...rest } = entity;
console.log(rest);  // { position: {...}, color: "red" }

📋 数组解构

const coords = [116.40, 39.90, 500];

// 基本数组解构
const [longitude, latitude, altitude] = coords;
console.log(longitude, latitude, altitude);  // 116.40 39.90 500

// 跳过元素
const [first, , third] = coords;
console.log(first, third);  // 116.40 500

// 默认值
const [a, b, c, d = 0] = coords;
console.log(d);  // 0（数组只有3个元素）

// 交换变量（无需临时变量！）
let x = 1, y = 2;
[x, y] = [y, x];
console.log(x, y);  // 2 1

// 剩余元素
const [head, ...tail] = [1, 2, 3, 4];
console.log(head, tail);  // 1 [2, 3, 4]

🎯 解构在函数参数中的应用

// 旧方式：参数一长串
function flyToOld(data) {
  const lon = data.lon;
  const lat = data.lat;
  const alt = data.alt || 10000;
  const duration = data.duration || 2;
  // ...
}

// 新方式：直接解构 + 默认值
function flyTo({ lon, lat, alt = 10000, duration = 2 }) {
  console.log(`飞向 ${lon},${lat} 高度${alt}m 耗时${duration}s`);
}

// 调用时传一个对象
flyTo({ lon: 116.4, lat: 39.9 });  // alt 和 duration 使用默认值
💡
CesiumJS 经典用法：flyTo、lookAt 等相机操作大量使用对象解构参数，如 viewer.camera.flyTo({ destination, orientation, duration })，只传需要的参数，其余走默认值。

展开运算符与剩余参数

SPREAD / REST

... 语法是 ES6 最灵活的特性之一：

  • 展开（Spread）：把数组/对象"拆开"
  • 剩余（Rest）：把多个值"收拢"为数组

🌸 展开运算符（Spread）

// 数组合并
const arr1 = [1, 2];
const arr2 = [3, 4];
const merged = [...arr1, ...arr2];
console.log(merged);  // [1, 2, 3, 4]

// 数组复制（浅拷贝）
const copy = [...arr1];

// 在数组任意位置插入
const inserted = [0, ...arr1, 99];
console.log(inserted);  // [0, 1, 2, 99]

// 函数传参
const nums = [5, 12, 3];
console.log(Math.max(...nums));  // 12（等价于 Math.max(5,12,3)）

🔄 对象展开（ES2018）

// 对象合并
const defaults = { theme: "dark", fontSize: 14 };
const userSettings = { fontSize: 18, language: "zh" };

const final = { ...defaults, ...userSettings };
console.log(final);
// { theme: "dark", fontSize: 18, language: "zh" }
// 后面的覆盖前面的同名字段

// 浅拷贝对象
const clone = { ...final };

// 不可变更新（重要！）
const state = { count: 1, items: ["a", "b"] };
const newState = { ...state, count: state.count + 1 };
// state 不变，newState 是新的对象

📥 剩余参数（Rest）

// 函数剩余参数：捕获所有额外参数为数组
function logAll(tag, ...messages) {
  console.log(`[${tag}]`, ...messages);
}
logAll("DEBUG", "a", "b", "c");
// [DEBUG] a b c

// 解构 + 剩余
const [first, second, ...others] = [10, 20, 30, 40, 50];
console.log(first, second, others);
// 10 20 [30, 40, 50]

// 替代 arguments 对象（箭头函数中格外有用）
const sum = (...nums) => nums.reduce((a, b) => a + b, 0);
console.log(sum(1, 2, 3, 4));  // 10
ℹ️
Spread vs Rest（同一符号，不同位置）：在使用侧（如函数调用、字面量）是展开；在定义侧（如函数参数、解构模式）是剩余。...arr 拆开，...args 收拢。

默认参数与增强对象字面量

DEFAULTS

🎯 函数默认参数

// 旧方式：在函数体里处理
function old(x, y) {
  y = y || 10;   // ⚠️ 如果 y=0，"或"会跳过
  return x + y;
}

// ES6 方式：直接在参数列表默认值
function add(x, y = 10) {
  return x + y;
}
console.log(add(5));      // 15
console.log(add(5, 0));   // 5（不会被跳过了）

// 默认值支持表达式和前序参数
function createPoint(x, y = x, z = x + y) {
  return [x, y, z];
}
console.log(createPoint(3)); // [3, 3, 6]

📦 增强对象字面量

const name = "CesiumJS";
const version = 1.119;

// ES6 增强写法
const app = {
  // ① 属性简写
  name,        // 等价于 name: name
  version,     // 等价于 version: version

  // ② 方法简写
  start() {   // 等价于 start: function()
    return `${this.name} v${this.version}`;
  },

  // ③ 动态属性名
  ["get" + "Name"]() {
    return this.name;
  }
};

console.log(app.start());     // CesiumJS v1.119
console.log(app.getName());    // CesiumJS

可选链与空值合并（ES2020）

MODERN

这两个操作符解决"访问可能不存在的属性"和"设置默认值"的两大痛点。

ES2020

🔗 可选链（Optional Chaining）`?.`

安全访问深层嵌套属性，一旦遇到 null 或 undefined 就短路返回 undefined，不会报错。

// 没有可选链：层层检查，代码冗长
// const city = user && user.address && user.address.city;

const user = {
  name: "张三",
  address: { city: "北京" }
};

// ① 属性访问
console.log(user?.address?.city);   // "北京"
console.log(user?.profile?.avatar); // undefined（不报错！）

// ② 方法调用（仅当存在时才调用）
const result = user.getName?.();
console.log(result);  // undefined（getName 不存在，不报错）

// ③ 数组索引访问
const arr = [10, 20];
console.log(arr?.[5]);  // undefined（索引越界，不报错）

❓ 空值合并（Nullish Coalescing）`??`

只有 null 和 undefined 才用默认值，0、""、false 不会被误判。

// || 的问题：0、""、false 都被当作 false
const count1 = 0 || 10;    // 10（0 被当成了 false！）
const name1  = "" || "默认";  // "默认"（空字符串被当成了 false！）

// ?? 只认 null 和 undefined
const count2 = 0 ?? 10;       // 0（正确！0 是有效值）
const name2  = "" ?? "默认";     // ""（正确！空字符串是有效值）
const count3 = null ?? 10;     // 10
const count4 = undefined ?? 10; // 10

// 组合使用：可选链 + 空值合并
const city = user?.address?.city ?? "未知城市";
console.log(city); // 兜底逻辑一气呵成

📊 ?? vs || 对比

值value || "default"value ?? "default"
null"default""default"
undefined"default""default"
0"default"0 ✅
"""default""" ✅
false"default"false ✅
NaN"default"NaN ✅
💡
黄金组合：?.?? 是处理"不确定数据结构"的最佳搭档。CesiumJS 中处理 Entity 属性、瓦片数据、API 返回值时，这两个操作符可以省去无数 if 检查。

Map、Set 与新的数据结构

COLLECTIONS

ES6 新增了 Map、Set、WeakMap、WeakSet，弥补了传统 Object 和 Array 的不足。

🗺️ Map：键值对集合

Map 的键可以是任意类型（对象、函数、原始值），且保持插入顺序。

const map = new Map();

// 设置和获取
map.set("name", "CesiumJS");
map.set(1, "数字键");
map.set({ id: 1 }, "对象键");
console.log(map.get("name")); // "CesiumJS"

// 常用方法
console.log(map.has("name"));  // true
console.log(map.size);        // 3

// 遍历
for (const [key, value] of map) {
  console.log(key, "→", value);
}

// ⚡ Map 优势：Object 的键只能是字符串/Symbol
// Map 中可以 key = 对象引用，适合做缓存映射
const cache = new Map();
const entity1 = { id: "abc" };
cache.set(entity1, { name: "北京塔", loaded: true });
console.log(cache.get(entity1).name);  // "北京塔"

🔢 Set：唯一值集合

Set 自动去重，值不可重复。

const set = new Set();
set.add(1);
set.add(2);
set.add(2); // 重复，不会添加
set.add(3);
console.log(set.size); // 3
console.log([...set]); // [1, 2, 3]

// 数组去重（经典技巧）
const arr = [1, 1, 2, 3, 3, 4];
const unique = [...new Set(arr)];
console.log(unique); // [1, 2, 3, 4]

// Set 运算
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);
// 交集
const intersection = new Set([...a].filter(x => b.has(x)));
// 并集
const union = new Set([...a, ...b]);

📋 Map vs Object 选型

特性MapObject
键类型任意类型只能用字符串/Symbol
顺序插入顺序不保证（ES6后部分保证）
性能（频繁增删）快较慢
大小size 属性需手动 Object.keys().length
JSON 序列化不支持直接 JSON.stringify原生支持

CesiumJS 关联场景

CESIUMJS

今天学的 ES6+ 语法在 CesiumJS 开发中几乎每行代码都会用到。来看几个真实场景：

🌍 场景 1：解构 + 默认参数创建实体

用对象解构和默认参数封装实体创建函数：

// 封装 CesiumJS Entity 创建（解构 + 默认参数）
function createMarker({
  name = "未命名",
  longitude = 116.4,
  latitude = 39.9,
  height = 0,
  color = "CYAN",
  label = name
} = {}) {  // = {} 允许不传参数
  return viewer.entities.add({
    name,
    position: Cesium.Cartesian3.fromDegrees(longitude, latitude, height),
    billboard: { image: "pin.png", color: Cesium.Color[color] },
    label: { text: label, font: "14px sans-serif" }
  });
}

// 调用：只传需要的参数
const marker1 = createMarker({ name: "天安门", latitude: 39.908 });
const marker2 = createMarker({ name: "陆家嘴", longitude: 121.5, color: "RED" });

📡 场景 2：展开运算符合并图层配置

CesiumJS 的图层和 ImageryProvider 配置常用展开运算符合并默认与用户配置：

// 默认图层配置
const defaultLayerConfig = {
  minimumLevel: 0,
  maximumLevel: 18,
  show: true,
  alpha: 1.0
};

// 用户自定义配置（可能只覆盖部分）
const userConfig = { minimumLevel: 5, alpha: 0.8 };

// 合并：展开 + 解构
const finalConfig = { ...defaultLayerConfig, ...userConfig };
// { minimumLevel: 5, maximumLevel: 18, show: true, alpha: 0.8 }

// 应用到 CesiumJS
const layer = viewer.imageryLayers.addImageryProvider(
  new Cesium.UrlTemplateImageryProvider({
    url: "https://tile.example.com/{z}/{x}/{y}",
    ...finalConfig  // 自动展开配置
  })
);

⏱️ 场景 3：箭头函数处理相机事件

监听相机移动时，用箭头函数自动绑定 this，避免 Scope 混乱：

class CameraTracker {
  constructor(viewer) {
    this.viewer = viewer;
    this.positions = new Map();  // 用 Map 存储位置历史
    this.trackCount = 0;

    // 箭头函数自动绑定 this，告别 .bind(this)
    viewer.camera.changed.addEventListener(() => {
      // this 指向 CameraTracker 实例
      const pos = this.viewer.camera.positionCartographic;
      const key = `${this.trackCount++}`;
      this.positions.set(key, {
        lon: Cesium.Math.toDegrees(pos?.longitude ?? 0),  // 可选链 + 空值合并
        lat: Cesium.Math.toDegrees(pos?.latitude ?? 0),
        alt: pos?.height ?? 0
      });

      console.log(
        `📍 位置 ${key}: ${lon.toFixed(2)}°E, ${lat.toFixed(2)}°N`
      );
    });
  }

  getLastPosition() {
    const keys = [...this.positions.keys()];  // 展开 Map 的键
    const lastKey = keys.at(-1);
    return this.positions.get(lastKey);
  }
}

// 场景：管理"已选中"的实体列表，用 Set 防止重复
const selectedEntities = new Set();
const entityInfoCache = new Map();

// 点击地球选中实体
handler.setInputAction((click) => {
  const picked = viewer.scene.pick(click.position);

  // 可选链：安全访问
  const entity = picked?.id;
  if (!entity) return;

  // Set.add 自动去重
  selectedEntities.add(entity);

  // Map 缓存（关联实体对象 → 元数据）
  entityInfoCache.set(entity, {
    selectedAt: new Date(),
    name: entity?.name ?? "未命名实体",  // 空值合并
    position: entity?.position?.getValue() ?? null
  });

  console.log(`已选中 ${selectedEntities.size} 个实体`);
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);

📋 今日知识点总结

  • ✅ 用 const（首选）和 let 替代 var，块级作用域更安全
  • ✅ 箭头函数更简洁，且 this 从外层继承，解决回调中 this 丢失问题
  • ✅ 模板字符串支持多行和 ${表达式} 插值，告别 (+ ) 拼接
  • ✅ 解构赋值从对象/数组中一次提取多个变量，可重命名、设默认值、嵌套解构
  • ✅ 展开运算符 ... 用于数组/对象合并与拷贝；剩余参数收集函数入参
  • ✅ 可选链 ?. 安全访问深层属性；空值合并 ?? 只对 null/undefined 兜底
  • ✅ Map 支持任意类型键且保持顺序；Set 自动去重
  • ✅ CesiumJS 源码中处处可见 ES6+ 语法：解构参数、箭头函数回调、展开运算符合并配置、可选链读取 Entity 属性
